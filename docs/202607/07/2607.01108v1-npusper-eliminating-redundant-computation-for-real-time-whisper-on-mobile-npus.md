## 摘要
我们提出了 NPUsper，这是一个通过消除冗余计算使 Whisper 在移动端 NPU 上高效运行的实时转录系统。为了避免先前流式系统中使用的大量填充（padding），NPUsper 通过解码器交叉注意力（cross-attention）中的时间模式在线检测幻觉标记（hallucinated tokens），从而允许每一轮推理以极少的结转（carryover）处理短音频输入。为了实现高效的移动端 NPU 执行，我们提出了受控展开（controlled unrolling），将自回归解码执行为 K 步块图（chunk graphs），从而消除了不必要的 KV 缓存计算并减少了图调度开销。与基准模型相比，NPUsper 在保持相当的转录准确率的同时，实现了高达 4.84 倍的逐词延迟降低、高达 33.2 倍的首字延迟（TTFT）降低以及高达 88.64% 的平均功耗降低。代码可在 https://github.com/npusper/NPUsper 获取。

## Abstract
We present NPUsper, a live transcription system that makes Whisper efficient on mobile NPUs by eliminating redundant computation. To avoid the heavy padding used by prior streaming systems, NPUsper detects hallucinated tokens online from temporal patterns in decoder cross-attention, allowing each inference round to process short audio inputs with minimal carryover. For efficient mobile-NPU execution, we propose controlled unrolling, which executes autoregressive decoding as K-step chunk graphs, removing unnecessary KV-cache computation and reducing graph-dispatch overhead. NPUsper achieves up to 4.84x lower per-word latency, up to 33.2x lower time-to-first-token (TTFT), and up to 88.64% lower average power consumption compared with baselines, while maintaining comparable transcription accuracy. The code is available at https://github.com/npusper/NPUsper.

### 1. 核心问题与整体含义（研究动机和背景）
论文针对 **Whisper 模型在移动端 NPU 上进行实时语音转录（Live Transcription）效率低下** 的核心问题展开研究。
*   **研究动机：** 尽管 Whisper 准确率高，但在移动端实时运行时面临两大挑战：
    1.  **幻觉与填充冗余：** Whisper 在处理短音频时极易产生幻觉（重复或无意义文本）。现有流式系统通过将音频填充（Padding）至 30 秒来抑制幻觉，但这导致了巨大的计算浪费。
    2.  **硬件执行不匹配：** 移动端 NPU 依赖预编译的静态图（固定张量形状），而 Whisper 的自回归解码过程具有动态增长的 KV 缓存，导致要么每步计算最大长度（冗余高），要么频繁调度不同形状的图（开销大）。

### 2. 方法论：核心思想与关键技术
NPUsper 的核心思想是**通过在线检测幻觉来消除音频填充，并利用受控展开优化 NPU 解码过程**。
*   **幻觉在线检测（Hallucination Detection）：**
    *   **核心发现：** 有效 Token 的交叉注意力（Cross-attention）在音频时间轴上呈单调前进趋势，而幻觉 Token 会出现“后向偏移”。
    *   **技术细节：** 提取解码器最后一层的头平均交叉注意力，计算连续内容 Token 间的注意力差异信号。通过定位正向峰值（$f^+$）和负向峰值（$f^-$），若 $f^+ < f^-$ 则判定为幻觉并立即停止解码。
*   **受控展开（Controlled Unrolling）：**
    *   将自回归解码过程封装为执行 $K$ 步的“块图”（Chunk Graphs）。
    *   **优势：** 相比单步调度，它减少了 NPU 图调度的频率；相比固定最大长度图，它仅计算当前步数所需的 KV 缓存，大幅降低了 MACs（乘加运算次数）。
*   **音频缓冲区管理：** 仅处理新到达的音频和极少量的结转（Carryover）上下文，通过分桶（Bucketing）策略将变长音频映射到有限的 NPU 静态图形状中。

*   **思路巧妙：** 避开了传统的“增加填充”思路，转而利用模型内部的注意力机制在线解决幻觉问题。

*   **端到端优化：** 从音频输入、幻觉检测到硬件执行层进行了全栈式优化，实用性强。

*   **模型尺寸限制：** 实验主要集中在 Whisper-base 模型，虽然验证了更大模型的幻觉检测，但在移动端 NPU 上运行 Large 模型的内存压力和实时性仍是挑战。
