## 摘要
由于视觉 Transformer (ViT) 模型的高计算需求以及现代混合 ViT 模型（包含全连接层和卷积层）的架构异构性，在边缘平台上部署这些模型仍然具有挑战性。这种异构性导致张量形状存在显著差异，需要灵活且高效的 FPGA 加速。在本文中，我们提出了 FlexViT，这是一种可重构的 FPGA 加速器，用于在资源受限的边缘设备上进行高效的 ViT 推理。FlexViT 基于 SECDA-TFLite 框架，采用软硬件协同设计方法，通过运行时 im2col 转换，将全连接层和卷积层映射到统一的高吞吐量 INT8 GEMM 引擎上。为了高效支持多样的层配置，我们提出了一种双模式数据流，通过在运行时重构计算阵列，在输入复用和权重复用之间动态切换。我们进一步引入了一种深度优先分块策略，可在单次处理中完成累加，消除了片外部分和传输并降低了内存带宽需求。我们在 PYNQ-Z2 FPGA 上实现了 FlexViT，并在代表性的 ViT 模型集上进行了评估。FlexViT 在加速器执行的层上实现了高达 2.74 倍的加速，与仅 CPU 执行相比，端到端加速比高达 1.40 倍。代码可在以下网址获取：https://github.com/gicLAB/FlexViT

## Abstract
Deploying Vision Transformer (ViT) models on edge platforms remains challenging due to their high computational demands and the architectural heterogeneity of modern hybrid ViT models, which incorporate both fully connected and convolutional layers. This heterogeneity leads to significant variation in tensor shapes, requiring flexible and efficient FPGA-based acceleration. In this paper, we present FlexViT, a reconfigurable FPGA accelerator for efficient ViT inference on resource-constrained edge devices. Built on the SECDA-TFLite framework, FlexViT employs a hardware-software co-design approach that maps both fully connected and convolutional layers onto a unified high-throughput INT8 GEMM engine using a runtime im2col transformation. To efficiently support diverse layer configurations, we propose a dual-mode dataflow that dynamically switches between input and weight reuse by reconfiguring the compute array at runtime. We further introduce a depth-first tiling strategy that completes accumulation in a single pass, eliminating off-chip partial-sum transfers and reducing memory bandwidth requirements. We implement FlexViT on a PYNQ-Z2 FPGA and evaluate it across a representative set of ViT models. FlexViT achieves up to 2.74x speedup on accelerator-executed layers, translating into up to 1.40x end-to-end speedup compared to CPU-only execution. The code is available at: https://github.com/gicLAB/FlexViT

*   **核心问题**：在边缘 FPGA 平台上部署 ViT 模型面临三大挑战：
    *   **层异构性**：现代混合 ViT（如 MobileViT）同时包含全连接（FC）和卷积（CONV）层，数据流和内存访问模式迥异。
    *   **维度差异**：不同层的张量形状变化剧烈（如“长瘦型”或“短宽型”），导致静态加速器利用率低下。
    *   **资源受限**：边缘 FPGA（如 Zynq-7000）的片上内存（BRAM）和计算单元（DSP）极其有限。
*   **整体含义**：FlexViT 旨在通过软硬件协同设计，创建一个既能处理标准 ViT 又能处理混合 ViT 的通用、高效且可重构的加速方案。

### 2. 方法论与关键技术
*   **统一计算引擎**：利用 CPU 端的运行时 `im2col` 转换，将卷积层线性化为 2D 矩阵，从而使 FC 和 CONV 层都能映射到同一个 **INT8 GEMM（通用矩阵乘法）引擎**上。
*   **双模式动态数据流**：
    *   **输入广播模式（Input-Broadcast）**：适用于输出宽度较大的层（如标准 ViT 的投影层），最大化激活值的复用。
    *   **权重广播模式（Weight-Broadcast）**：适用于空间维度较大但通道较窄的层（如混合 ViT 的卷积层），最大化权重的复用。
    *   **调度算法**：主机驱动程序根据层类型和维度，通过分析 DMA 传输开销动态选择最优模式。
*   **深度优先分块策略（Depth-first Tiling）**：
    *   通过设置足够大的片上缓冲区深度（$T_K=1024$），在本地寄存器中完成完整的点积累加。
    *   **单次处理（Single-pass）**：消除了将 32 位中间部分和（Partial-sum）写回主存的需要，仅输出最终的 8 位量化结果，大幅降低内存带宽压力。
*   **混合算术架构**：在 FPGA 上将 8 位乘法拆分，部分利用 LUT 逻辑，部分利用硬核 DSP，以在资源受限的芯片上提升计算密度。

### 6. 主要结论与发现
*   **层级加速**：在加速器负责执行的层（FC 和 CONV）上，实现了 **1.77x 至 2.74x** 的加速。

### 7. 优点

*   **端到端可验证**：代码开源，且在低成本边缘开发板上完成了真实部署，而非停留在理论仿真。

### 8. 不足与局限
*   **非线性层瓶颈**：加速器仅处理 GEMM 操作，Softmax 和 LayerNorm 等操作仍留在 CPU，限制了整体加速比的进一步提升。
