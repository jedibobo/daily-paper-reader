## 摘要
为神经网络处理器（NPU）开发高性能算子是当前工业界的关键瓶颈，要求开发者手动应对隐式的硬件约束和严格的内存层级。虽然大语言模型展现出巨大的自动化潜力，但由于根本性地缺乏硬件特定先验，它们在 NPU 任务上表现不佳。盲目移植相似 NPU 算子的代码片段或许能通过编译，但往往会因违反底层硬件约束而导致运行时崩溃和性能下降。为了解决这一问题，我们提出了 Hawk，这是一个无需训练的框架，通过三个核心模块利用硬件感知知识：(1) 运行时知识综合模块，采用三部分可执行知识表示，将错误上下文与可执行语义内在耦合；(2) 瓶颈感知知识检索模块，实现了一种二维检索范式，将查询投影到正交的语法空间和硬件对齐的语义空间中；(3) 效果驱动知识蒸馏模块，利用大语言模型驱动的语义仲裁，根据经验执行反馈，通过剪枝错误和整合冗余来持续蒸馏知识。在真实 NPU 工作负载上的广泛评估表明，Hawk 将生成准确率从 49.4% 提升至 80.0%，同时与最先进的基准相比，实现了高达 2.2 倍的执行加速。

## Abstract
Developing high-performance kernels for Neural Processing Units (NPUs) is a critical industry bottleneck, requiring developers to manually navigate implicit hardware constraints and strict memory hierarchies. While large language models offer immense automation potential, they fail catastrophically on NPUs due to a fundamental lack of hardware-specific priors. Naively transplanting code snippets from similar NPU kernels may pass the compiler, but it consistently triggers runtime crashes and performance degradation by blindly violating underlying hardware constraints. To overcome this, we introduce Hawk, a training-free framework that harnesses hardware-aware knowledge through three core modules: (1) Run-Time Knowledge Synthesis Module, which employs a Triple-Part Executable Knowledge Representation to inherently couple the error context with executable semantics; (2) Bottleneck-Aware Knowledge Retrieval Module, which implements a 2D-Retrieval paradigm to project queries into orthogonal syntactic and hardware-aligned semantic spaces; and (3) Effect-Driven Knowledge Distillation Module, which leverages LLM-driven semantic arbitration to continuously distill the knowledge by pruning errors and consolidating redundancies based on the empirical execution feedback. Extensive evaluations on real-world NPU workloads demonstrate that Hawk elevates generation accuracy from 49.4% to 80.0%, while achieving up to a 2.2x execution speedup over state-of-the-art baselines.

#### 1. 核心问题与整体含义（研究动机和背景）
在 AI 加速领域，为神经网络处理器（NPU）编写高性能算子（Kernel）是一项极具挑战的任务。与 GPU 相比，NPU（如华为昇腾 Ascend）具有更复杂的显式内存层级（Global Memory, L1, UB, L0）和严格的硬件约束。
*   **研究动机**：现有的 LLM 在生成 CUDA 代码时表现优异，但在生成 NPU 代码（如 Ascend C）时，编译成功率和功能正确率极低（仅约 13.3%）。
*   **核心瓶颈**：LLM 缺乏对 NPU 底层硬件约束（如 Unified Buffer 容量限制、指令流水线同步、数据对齐等）的先验知识。简单的代码移植或 Few-shot 提示往往会导致运行时崩溃或性能极差（比专家代码慢 20-40 倍）。
*   **整体含义**：Hawk 旨在通过一个**无需训练（Training-free）**的框架，将硬件感知的经验知识结构化并动态引入 LLM 的生成过程，从而解决 NPU 算子生成的正确性和性能问题。

#### 2. 方法论：核心思想与关键技术
Hawk 提出了一个闭环的知识利用体系，包含三个核心模块：
*   **运行时知识合成模块（Run-Time Knowledge Synthesis）**：
    *   **三部分可执行知识表示**：将知识结构化为“索引触发器（Metadata）”、“语义原理（Rationale）”和“精确代码模板（Artifact）”。这确保了 LLM 不仅能获得代码片段，还能理解其背后的硬件约束（如 Tiling 策略）。
    *   **触发机制**：当算子生成遇到困难（如多次修复失败）、发现文档与硬件行为不符或实现显著性能提升时，自动合成新知识。
*   **瓶颈感知知识检索模块（Bottleneck-Aware Knowledge Retrieval）**：
    *   **二维检索范式**：结合了**语法空间**（基于 BM25 算法匹配 API 签名和错误码）和**硬件对齐的语义空间**（基于稠密向量嵌入匹配底层瓶颈描述）。
    *   **RRF 融合**：使用互惠排名融合（Reciprocal Rank Fusion, RRF）算法整合两个维度的检索结果，确保检索到的知识既能解决编译错误，又能优化硬件性能。
*   **效果驱动知识蒸馏模块（Effect-Driven Knowledge Distillation）**：
    *   **语义仲裁**：利用 LLM 作为仲裁者，根据真实的编译和执行反馈，自动剔除错误的知识条目，合并冗余条目，并细化知识的使用边界（如特定 Tensor 大小的限制）。

#### 6. 主要结论与发现
*   **性能提升**：Hawk 将算子生成的平均准确率从 49.4% 提升至 **80.0%**。

*   **知识迁移**：证明了从简单算子（L1）中提取的硬件知识可以有效辅助复杂算子（L2）的生成，解决了 NPU 开发中的“冷启动”问题。

#### 7. 优点
*   **无需微调**：通过 RAG（检索增强生成）和 Agent 协作的方式，规避了昂贵的模型微调成本，且能快速适应硬件 API 的更新。
*   **硬件感知**：不同于通用的代码助手，Hawk 深入到了 NPU 的底层约束（如 UB 限制、HF32 模式等），真正解决了“能跑通”和“跑得快”的问题。
*   **闭环进化**：系统能够从失败中学习，通过执行反馈自动合成和修正知识，具有很强的自我演进能力。

#### 8. 不足与局限
*   **平台依赖性**：虽然方法论具有通用性，但目前的实现和实验高度依赖于华为昇腾（Ascend）生态和 CANN 工具链，在其他 NPU（如寒武纪、昆仑芯）上的表现尚待验证。
*   **冷启动依赖**：系统初期仍需要一定量的专家代码（如文中提到的 158 条初始知识）进行“预降水”，完全零知识下的表现可能受限。
*   **迭代成本**：虽然不需要训练，但多次的“生成-编译-执行-反馈”循环在实际生产环境中可能会带来一定的推理和编译时间开销。
