## 摘要
本文介绍了一种基于 MinHash 的相似度调度框架，该框架针对基于大语言模型（LLM）的神经架构搜索（NAS），在神经架构代码上构建了渐进式课程。通过在归一化的 7-gram 源代码 shingle 上使用 128 排列的 MinHash 签名，我们将参考池划分为不同的相似度区间，并以架构异质性递增的方式呈现，每一阶段的最佳 LoRA 适配器都会累积合并到主干网络中。我们在 CIFAR-10 图像分类任务的 LEMUR 基准测试中，使用 OlympicCoder-7B 对该框架进行了评估，在六个渐进式微调步骤中，每个 epoch 生成 N=15 个候选架构。在无需后处理修复的情况下，该课程在模型高相似度水平下达到了 60% 的峰值成功率。在最高多样性水平的课程与基础模型之间进行的 2*2 消融实验（包含有无部分接口修复的对比）显示，在没有修复的情况下，基础模型（47% 峰值成功率）显著优于课程模型（7% 成功率），而加入部分修复后，两者的成功率均提升至 53%。这一模式与合并层级的权重漂移逐渐抹除评估器接口先验知识的现象一致，并表明接口修复和课程调度针对的是不同的失败模式。我们进一步报告了在 SVHN 上的跨数据集迁移观察结果，其中没有经过课程预热的基础模型直接生成的峰值成功率为 27%，准确率（60.5%）远低于 CIFAR-10 的对应结果，这与 unq 系列锚点架构合成难度的增加相一致。

## Abstract
Introduce a MinHash-based similarity scheduling framework that constructs a progressive curriculum over neural architecture code for LLM-based neural architecture search (NAS). Using 128-permutation MinHash signatures over normalised 7-gram source code shingles, we partition the reference pool into similarity bands and present them in increasing architectural heterogeneity, with the best LoRA adapter from each stage merged cumulatively into the backbone. We evaluate the framework on OlympicCoder-7B within the LEMUR benchmark on CIFAR-10 image classification, generating N =15 candidate architectures per epoch across six progressive fine-tuning steps. The curriculum achieves 60% peak success rate at the high-similarity level without post-processing repair. A 2*2 ablation at the most diverse level curriculum versus base model, with versus without partial interface repair reveals that without repair the base model (47% peak SR) substantially outperforms the curriculum model (7% SR), while adding partial repair brings both to 53% SR. This pattern is consistent with merge-level weight drift progressively erasing evaluator-interface priors, and suggests that interface repair and curriculum scheduling target distinct failure modes. We further report a cross-dataset transfer observation on SVHN, where direct base-model generation without curriculum warmup yields 27% peak SR at substantially lower accuracy (60.5%) than the CIFAR-10 equivalent, consistent with the increased synthesis difficulty of the unq-family anchor architecture.

### 1. 核心问题与研究动机
*   **核心问题**：在大语言模型（LLM）驱动的神经架构搜索（NAS）中，如何有效地组织参考架构（Reference Architectures）的微调顺序，以提高生成代码的可执行性和架构质量。
*   **研究动机**：现有的 LLM-NAS 方法通常将参考架构视为无序集合。然而，不同架构在模块组成、参数路由和张量形状约定上存在巨大差异。同时暴露所有结构变体可能会破坏 LLM 生成符合评估器接口要求的代码先验。本文探索了“由易到难”（由结构相似到结构异构）的课程学习是否能提升生成的可靠性。

### 2. 方法论
*   **核心思想**：引入基于 MinHash 的相似度调度框架，将参考池划分为相似度区间，按架构异质性递增的顺序进行渐进式微调。
*   **关键技术细节**：
    *   **MinHash 相似度计算**：对源代码进行归一化处理，提取 7-gram 词块（shingles），计算 128 位排列的 MinHash 签名。通过 Jaccard 系数近似评估代码间的结构相似度。
    *   **课程分层**：将相似度分为三个等级：L1（高相似度，J∈[0.95, 0.98)）、L2（中等相似度，J∈[0.85, 0.95)）、L3（低相似度/异构，J∈[0.30, 0.85)）。
    *   **渐进式微调与合并**：在每个阶段，使用 LoRA 适配器进行微调。每一阶段结束后，将表现最好的适配器权重合并回主干模型（Backbone），再进行下一阶段训练，以保留编码约定。
    *   **自动锚点选择**：自动从数据库中选择准确率最高且在各相似度区间均有足够邻居的架构作为微调的“锚点”。

### 3. 实验设计
*   **数据集/场景**：主要在 **CIFAR-10** 图像分类任务上评估，并在 **SVHN** 上进行了跨数据集迁移观察。

*   **对比方法与消融**：
    *   **基础模型 vs. 课程模型**：对比未经微调的原始模型与经过课程微调的模型。

    *   **随机排序对比**：将课程顺序与随机打乱的微调顺序进行对比。
*   **评估指标**：成功率（SR，可执行代码占比）、峰值准确率、复合得分（SR × 峰值准确率）。

### 4. 资源与算力
*   **硬件**：单张 **NVIDIA GeForce RTX 4090 (24 GB VRAM)**。
*   **模型配置**：使用 **OlympicCoder-7B**（基于 Qwen2）。
*   **训练细节**：采用 4-bit NF4 量化加载，LoRA 微调（r=32），bfloat16 精度。每个 epoch 生成 N=15 个候选架构。评估器训练限制为 1 个 epoch，每个候选模型执行时限 8 分钟。

### 5. 实验数量与充分性
*   **实验规模**：针对 CIFAR-10 进行了 6 个阶段的渐进式微调，涵盖了多个 k 值（邻居数量）配置。
*   **充分性评价**：
    *   **初步性**：作者明确指出实验是基于**单种子（Single-seed）**、单模型、单数据集的初步观察，而非统计学上的显著性验证。
    *   **局限性**：每轮生成的样本量较小（N=15），且由于算力限制，未进行大规模重复实验。
    *   **客观性**：作者诚实地报告了课程模型在某些阶段（如无修复的 L3）表现不如基础模型的现象，并给出了合理的解释。

### 6. 主要结论与发现
*   **课程有效性**：在高相似度阶段（L1），无需后处理修复即可达到 **60% 的峰值成功率**。
*   **权重漂移风险**：在 L3 阶段，未经修复的课程模型成功率（7%）远低于基础模型（47%）。这表明连续的权重合并可能导致模型“遗忘”了最初学习到的评估器接口规范（权重漂移）。
*   **接口修复的必要性**：引入简单的规则修复（补全缺失声明）后，课程模型和基础模型的成功率均提升至 53%，说明接口修复与课程学习解决的是不同的失效模式。
*   **结构创新**：分析表明，课程微调后的模型并非简单模仿参考代码，而是能生成与锚点架构结构不同（Jaccard 相似度下降）但性能稳定的新架构。
*   **稳定性提升**：与随机排序相比，课程学习显著降低了训练过程中的波动和灾难性失败的风险。

### 7. 优点
*   **高效的相似度检索**：利用 SQLite UDF 在数据库层实现 MinHash 检索，实现了亚秒级的相似度查询，无需昂贵的深度学习嵌入计算。
*   **1-GPU 可行性**：证明了在消费级显卡上通过课程微调 LLM 进行 NAS 的可行性。
*   **结构化分析**：不仅关注准确率，还深入分析了生成代码与参考代码之间的结构相似度演变。

### 8. 不足与局限
*   **统计效力不足**：缺乏多随机种子的重复实验，结论存在偶然性风险。
*   **数据集覆盖有限**：目前仅 CIFAR-10 拥有足够密集的架构池来构建完整课程，其他数据集（如 MNIST、ImageNette）尚不具备条件。
*   **接口依赖**：模型对特定评估器接口的依赖较强，权重合并带来的接口先验丢失是该方法的一个主要技术瓶颈。
