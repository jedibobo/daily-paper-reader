## 摘要
大语言模型（LLM）推理工作负载是数据中心能耗快速增长的主要诱因。优化这些部署需要将特定的 LLM 与最高效的 GPU 相匹配，但运营商目前缺乏在不对每种组合进行详尽性能分析（profiling）的情况下实现这一目标的工具。虽然已有一些预测模型，但它们仍需要分析数据，且难以泛化到训练中未见过的硬件。为此，我们推出了 WattGPU，包含两个分别用于预测平均 GPU 功耗和词元间延迟（ITL）的预测模型。我们的方法仅利用公开的 LLM 元数据和 GPU 规格，无需硬件访问或性能分析，同时能够泛化到未见的 NVIDIA 服务器级 GPU 和 LLM。我们在包含 42 个开源 LLM（0.1B–27B 参数）和 8 个 GPU 的数据集上，通过严格的留一 GPU（leave-one-GPU-out）和留一 LLM（leave-one-LLM-out）交叉验证，在离线和服务器场景下对模型进行了评估。在未见过的 GPU 上，平均功耗模型在离线场景下的中值绝对百分比误差 $\leq3.4\%$，服务器场景下 $\leq13.5\%$；而延迟模型在服务器模式下达到 $\leq8.5\%$，两者在服务器场景下均保持了极强的 GPU 排名相关性（Kendall $\tau\geq0.76$）。与标准的基于物理原理的基准模型——用于功耗的负载缩放热设计功耗（TDP）和用于延迟的 Roofline 模型——相比，我们的模型在服务器场景下的未见 LLM-GPU 组合上将中值绝对百分比误差降低了约 4 倍，在完全未见的 GPU 上降低了约 2 倍。WattGPU 的数据和代码已在 https://github.com/maufadel/wattgpu 公开。

## Abstract
Large Language Model (LLM) inference workloads are a rapidly growing contributor to data center energy consumption. Optimizing these deployments requires matching specific LLMs to the most efficient GPUs, but operators currently lack the tools to do so without exhaustively profiling each combination. While some predictive models exist, they still require profiling data and struggle to generalize to hardware unseen during training. To address this, we introduce \textit{WattGPU}, featuring two predictive models for mean GPU power draw and Inter-Token Latency (ITL). Our approach leverages only publicly available LLM metadata and GPU specifications, eliminating the need for hardware access or profiling while enabling generalization to unseen NVIDIA server-grade GPUs and LLMs. We evaluate our models using rigorous leave-one-GPU-out and leave-one-LLM-out cross-validation on a dataset of 42 open-source LLMs (0.1B--27B parameters) and 8 GPUs under both offline and server scenarios. The mean power draw model achieves a median absolute percentage error of $\leq3.4\%$ for offline and $\leq13.5\%$ for server scenarios on unseen GPUs, while the latency model achieves $\leq8.5\%$ in server mode, both maintaining strong GPU ranking correlations for server scenarios (Kendall $τ\geq0.76$). Compared to standard physically grounded baselines -- Load-Scaled Thermal Design Power (TDP) for power draw and roofline for latency -- our models reduce median absolute percentage error by approximately 4$\times$ on unseen LLM-GPU combinations for server scenarios or approximately 2$\times$ for completely unseen GPUs. WattGPU's data and code are publicly available at https://github.com/maufadel/wattgpu.

*   **核心问题**：如何在**无需物理访问硬件**且**面对训练阶段未见过的 GPU 或 LLM** 时，准确预测推理过程中的平均功耗和延迟？

#### 2. 方法论
*   **核心思想**：利用机器学习（XGBoost）构建预测模型，仅输入公开的 GPU 规格参数和 LLM 架构元数据，实现对性能指标的端到端预测。
*   **关键技术细节**：
    *   **特征工程**：除了基础参数（如 TDP、显存带宽、参数量），引入了三个派生特征：
        *   **Boosting Ratio**：加速频率与基础频率之比，捕捉频率缩放对功耗的影响。
        *   **Bandwidth Latency**：模型权重从显存读取一次所需的理论时间。
        *   **Compute Latency ($C_{lat}$)**：假设算力利用率 100% 时计算一个词元的理论下限时间，公式为：$C_{lat} = \frac{2 \times \text{参数量} \times 10^9}{\text{GPU FP16 FLOPS}}$。
    *   **目标归一化**：为了实现跨架构泛化，功耗模型预测的是“相对功耗”（实际功耗 / TDP），而非绝对值。
    *   **算法流程**：采用梯度提升回归树（XGBoost），通过网格搜索优化超参数（如树深度、正则化系数等）。

#### 3. 实验设计
*   **数据集**：使用 **Watt Counts** 公开数据集，包含 42 个开源 LLM（规模从 0.1B 到 27B）和 8 款 NVIDIA 服务器级 GPU（V100, T4, A100, A30, L40S, L4, H100, H200）。
*   **实验场景**：
    *   **Offline（离线）**：高并发、最大吞吐量模式。
    *   **Server（服务器）**：模拟真实用户请求（泊松分布），关注低负载和高负载下的表现。
*   **对比基准（Benchmark）**：
    *   **功耗预测**：对比 Plain TDP（直接用 TDP 值）和 Load-Scaled TDP（根据负载比例缩放的 TDP）。
    *   **延迟预测**：对比 Roofline 模型（基于显存带宽的理论上限）。
*   **验证策略**：采用 5 折交叉验证、**留一 GPU 验证（LOGO）**和**留一 LLM 验证（LOLO）**，重点测试模型对完全未知硬件/模型的泛化能力。

*   **数据采集算力**：实验数据源自 Watt Counts 数据集，该数据集的采集涉及 8 种不同架构的 NVIDIA GPU 在大量 LLM 组合下的实际运行。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了 42 个模型和 8 款跨越 5 代架构（Volta 到 Hopper）的 GPU，样本量充足。
*   **充分性与公平性**：
    *   通过 LOGO 和 LOLO 验证，排除了训练集与测试集之间硬件/模型的重叠，证明了模型的泛化性。
    *   使用了中值绝对百分比误差（MdAPE）和 Kendall $\tau$ 相关系数，既评估了数值准确性，也评估了排序能力（即模型能否正确告诉用户哪款 GPU 更好），评估维度客观全面。

#### 6. 主要结论与发现
*   **高精度预测**：在完全未见的 GPU 上，服务器场景下的功耗预测误差 $\leq 13.5\%$，延迟预测误差 $\leq 8.5\%$。
*   **显著优于基准**：相比物理基准模型（TDP 或 Roofline），WattGPU 的预测误差降低了 2 到 4 倍。

#### 7. 优点
*   **零成本预测**：无需实际购买或访问 GPU 即可获得可靠的性能评估，对中小规模运营商极具价值。
*   **跨架构泛化**：这是首个被证明能在未见过的 GPU 架构上保持高精度的 LLM 推理预测模型。
*   **实用性强**：模型仅依赖公开元数据，易于集成到自动化部署和容量规划工具中。

#### 8. 不足与局限
*   **硬件覆盖有限**：目前仅针对 NVIDIA GPU，未测试 AMD 或 Intel 的加速卡。
*   **模型类型限制**：实验主要针对稠密（Dense）模型，尚未覆盖混合专家模型（MoE）、量化模型或多 GPU 分布式推理场景。
*   **离线延迟预测挑战**：在离线场景下，由于 vLLM 等推理引擎的连续批处理（Continuous Batching）行为复杂，ITL 的预测误差相对较高（约 24.9%）。
