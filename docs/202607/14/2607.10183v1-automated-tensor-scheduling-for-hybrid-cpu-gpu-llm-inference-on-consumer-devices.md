## 摘要
在笔记本电脑和台式机等消费级设备上运行大语言模型（LLM）具有挑战性，因为模型权重通常超过 GPU 显存容量，因此需要通过卸载（offloading）推理来利用 CPU 内存扩展有效模型容量。然而，现有的卸载系统通常依赖于粗粒度的层级或专家级调度，这忽略了同一层内张量之间显著的异构性，且难以适应此类设备上不断变化的硬件负载情况。本文提出了 ATSInfer，这是一种面向消费级设备的 CPU-GPU 混合推理系统，可在张量粒度上执行卸载。ATSInfer 将静态张量放置与负载感知的动态传输相结合，并引入了异步 CPU-GPU 协作，以在异构后端之间高效调度硬件存储、数据移动和计算。我们实现了 ATSInfer，并在代表性的消费级平台上使用稠密模型和混合专家（MoE）模型对其进行了评估。与现有系统相比，ATSInfer 将预填充（prefill）吞吐量提高了多达 1.94 倍，解码（decode）吞吐量提高了多达 3.29 倍，同时还提高了 GPU 利用率并更有效地利用了 PCIe 带宽。这些结果表明，ATSInfer 可以显著提升个人消费级设备上本地 LLM 部署的用户体验。

## Abstract
Running large language models on consumer devices such as laptops and desktops is challenging because model weights often exceed GPU memory capacity, making offloading inference necessary to extend effective model capacity with CPU memory. Existing offloading systems, however, typically rely on coarse layer-level or expert-level scheduling, which overlooks substantial heterogeneity among tensors within the same layer and adapts poorly to changing hardware load conditions on such devices. This paper presents ATSInfer, a hybrid CPU-GPU inference system for consumer devices that performs offloading at tensor granularity. ATSInfer combines static tensor placement with load-aware dynamic transfer, and introduces asynchronous CPU-GPU coordination to efficiently schedule hardware storage, data movement, and computation across heterogeneous backends. We implement ATSInfer and evaluate it on representative consumer platforms using both dense and MoE models. Compared with existing systems, ATSInfer improves prefill throughput by up to 1.94$\times$ and decode throughput by up to 3.29$\times$, while also increasing GPU utilization and making more effective use of PCIe bandwidth. These results show that ATSInfer can substantially improve the user experience of local LLM deployment on personal consumer devices.

### 1. 核心问题与整体含义（研究动机和背景）
在消费级设备（如笔记本、台式机）上运行大语言模型（LLM）面临严重的**显存瓶颈**：模型权重通常远超 GPU 显存容量。现有的“卸载（Offloading）”技术（如 llama.cpp）存在两个主要局限：
*   **调度粒度过粗**：通常以“层”或“专家（MoE）”为单位进行放置，忽略了同一层内不同张量（如 Q/K/V 与 Feed-forward 张量）在计算强度和传输开销上的显著差异。
*   **负载感知缺失**：现有的静态策略无法应对消费级设备动态变化的硬件环境（如后台程序抢占资源、笔记本因发热导致的降频/热节流）。
**ATSInfer** 的核心目标是通过**张量粒度**的自动化调度，充分利用 CPU、GPU 和 PCIe 带宽，提升本地推理的响应速度。

### 2. 方法论：核心思想与关键技术
ATSInfer 提出了一个三位一体的混合推理框架：
*   **异步 CPU-GPU 协作机制**：
    *   **内存布局优化**：将显存划分为“常驻区”（静态放置）和“临时缓冲区”（动态提升）。
    *   **双流流水线**：使用计算流（Compute Stream）处理算子执行，使用传输流（Transfer Stream）通过 Copy Engine 异步移动权重，实现计算与数据传输的深度重叠。
*   **静态张量放置（Static Tensor Placement）**：
    *   引入**经验性能密度（Empirical Performance Density）**指标：衡量单位显存占用能带来的延迟缩减收益。
    *   将放置问题建模为带惩罚项（后端切换开销）的**背包问题**，通过动态规划（DP）在初始化阶段确定哪些高收益张量应常驻 GPU。
*   **负载感知的动态传输（Load-Aware Dynamic Transfer）**：
    *   在运行时实时监测 CPU/GPU 速度和 PCIe 带宽。
    *   利用 DP 算法动态决定是否将原本在 CPU 上的张量临时“提升”到 GPU 执行，前提是传输开销能被之前的计算窗口掩盖。
    *   设置阈值触发重调度，平衡优化收益与图重建开销。
