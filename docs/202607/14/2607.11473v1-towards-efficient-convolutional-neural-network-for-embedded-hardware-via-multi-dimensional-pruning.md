## 摘要
在本文中，我们提出了 TECO，这是一个多维剪枝框架，旨在协同剪枝卷积神经网络（CNN）的三个维度（深度、宽度和分辨率），以提高在嵌入式硬件上的执行效率。在 TECO 中，我们首先引入了一个两阶段重要性评估框架，该框架根据每个维度内部的局部重要性和跨不同维度的全局重要性，高效且全面地评估每个剪枝单元。基于该评估框架，我们提出了一种启发式剪枝算法，逐步剪枝 CNN 的三个维度，以实现在准确率与效率之间的最佳权衡。在多个基准测试上的实验验证了 TECO 相比现有最先进（SOTA）方法的优势。代码和预训练模型可在 https://github.com/ntuliuteam/Teco 获取。

## Abstract
In this paper, we propose TECO, a multi-dimensional pruning framework to collaboratively prune the three dimensions (depth, width, and resolution) of convolutional neural networks (CNNs) for better execution efficiency on embedded hardware. In TECO, we first introduce a two-stage importance evaluation framework, which efficiently and comprehensively evaluates each pruning unit according to both the local importance inside each dimension and the global importance across different dimensions. Based on the evaluation framework, we present a heuristic pruning algorithm to progressively prune the three dimensions of CNNs towards the optimal trade-off between accuracy and efficiency. Experiments on multiple benchmarks validate the advantages of TECO over existing state-of-the-art (SOTA) approaches. The code and pre-trained models are available at https://github.com/ntuliuteam/Teco.

现代 CNN 模型（如 ResNet50）计算量巨大，难以直接部署在资源受限的嵌入式设备（如自动驾驶、智能相机）上。现有的剪枝技术大多局限于单一维度（仅剪枝宽度/通道、深度/层或分辨率），忽略了维度间的冗余相关性，导致在大幅压缩时准确率下降严重。
**核心动机**：提出一种能够同时在深度、宽度和分辨率三个维度上进行协同剪枝的框架，以在计算效率（MACs）、推理延迟和模型准确率之间找到最优平衡。

### 2. 方法论：核心思想与关键技术
TECO 框架的核心是一个**两阶段重要性评估机制**和**启发式剪枝算法**：

*   **维度内评估策略 (INES, Inner-Dimensional Evaluation Strategy)**：
    *   **目的**：快速筛选出每个维度内部最不重要的单元。
    *   **技术细节**：引入“全门控残差块”，在每个通道后添加通道门（CG），在每层后添加层门（LG）。通过一次反向传播计算门控权重的梯度平方，作为局部重要性指标。
    *   **效率**：将每个维度的候选单元从多个减少到一个，极大降低了后续跨维度评估的计算开销。
*   **跨维度评估策略 (ITES, Inter-Dimensional Evaluation Strategy)**：
    *   **目的**：公平比较不同维度单元（层、通道、像素）的重要性。
    *   **全局重要性公式**：$I(u) = \frac{A(u)}{\alpha M(u) + (1-\alpha)T(u)}$。
        *   $A(u)$：剪枝该单元导致的交叉熵损失增加（准确率损失）。
        *   $M(u)$：MACs（计算量）的减少。
        *   $T(u)$：推理延迟的减少（基于预先构建的线性延迟模型估算）。
        *   $\alpha$：平衡计算量和延迟权重的超参数。
*   **启发式架构下降算法 (Heuristic Architecture Descent)**：
    *   采用类似梯度下降的迭代过程：先通过 INES 选出各维度的“最弱”候选者，再通过 ITES 确定全局最冗余的单元进行剪枝，随后进行微调（Fine-tune）恢复精度，循环往复直至达到资源预算。
