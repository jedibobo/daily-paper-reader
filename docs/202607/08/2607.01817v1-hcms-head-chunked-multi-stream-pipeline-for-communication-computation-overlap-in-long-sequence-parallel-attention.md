## 摘要
基于 All-to-all 的序列并行方法在处理中长序列时，通信和计算严格串行执行，导致硬件资源利用不足。本文提出了头分块多流流水线（HCMS），它利用多头注意力的计算独立性，将注意力头划分为多个分块，并通过双 CUDA 流实现细粒度的通信-计算重叠。HCMS 与 FlashAttention 和 SDPA 等现有优化正交兼容，无需修改底层算子，支持非均匀划分，同时保持数值等价性。在四个 GPU 平台、2-8 个 GPU 规模上的实验验证了其有效性：对于 31K-56K token 的典型视频生成序列长度，HCMS 相比 Ulysses 基线实现了 10%-17.5% 的加速，相比 Ring Attention 实现了 5%-14.5% 的加速；在 Wan2.2 模型上实现了 6.8% 的端到端加速。理论分析表明，HCMS 的收益与通信占比 ρ 正相关，建议在 ρ > 20% 时使用。

## Abstract
All-to-all based sequence parallelism methods execute communication and computation strictly in serial when processing medium-long sequences, resulting in hardware resource underutilization. This paper proposes Head-Chunked Multi-Stream Pipeline (HCMS), which exploits the computational independence of multi-head attention by partitioning attention heads into multiple chunks and achieving fine-grained communication-computation overlap through dual CUDA streams. HCMS is orthogonally compatible with existing optimizations such as FlashAttention and SDPA, requires no modification to underlying kernels, supports uneven partitioning while maintaining numerical equivalence. Experiments validate the effectiveness across four GPU platforms at 2-8 GPU scales: for typical video generation sequence lengths of 31K-56K tokens, HCMS achieves 10\%-17.5\% speedup over the Ulysses baseline and 5\%-14.5\% speedup over Ring Attention; end-to-end acceleration of 6.8\% is achieved on the Wan2.2 model. Theoretical analysis shows that HCMS benefits are positively correlated with communication ratio $ρ$, and its use is recommended when $ρ>20\%$.

### 2. 方法论：HCMS 核心思想与技术细节
*   **核心思想**：将原本作为一个整体进行通信和计算的注意力头（Heads）划分为 $C$ 个独立的“头分块”（Chunks）。

    1.  **头分块策略（Head Chunking）**：利用多头注意力的独立性，将 $H$ 个头拆分为 $C$ 组。即使 $H$ 不能被 $C$ 整除，也支持非均匀划分，且实验证明性能差异极小（<1%）。
    2.  **双 CUDA 流流水线**：
        *   **通信流（$S_{comm}$）**：负责执行输入和输出的 All-to-all 通信。
        *   **计算流（$S_{comp}$）**：负责执行注意力计算（如 FlashAttention）。
    3.  **事件同步机制**：通过 CUDA Event 协调两个流。当分块 $c$ 的输入通信完成，立即触发其计算；同时，通信流可以继续搬运分块 $c+1$ 的数据，从而实现重叠。
*   **算法流程**：输入数据先进行 QKV 投影和位置编码，随后进入 HCMS 循环：分块通信 -> 触发计算 -> 计算完成触发输出通信。最后合并所有分块结果进行输出投影。

### 6. 主要结论与发现
*   **加速效果**：在 31K-56K 典型序列下，HCMS 比 Ulysses 提速 **10%-17.5%**，比 Ring Attention 提速 **5%-14.5%**。
*   **端到端收益**：在 Wan2.2 模型中，注意力层提速 18.3%，整机流水线提速 **6.8%**。
*   **适用性规律**：HCMS 的收益与通信占比 $\rho$ 正相关。理论模型推导出最优分块数 $C^* = \sqrt{T_{comm}/\beta}$。建议在 **$\rho > 20\%$** 时使用；若 $\rho < 10\%$（计算密集型），收益则不明显（<1%）。
*   **其他发现**：HCMS 能降低约 **8.7%** 的显存峰值，且完全支持 PyTorch 自动求导。

### 7. 优点与亮点
*   **高兼容性**：它是“正交”的优化，不需要修改底层的 FlashAttention 或 SDPA 内核，直接在 Python/PyTorch 层级即可实现。
*   **细粒度重叠**：相比 Ring Attention 的块级（Block-level）重叠，HCMS 的头级（Head-level）重叠粒度更细，且通信轮数远少于 Ring Attention（2 轮 vs $P-1$ 轮）。
*   **数学严谨**：提供了详细的理论性能模型和最优分块数推导，具有很强的工程指导意义。

### 8. 不足与局限
*   **计算密集型场景受限**：当序列极长（如 >280K）导致计算时间远超通信时间时，加速效果会迅速衰减。
*   **同步开销**：如果分块数 $C$ 设置过大，CUDA Event 的同步开销和内核启动开销会抵消重叠带来的收益。
*   **硬件依赖**：该方法在 PCIe 互连（通信慢）的环境下收益巨大，但在 NVLink 高速互连（通信极快）的集群上，由于 $\rho$ 较低，收益可能会变小。
*   **规模限制**：目前实验最高仅验证到 8 个 GPU，更大规模（如跨节点、百卡并行）的性能表现尚待观察。
