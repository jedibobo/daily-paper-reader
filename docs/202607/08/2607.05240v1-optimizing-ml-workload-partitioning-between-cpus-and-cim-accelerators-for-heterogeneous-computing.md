## 摘要
存内计算 (CIM) 加速器在内存中执行矩阵向量乘法 (MVM)，使其成为机器学习 (ML) 工作负载的一种极具吸引力的解决方案。然而，现有的针对 CIM 加速器的 ML 工作负载划分方法未能充分考虑阻变随机存取存储器 (RRAM) 的约束，如有限的内存容量、高写入延迟和有限的耐受性。它们还忽略了并行性、底层架构效应，以及将中央处理器 (CPU) 作为补充计算资源的作用。为了解决这些局限性，我们提出了一种基于整数线性规划 (ILP) 的异构 CPU-CIM 系统工作负载划分框架。该框架在 RRAM 约束下最小化端到端推理延迟，捕捉并行性，并将经验分析与分析模型相结合。使用我们的框架，异构 CPU-CIM 执行在边缘 CPU 上相比仅 CPU 执行实现了高达 30.9 倍的加速，在高性能 CPU 上实现了 7.3 倍的加速。设计空间探索 (DSE) 为未来的 CIM 加速器提供了进一步的设计见解。

## Abstract
Computing-in-Memory (CIM) accelerators execute Matrix-Vector Multiplications (MVMs) in memory, making them a compelling solution for Machine Learning (ML) workloads. However, existing ML workload partitioning approaches for CIM accelerators do not fully account for Resistive Random Access Memory (RRAM) constraints such as limited memory, high write latency, and limited endurance. They also neglect parallelism, low-level architectural effects, or the Central Processing Unit (CPU) as a complementary compute resource. To address these limitations, we propose an Integer Linear Programming (ILP)-based workload partitioning framework for heterogeneous CPU-CIM systems. It minimizes end-to-end inference latency under RRAM constraints, captures parallelism, and combines empirical profiling with analytical models. Using our framework, heterogeneous CPU-CIM execution achieves speedups of up to 30.9x over CPU-only execution on an edge CPU and 7.3x over a high-performance CPU. A Design Space Exploration (DSE) yields further design insights for future CIM accelerators.

这篇论文针对异构计算环境下，如何在 CPU 和存内计算（CIM）加速器之间优化机器学习（ML）工作负载划分进行了深入研究。以下是对该论文的结构化总结：

### 1. 核心问题与研究动机
*   **核心问题**：如何在考虑 RRAM（阻变随机存取存储器）硬件约束（如有限容量、高写入延迟、有限寿命）的前提下，将 ML 算子最优地分配给 CPU 和 CIM 加速器，以最小化端到端推理延迟。
*   **研究动机**：
    *   **硬件瓶颈**：CIM 虽然能解决冯·诺依曼瓶颈，但 RRAM 的写入开销极大，导致动态权重更换不切实际，必须进行静态分配。
    *   **现有研究不足**：以往的划分方法通常将设备视为“黑盒”，忽略了底层架构特性（如并行性、总线带宽）或假设模型能完全装入 CIM，且往往忽视了 CPU 作为协同计算资源的潜力。

### 2. 方法论
该框架主要由**混合延迟表征**和**基于 ILP 的工作负载划分**两部分组成：
*   **混合延迟表征**：
    *   **CPU 侧（经验分析）**：使用 ONNX Runtime 在目标硬件上进行实际性能分析（Profiling），捕捉复杂的架构优化效应。
    *   **CIM 侧（解析模型）**：基于 PUMA 架构建立解析模型。计算算子所需的 Crossbar 数量，并将延迟建模为加载、计算（MVM）和存储延迟之和。
    *   **通信侧**：建立共享总线模型，计算设备间数据传输的延迟。
*   **ILP（整数线性规划）模型**：
    *   **目标函数**：最小化有向无环图（DAG）中终端节点的完成时间（即总推理延迟）。
    *   **约束条件**：
        *   **算子支持约束**：某些算子只能在 CPU 上运行。
        *   **资源约束**：CIM 侧分配的算子总 Crossbar 需求不能超过硬件预算。
        *   **依赖与并行约束**：考虑算子间的先后顺序；对于无依赖的算子，允许在 CPU 和 CIM 上并行执行，若同时分配给 CPU 则需序列化处理。

### 3. 实验设计
*   **数据集/场景**：选取了四种具有代表性的 CNN 模型：ResNet-18、ResNet-50（计算密集型）、MobileNetV2（深度可分离卷积，对 CIM 不友好）和 YOLOv5n（复杂分支结构）。
*   **Benchmark 与对比**：
    *   **基准（Baseline）**：纯 CPU 执行（分为边缘级 ARM 和高性能 x86 两种环境）。
    *   **对比变量**：不同的 Crossbar 数量（25-200）和不同的 MVM 延迟（50ns-1000ns）。
*   **硬件设置**：
    *   **边缘端**：ARM Cortex-A72 (1.5 GHz)。
    *   **高性能端**：AMD Ryzen 9 3900X (3.8 GHz)。

### 4. 资源与算力
*   **算力资源**：论文使用了 AMD Ryzen 9 3900X 和 ARM Cortex-A72 作为实验主机。
*   **求解器**：使用 **Gurobi** 优化器来求解 ILP 问题。
*   **训练/运行时间**：对于线性结构模型（如 ResNet），ILP 求解在 1 秒内完成；对于复杂分支的 YOLOv5n，求解器在 600 秒超时前可获得 MIP 间隙为 6.97% 的近优解。

### 5. 实验数量与充分性
*   **实验规模**：进行了多维度的设计空间探索（DSE），涵盖了 2 种 CPU 主机、4 种模型、4 种 Crossbar 规模和 4 种 MVM 延迟参数，总计进行了数十组组合实验。
*   **充分性评价**：实验设计较为充分，不仅验证了加速效果，还深入分析了算子分配比例、硬件参数敏感度以及算法的扩展性（MIP Gap 分析），客观地展示了不同模型在异构系统上的表现差异。

### 6. 主要结论与发现
*   **显著加速**：相比纯 CPU，异构划分在 ARM 平台上实现了高达 **30.9 倍**的加速，在 x86 平台上实现了 **7.3 倍**的加速。
*   **模型敏感性**：ResNet 等传统卷积模型受益最大；MobileNetV2 由于深度可分离卷积导致 Crossbar 利用率低，加速效果有限。
*   **资源饱和点**：增加 Crossbar 数量在初期能显著提升性能，但当所有计算密集型算子都已移至 CIM 后，性能提升趋于饱和（通常在 100 个 Crossbar 左右）。
*   **ILP 复杂度**：ILP 的求解难度主要取决于计算图的**分支复杂度**而非算子总数。

### 7. 优点
*   **软硬结合**：结合了经验 Profiling 和解析建模，既保证了 CPU 侧的准确性，又保留了 CIM 侧设计的灵活性。
*   **约束真实**：充分考虑了 RRAM 的静态映射约束和存储容量限制，具有很强的工程参考价值。
*   **并行优化**：ILP 模型显式考虑了 CPU 和 CIM 的并行执行可能性，而不仅仅是简单的任务迁移。

### 8. 不足与局限
*   **CIM 模型简化**：CIM 侧采用解析模型而非实际硬件测量，可能忽略了某些底层电路级的非理想效应或动态功耗波动。
*   **复杂图求解压力**：对于具有极多分支的超大规模网络，ILP 求解可能面临维度灾难，文中 YOLOv5n 的超时现象说明了这一点。
*   **静态局限**：该框架目前仅支持静态划分，无法应对推理过程中可能出现的动态负载变化。
