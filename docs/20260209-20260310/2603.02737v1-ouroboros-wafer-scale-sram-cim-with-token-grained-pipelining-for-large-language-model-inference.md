## 摘要
传统的 LLM 推理架构由于存储层级间频繁的数据移动，面临高能耗和高延迟的问题。我们提出了 Ouroboros，一种基于晶圆级 SRAM 的存内计算（CIM）架构，它在原位执行所有操作，从而消除了片外数据迁移。为了最大化其有限的一级容量，我们引入了三项创新：Token 粒度流水线：取代序列级流水线以缓解长度变化的影响，提高利用率并减少激活存储；分布式动态 KV 缓存管理：将存储与计算解耦，利用碎片化的 SRAM 进行高效的 KV 存储；通信感知映射：优化整个晶圆上的核心分配，以实现局部性和容错性。实验结果表明，Ouroboros 在吞吐量和能效方面分别实现了平均 4.1 倍和 4.2 倍的提升，在 13B 模型上峰值提升分别达到 9.1 倍和 17 倍。（*由于 arXiv 通知“摘要字段不能超过 1,920 个字符”，此处显示的摘要已缩短。完整摘要请下载论文原文。）

## Abstract
Conventional LLM inference architectures suffer from high energy and latency due to frequent data movement across memory hierarchies. We propose Ouroboros, a wafer-scale SRAM-based Computing-in-Memory (CIM) architecture that executes all operations in situ, eliminating off-chip migration. To maximize its limited first-level capacity, we introduce three innovations:   Token-Grained Pipelining: Replaces sequence-level pipelining to mitigate length variations, boosting utilization and reducing activation storage. Distributed Dynamic KV Cache Management: Decouples memory from compute to leverage fragmented SRAM for efficient KV storage. Communication-Aware Mapping: Optimizes core allocation for locality and fault tolerance across the wafer.   Experimental results show Ouroboros achieves average gains of $4.1\times$ in throughput and $4.2\times$ in energy efficiency, peaking at $9.1\times$ and $17\times$ for the 13B model. ### 2. 方法论：核心思想与关键技术
Ouroboros 架构通过软硬件协同设计解决了晶圆级 CIM 的利用率和通信挑战：
*   **Token 粒度流水线（TGP）**：
    *   **核心思想**：传统流水线以“序列”为单位，由于输入长度不一会导致严重的流水线气泡。TGP 将流水线粒度细化到单个 Token。
    *   **技术细节**：利用 Decoder-only 模型的因果掩码（Causal Mask），在 Prefill 阶段实现 Token 级的并行计算，无需等待整个序列生成。这显著提高了核心利用率并减少了中间激活值的存储需求（降低数千倍）。
*   **分布式动态 KV 缓存管理**：
    *   **核心思想**：将计算任务与 KV 存储解耦，利用已分配核心中的碎片化 SRAM 空间。
    *   **技术细节**：设计了三级地址转换机制（页面表、位图、寄存器），支持 KV Cache 的动态增长和负载均衡，避免了大规模分布式系统中的中心化控制延迟。
*   **通信感知与容错映射策略**：
    *   **核心思想**：平衡层间通信（流水线阶段间）与层内通信（归约操作）。
    *   **算法流程**：使用**混合整数二次规划（MIQP）**进行层间核心映射，最小化跨核心/跨芯片的数据传输；使用**动态规划（DP）**优化核心内部的 H-Tree 通信；并设计了基于重映射的容错机制以绕过晶圆制造中的缺陷核心。

*   **关键发现**：Token 粒度流水线是消除 LLM 动态负载不均的关键；全 SRAM 存内计算在处理内存受限的 Decoding 阶段具有压倒性优势。

### 7. 优点（亮点）
*   **架构创新**：首次将晶圆级芯片与存内计算结合处理 LLM，从物理层面解决了数据迁移瓶颈。
*   **细粒度调度**：TGP 策略巧妙利用了 LLM 的因果特性，解决了流水线并行中的负载均衡难题。
*   **全栈优化**：从底层的 CIM 电路实现到顶层的编译器映射算法进行了全栈协同设计。

### 8. 不足与局限
*   **存储密度限制**：尽管使用了整个晶圆，54GB 的 SRAM 仍难以单片容纳超大规模模型（如 GPT-4 级别），必须依赖多晶圆互联。
*   **制造挑战**：晶圆级芯片的良率和散热是巨大的工程挑战，虽然论文提出了容错映射，但实际量产成本极高。
*   **模型结构限制**：对于非 Decoder-only 架构（如 Encoder 模型），由于缺乏因果掩码，TGP 的性能提升会显著下降（实验中 T5 模型的提升较小）。
*   **精度问题**：虽然使用了数字 CIM 以保证精度，但未深入讨论大规模分布式 CIM 系统中的时钟同步和信号完整性问题。
