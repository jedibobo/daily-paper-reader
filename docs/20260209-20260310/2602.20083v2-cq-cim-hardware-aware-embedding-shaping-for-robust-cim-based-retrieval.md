## 摘要
在边缘设备上部署检索增强生成（RAG）具有巨大需求，但受限于传统架构中大规模数据移动和计算带来的延迟。存内计算（CiM）架构通过直接在其交叉杆结构内执行向量搜索来解决这一瓶颈。然而，由于高精度、高维度的嵌入与 CiM 的低精度、低维度阵列约束不兼容，CiM 在 RAG 中的应用受到根本性的“表示鸿沟”限制。这种鸿沟因 CiM 实现方式的多样性（如 SRAM、ReRAM、FeFET）而进一步加剧，每种实现都有独特的设计（如 2-bit 单元、512x512 阵列）。因此，RAG 数据必须经过简单的整形以适配每个目标实现。目前的数据整形方法将维度和精度分开处理，这降低了数据的保真度。这不仅抵消了 CiM 在 RAG 上的优势，还困扰着硬件设计者，使其难以确定故障是由电路设计还是退化的输入数据引起的。因此，CiM 的采用仍然有限。在本文中，我们提出了 CQ-CiM，这是一个统一的、硬件感知的数据整形框架，它通过联合学习压缩和量化，为各种 CiM 设计生成 CiM 兼容的低比特嵌入。据我们所知，这是首个针对 RAG 在全面 CiM 应用中进行数据整形的工作。

## Abstract
Deploying Retrieval-Augmented Generation (RAG) on edge devices is in high demand, but is hindered by the latency of massive data movement and computation on traditional architectures. Compute-in-Memory (CiM) architectures address this bottleneck by performing vector search directly within their crossbar structure. However, CiM's adoption for RAG is limited by a fundamental ``representation gap,'' as high-precision, high-dimension embeddings are incompatible with CiM's low-precision, low-dimension array constraints. This gap is compounded by the diversity of CiM implementations (e.g., SRAM, ReRAM, FeFET), each with unique designs (e.g., 2-bit cells, 512x512 arrays). Consequently, RAG data must be naively reshaped to fit each target implementation. Current data shaping methods handle dimension and precision disjointly, which degrades data fidelity. This not only negates the advantages of CiM for RAG but also confuses hardware designers, making it unclear if a failure is due to the circuit design or the degraded input data. As a result, CiM adoption remains limited. In this paper, we introduce CQ-CiM, a unified, hardware-aware data shaping framework that jointly learns Compression and Quantization to produce CiM-compatible low-bit embeddings for diverse CiM designs. To the best of our knowledge, this is the first work to shape data for comprehensive CiM usage on RAG.

### 1. 核心问题与整体含义（研究动机和背景）
论文聚焦于在边缘设备上部署**检索增强生成（RAG）**的性能瓶颈。虽然**存内计算（CiM）**架构能通过交叉杆（Crossbar）结构并行执行向量搜索（MIPS），极大地降低延迟和功耗，但面临严重的“**表示鸿沟（Representation Gap）**”：
*   **维度与精度不匹配**：主流嵌入模型输出高维度（如 768D/2048D）、高精度（FP32）向量，而 CiM 硬件阵列受限于物理尺寸（如 128x128）和存储单元精度（通常仅 1-2 bit）。
*   **硬件多样性与非理想性**：不同 CiM 实现（SRAM, ReRAM, FeFET）具有不同的噪声特性和精度限制。
*   **现有方法缺陷**：传统方法（如 PCA 或简单量化）将维度压缩与精度量化脱节处理，且忽略了硬件噪声，导致检索精度大幅下降。

### 2. 方法论：核心思想与关键技术
论文提出了 **CQ-CiM**，一个统一的、硬件感知的端到端数据整形框架，其核心思想是**联合学习压缩与量化**：
*   **参数高效适配（LoRA Adapter）**：在冻结的预训练嵌入模型中注入 LoRA 模块，以极小的参数开销使模型能够感知下游的压缩和量化任务。
*   **可学习压缩头（Dense Compression Head）**：使用全连接层将高维向量投影到适配 CiM 阵列的低维空间（如 128D），相比 PCA 更具灵活性。
*   **硬件感知噪声注入（Noise Injection）**：在压缩后、量化前，根据具体硬件（如 FeFET/ReRAM）的实测偏差参数注入高斯噪声，使模型在训练阶段就具备对硬件非理想性的鲁棒性。
*   **非线性量化头（N2UQ）**：采用“非均匀到均匀量化”技术，通过学习最优阈值来处理嵌入向量分布不均的问题，将其映射为低比特（1-bit, 1.58-bit, 2-bit）格式。
*   **联合损失函数**：
    *   **对比损失（Contrastive Loss）**：通过 Dropout 构造正负样本对，保持语义几何结构。
    *   **重构损失（MSE Loss）**：最小化原始 FP32 嵌入与整形后低比特嵌入之间的距离，稳定训练过程。

### 3. 实验设计
*   **数据集**：涵盖 5 个信息检索基准（ARCChallenge, NanoHotpotQA, CQADupStack, FiQA, ArguAna）以及 NanoFever。

    *   **UMAP**：非线性流形降维。
    *   **PCA + Uniform Quantization**：传统的线性降维加均匀量化。
    *   **Vanilla**：原生截断/缩放方法。
    *   **PQ-8bit**：作为高精度向量量化的参考（非 CiM 兼容）。
*   **评估指标**：Recall@5, nDCG@10，以及端到端 RAG 准确率。
*   **硬件模拟**：基于 28nm FeFET 阵列的实测数据，模拟了 5 种不同偏差水平的设备（D-1 到 D-5）。

### 5. 实验数量与充分性
*   **实验规模**：论文进行了多维度测试，包括 5 个检索数据集的横向对比、4 种主流嵌入模型（Qwen3, Granite 等）的泛化性测试、以及针对 LoRA 秩、压缩方式、量化算法的消融实验。
*   **客观性与公平性**：实验考虑了多种比特位（1, 1.58, 2 bit）和多种硬件噪声模型，并结合了下游 LLM（如 Llama-3, Mistral）进行端到端验证。对比实验均在相同维度和精度限制下进行，具有较强的说服力。

### 6. 主要结论与发现
*   **性能领先**：CQ-CiM 在极低比特（1-bit, 2-bit）设置下显著优于 PCA 和 UMAP，尤其在 1-bit 极端情况下，传统方法往往失效，而 CQ-CiM 仍能保持较高的检索精度。
*   **鲁棒性强**：通过噪声注入训练，嵌入向量对 CiM 设备的电导波动（Level-flip noise）表现出极强的抗干扰能力，端到端 RAG 准确率下降轻微。
*   **联合优化的必要性**：消融实验证明，LoRA 适配、Dense 压缩和 N2UQ 量化三者结合的效果远好于单一组件。

### 7. 优点
*   **首创性**：首次提出了针对 CiM 硬件约束的统一 RAG 数据整形框架。
*   **硬件感知**：并非通用的量化算法，而是深度结合了非易失性存储器（NVM）的物理特性（如电导偏差）。
*   **高效性**：训练时间极短，适合快速适配不同的硬件目标。
*   **端到端闭环**：不仅关注检索指标，还验证了对下游大模型生成质量的影响。

### 8. 不足与局限
*   **硬件覆盖**：虽然模拟了多种设备，但主要基于 FeFET 和 ReRAM 的统计模型，缺乏在真实 CiM 芯片上运行的大规模实测数据。
*   **超参数敏感性**：联合损失函数中的权重 $\lambda_{MSE}$ 需要根据不同压缩比进行手动调整（如 5-10 之间），自动化程度有待提高。
*   **应用范围**：目前主要针对 MIPS（最大内积搜索）操作，对于其他复杂的相似度度量（如余弦相似度的硬件实现）讨论较少。
