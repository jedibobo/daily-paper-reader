## 摘要
在这项工作中，我们介绍了 HiAER-Spike，这是一个模块化、可重构的事件驱动类脑计算平台，旨在执行包含多达 1.6 亿个神经元和 400 亿个突触的大型脉冲神经网络——其神经元数量约为小鼠大脑的两倍，且运行速度快于实时。该系统在加州大学圣迭戈分校超级计算机中心组装，包含一个协同设计的软硬件栈，针对运行时的海量并行处理和脉冲的分层地址事件路由（HiAER）进行了优化，同时提升了网络存储和执行的内存效率。该架构能够高效处理稀疏连接和稀疏活动，为边缘计算和云计算提供鲁棒且低延迟的事件驱动推理。HiAER-Spike 提供了一个与硬件层细节无关的 Python 编程接口，使用户无需面对通用脉冲神经网络配置和执行中的复杂性，且对拓扑结构的限制极小。该系统通过 Web 门户向更广泛的社区开放。在下文中，我们概述了软硬件栈，解释了底层设计原则，展示了系统的部分功能，并征求广大类脑计算社区的反馈。示例展示了 HiAER-Spike 在基准测试 CIFAR-10、基于 DVS 事件的手势识别、MNIST 和 Pong 任务中进行事件驱动视觉处理的能力。

## Abstract
In this work, we present HiAER-Spike, a modular, reconfigurable, event-driven neuromorphic computing platform designed to execute large spiking neural networks with up to 160 million neurons and 40 billion synapses - roughly twice the neurons of a mouse brain at faster than real time. This system, assembled at the UC San Diego Supercomputer Center, comprises a co-designed hard- and software stack that is optimized for run-time massively parallel processing and hierarchical address-event routing (HiAER) of spikes while promoting memory-efficient network storage and execution. The architecture efficiently handles both sparse connectivity and sparse activity for robust and low-latency event-driven inference for both edge and cloud computing. A Python programming interface to HiAER-Spike, agnostic to hardware-level detail, shields the user from complexity in the configuration and execution of general spiking neural networks with minimal constraints in topology. The system is made easily available over a web portal for use by the wider community. In the following, we provide an overview of the hard- and software stack, explain the underlying design principles, demonstrate some of the system's capabilities and solicit feedback from the broader neuromorphic community. Examples are shown demonstrating HiAER-Spike's capabilities for event-driven vision on benchmark CIFAR-10, DVS event-based gesture, MNIST, and Pong tasks.

*   **研究背景**：为了填补大规模 SNN 仿真与实际部署之间的空白，研究团队开发了 **HiAER-Spike**。这是一个软硬件协同设计的可重构平台，旨在支持高达 **1.6 亿个神经元**和 **400 亿个突触**（规模约为小鼠大脑的两倍），并提供类似深度学习框架的易用接口。

### 2. 方法论：核心思想与关键技术
*   **分层路由架构 (HiAER)**：采用分层地址事件路由（Hierarchical Address-Event Routing），通过片上网络 (NoC)、板间 FireFly 连接和服务器间以太网实现脉冲的高效分发。
*   **内存优化存储**：
    *   **邻接表 (Adjacency List)**：放弃传统的交叉阵列（Crossbar）结构，改用邻接表存储突触连接，极大提高了稀疏网络的存储效率。
    *   **混合存储策略**：利用 FPGA 上的 **HBM (高带宽内存)** 存储海量的突触权重，利用片上 **BRAM/URAM** 存储高频访问的神经元膜电位状态，平衡了容量与带宽。
*   **软硬件协同设计**：
    *   **硬件层**：基于 Xilinx Virtex UltraScale+ FPGA，每个 FPGA 包含 32 个并行处理核心。
    *   **软件层 (hs_api)**：提供 Python 接口，允许用户使用类 PyTorch 的方式定义网络。内置编译器负责网络分区、资源分配及 HBM 内存映射。
*   **神经元模型**：支持 Leaky Integrate-and-Fire (LIF) 和 ANN（二进制）神经元，支持 16 位整数运算和随机噪声注入。

*   **Benchmark 与对比对象**：

### 5. 实验数量与充分性
*   **实验覆盖面**：涵盖了 MLP、LeNet-5、Spiking CNN 等多种拓扑结构，以及从简单的数字识别到复杂的强化学习任务。
*   **充分性评估**：
    *   **验证性实验**：通过对比 PyTorch 仿真结果与 FPGA 硬件运行结果，验证了转换算法的精确性（多数任务误差极小）。

### 6. 主要结论与发现
*   **高性能与低延迟**：在 MNIST 任务上，HiAER-Spike 的延迟仅为 4.2 $\mu s$，能耗低至 1.1 $\mu J$，显著优于 Loihi 和 SpiNNaker。

### 7. 优点（亮点）
*   **可重构性**：基于 FPGA，相比 ASIC 芯片，能够更快速地迭代神经元模型和学习规则（如 STDP）。
*   **易用性**：通过 NSG 门户和 Python API，降低了非硬件专家使用类脑计算平台的门槛。
*   **内存效率**：邻接表与 HBM 的结合解决了大规模 SNN 存储的瓶颈，支持极高的突触密度。

*   **精度权衡**：为了追求能效和速度，部分实验中的准确率略低于经过深度优化的专用 ASIC 平台（如 TrueNorth）。
*   **全系统实测缺失**：论文主要展示了单核和单卡的基准数据，多服务器集群在大规模并行时的网络拥塞和同步开销仍需进一步实测验证。
