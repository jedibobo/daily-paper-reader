## 摘要
实时 AI 体验需要针对资源受限硬件上的高效部署进行优化的端侧大语言模型 (OD-LLM)。最具实用价值的 OD-LLM 能够产生近乎实时的响应，并表现出广泛的硬件兼容性，从而最大限度地扩大用户覆盖范围。我们提出了一种在移动端延迟约束下，利用硬件在环 (hardware-in-the-loop) 架构搜索来设计此类模型的方法论。该系统适用于工业级规模的部署：它生成的模型无需自定义算子即可部署，并与 Executorch 等标准移动端运行时兼容。我们的方法避免了专门的注意力机制，而是采用注意力跳跃 (attention skipping) 来实现长上下文加速。我们的方法联合优化了模型架构（层数、维度）和注意力模式。为了高效评估候选模型，我们将每个模型视为具有继承权重的预训练骨干网络的剪枝版本，从而以极少的持续预训练实现高精度。我们在分阶段过程中利用了延迟评估的低成本特性：首先学习一个准确的延迟模型，然后在延迟和质量之间搜索帕累托前沿 (Pareto-frontier)。这产生了 MobileLLM-Flash，一系列用于高效端侧使用的基础模型（350M、650M、1.4B），具有强大的能力，支持高达 8k 的上下文长度。MobileLLM-Flash 在移动端 CPU 上的预填充 (prefill) 和解码 (decode) 速度分别提升了高达 1.8 倍和 1.6 倍，且质量相当或更优。我们对帕累托前沿设计选择的分析为 OD-LLM 设计提供了可操作的原则。

## Abstract
Real-time AI experiences call for on-device large language models (OD-LLMs) optimized for efficient deployment on resource-constrained hardware. The most useful OD-LLMs produce near-real-time responses and exhibit broad hardware compatibility, maximizing user reach. We present a methodology for designing such models using hardware-in-the-loop architecture search under mobile latency constraints. This system is amenable to industry-scale deployment: it generates models deployable without custom kernels and compatible with standard mobile runtimes like Executorch. Our methodology avoids specialized attention mechanisms and instead uses attention skipping for long-context acceleration.   Our approach jointly optimizes model architecture (layers, dimensions) and attention pattern. To efficiently evaluate candidates, we treat each as a pruned version of a pretrained backbone with inherited weights, thereby achieving high accuracy with minimal continued pretraining. We leverage the low cost of latency evaluation in a staged process: learning an accurate latency model first, then searching for the Pareto-frontier across latency and quality.   This yields MobileLLM-Flash, a family of foundation models (350M, 650M, 1.4B) for efficient on-device use with strong capabilities, supporting up to 8k context length. MobileLLM-Flash delivers up to 1.8x and 1.6x faster prefill and decode on mobile CPUs with comparable or superior quality. Our analysis of Pareto-frontier design choices offers actionable principles for OD-LLM design.

#### 2. 方法论与核心技术
*   **核心思想**：采用**硬件在环（Hardware-in-the-loop）**的架构搜索，直接以移动端实际测量延迟为约束，联合优化模型深度、维度及注意力模式。
*   **关键技术细节**：
    *   **两阶段贝叶斯优化（BO）**：
        *   **阶段 1**：密集采样架构空间并测量手机端实际延迟，训练一个准确的延迟预测模型（成本极低）。
        *   **阶段 2**：利用延迟模型引导多目标优化，搜索“准确率-延迟”的帕累托前沿（Pareto-frontier），仅对潜力候选者进行昂贵的质量评估。
    *   **基于剪枝的架构搜索**：不从零训练，而是通过对预训练骨干网络进行结构化剪枝（基于激活能量指标）并继承权重。这种方式比从零训练更具数据效率，且能快速达到稳定排名。

#### 6. 主要结论与发现
*   **性能领先**：MobileLLM-Flash 在移动 CPU 上的预填充速度比 LFM2 快 1.8 倍，解码速度快 1.6 倍，且在多数 Benchmark 上准确率占优。
*   **设计原则 1**：在端侧部署中，“浅而宽”的模型比“深而窄”的模型能更好地平衡精度与延迟。
*   **设计原则 2**：注意力跳过（Skip Attention）在标准移动运行时中比滑动窗口（SWA）更高效；最佳模式是交替使用跳过层和全局注意力层，且不建议连续跳过超过 3 层。
*   **词表影响**：使用较大的词表（202k）虽然增加了嵌入层参数，但提高了单 token 的信息密度，从而减少了推理时的 token 总数，降低了整体开销。

#### 7. 优点
*   **极强的实用性**：完全兼容 ExecuTorch，无需任何自定义算核即可在标准移动端实现加速，具有极高的工业部署价值。
*   **硬件在环优化**：直接解决代理指标（FLOPs）误导设计的问题，确保了真实的端侧体验。
*   **高效搜索**：结合剪枝和两阶段 BO，大幅降低了寻找最优架构的算力成本。

#### 8. 不足与局限
*   **超参数耦合**：搜索过程中未同时优化训练超参数（如学习率、优化器设置），可能导致某些架构候选者因默认配置不当而表现欠佳。
*   **算子限制**：为了兼容性，主动放弃了对新兴子二次方注意力机制（如 SSM/线性注意力）的探索，这在未来软件支持成熟后可能成为限制。
*   **硬件单一性**：虽然在旗舰手机上表现优异，但在中低端设备或不同架构（如 iOS 设备的 GPU 加速）上的表现尚未充分展示。
