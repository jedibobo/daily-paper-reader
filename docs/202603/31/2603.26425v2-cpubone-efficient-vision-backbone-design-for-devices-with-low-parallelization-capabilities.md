## 摘要
最近关于视觉骨干网络架构的研究主要集中在优化具有高并行处理能力的硬件平台的效率。这一类别日益包括手机和嵌入式 AI 加速模块等嵌入式系统。相比之下，CPU 无法以同样的方式并行化操作，因此模型受益于一种特定的设计理念，即通过提高每秒乘加运算次数（MACpS）来平衡运算量（MACs）和硬件执行效率。为此，我们研究了针对标准卷积的两种改进，旨在降低计算成本：分组卷积和减小卷积核尺寸。虽然这两种改进都大幅减少了推理所需的总 MACs，但维持低延迟仍需保持硬件效率。我们在多种 CPU 设备上的实验证实，这些改进成功地在 CPU 上保留了较高的硬件效率。基于这些见解，我们推出了 CPUBone，这是一个针对基于 CPU 推理优化的新型视觉骨干模型系列。CPUBone 在广泛的 CPU 设备上实现了最先进的速度-准确度权衡（SATs），并能有效地将其效率迁移到目标检测和语义分割等下游任务中。模型和代码可在 https://github.com/altair199797/CPUBone 获取。

## Abstract
Recent research on vision backbone architectures has predominantly focused on optimizing efficiency for hardware platforms with high parallel processing capabilities. This category increasingly includes embedded systems such as mobile phones and embedded AI accelerator modules. In contrast, CPUs do not have the possibility to parallelize operations in the same manner, wherefore models benefit from a specific design philosophy that balances amount of operations (MACs) and hardware-efficient execution by having high MACs per second (MACpS). In pursuit of this, we investigate two modifications to standard convolutions, aimed at reducing computational cost: grouping convolutions and reducing kernel sizes. While both adaptations substantially decrease the total number of MACs required for inference, sustaining low latency necessitates preserving hardware-efficiency. Our experiments across diverse CPU devices confirm that these adaptations successfully retain high hardware-efficiency on CPUs. Based on these insights, we introduce CPUBone, a new family of vision backbone models optimized for CPU-based inference. CPUBone achieves state-of-the-art Speed-Accuracy Trade-offs (SATs) across a wide range of CPU devices and effectively transfers its efficiency to downstream tasks such as object detection and semantic segmentation. Models and code are available at https://github.com/altair199797/CPUBone.

*   **核心矛盾**：CPU 的并行化能力有限，导致那些在 GPU 上表现优异的模型（通常具有高 MACs 但高并行度）在 CPU 上运行缓慢。
*   **研究目标**：设计一种专门针对 CPU 优化的视觉骨干网络，通过平衡运算量（MACs）和硬件执行效率（MACpS，每秒乘加运算次数）来提升推理速度。

### 2. 方法论
*   **核心思想**：通过改进移动倒置瓶颈块（MBConv），在显著降低 MACs 的同时，确保在 CPU 上的硬件执行效率（MACpS）不下降。
*   **关键技术细节**：
    *   **分组卷积（Grouping）**：在 MBConv 的第一层卷积中设置 `groups=2`。相比于深度卷积（Depthwise Conv，MACpS 极低），`groups=2` 能在减少约 23%-45% MACs 的同时，保持与标准卷积接近的执行效率。
    *   **减小卷积核（Kernel Size）**：将常用的 3x3 卷积核减小为 2x2。这能减少约 56% 的卷积层 MACs。作者发现 2x2 卷积在 CPU 上依然保持高 MACpS，但在 GPU 上效率会大幅下降。
    *   **融合与非融合策略**：实验发现，当通道数 < 256 时，融合 MBConv（FuMBConv）效率更高；当通道数 ≥ 256 时，非融合的标准 MBConv 效率更高。
    *   **架构设计**：基于 LowFormer 宏观架构，在最后两个阶段引入轻量级注意力机制，并使用最近邻上采样替代转置卷积以节省开销。
*   **公式逻辑**：通过 $M \propto \frac{1}{groups}$ 和 $M \propto K_H \times K_W$ 理论计算 MACs 的缩减比例，并结合实验测量的 MACpS 来验证实际延迟。

### 6. 主要结论与发现
*   **CPU 优化悖论**：在 GPU 上能加速的策略（如深度卷积、极小卷积核）在 CPU 上未必高效。
*   **最优分组**：`groups=2` 是 CPU 上平衡 MACs 减少与硬件效率的最佳点。
*   **2x2 卷积的价值**：在 CPU 上，2x2 卷积是极佳的减速手段，但在 GPU 上会导致效率崩溃。

*   **针对性强**：填补了视觉模型在低并行度硬件（CPU）上优化不足的空白。
*   **指标科学**：引入 MACpS 这一指标，比单纯看 MACs 或 FLOPs 更能反映真实硬件性能。
*   **实用性高**：在树莓派、手机 CPU 等资源受限设备上表现极其优异，具有很强的工业落地价值。

### 8. 不足与局限
*   **硬件局限性**：该模型是为 CPU “量身定制”的，在 GPU 上的表现（MACpS）较差，不适合跨平台通用（如在 GPU 服务器上运行）。
