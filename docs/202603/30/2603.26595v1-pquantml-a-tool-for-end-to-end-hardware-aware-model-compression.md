## 摘要
PQuantML 是一个全新的开源、硬件感知神经网络模型压缩库，专为端到端工作流量身定制。出于在具有严格延迟约束的环境中部署高性能模型的需求，PQuantML 通过提供统一的接口来联合或单独应用剪枝和量化，从而简化了压缩模型的训练。该库实现了多种不同粒度的剪枝方法，以及支持高粒度量化（HGQ）的定点量化。我们在代表性任务上评估了 PQuantML，例如喷注子结构分类（即所谓的喷注标记），这是一个与大型强子对撞机（LHC）实时数据处理相关的边缘端问题。通过结合多种剪枝方法与定点量化，PQuantML 在保持准确性的同时实现了显著的参数量和位宽缩减。此外，我们还将所得的压缩结果与 QKeras 和 HGQ 等现有工具进行了对比。

## Abstract
PQuantML is a new open-source, hardware-aware neural network model compression library tailored to end-to-end workflows. Motivated by the need to deploy performant models to environments with strict latency constraints, PQuantML simplifies training of compressed models by providing a unified interface to apply pruning and quantization, either jointly or individually. The library implements multiple pruning methods with different granularities, as well as fixed-point quantization with support for High-Granularity Quantization. We evaluate PQuantML on representative tasks such as the jet substructure classification, so-called jet tagging, an on-edge problem related to real-time LHC data processing. Using various pruning methods with fixed-point quantization, PQuantML achieves substantial parameter and bit-width reductions while maintaining accuracy. The resulting compression is further compared against existing tools, such as QKeras and HGQ.

#### 1. 核心问题与整体含义（研究动机和背景）
在高能物理（HEP）实验（如大型强子对撞机 LHC）中，数据产生速率极高（40 MHz），必须依靠基于 FPGA 的硬件触发系统（L1T）在微秒级延迟内做出决策。为了将复杂的深度学习模型部署到资源受限的 FPGA 上，必须进行**模型压缩**（剪枝与量化）。
目前存在的问题是：现有的工具（如 QKeras 或 HGQ）往往侧重于单一的压缩手段，缺乏一个统一、易用且能与硬件合成工具（如 hls4ml）无缝衔接的端到端框架。PQuantML 的出现旨在填补这一空白，提供一个集成了多种剪枝算法、高粒度量化及自动化超参优化的开源库。

#### 2. 方法论：核心思想与技术细节
PQuantML 的核心思想是**硬件感知训练（Hardware-aware Training）**，将压缩约束直接引入训练循环。
*   **统一接口**：支持 PyTorch 和 Keras/TensorFlow 双后端，通过“层替换（Layer Replacement）”功能，用户可以一键将普通模型转换为支持压缩的模型。
*   **剪枝技术**：
    *   **结构化剪枝**：如激活剪枝（AP）和基于 MDMM（修正微分乘子法）的约束优化剪枝，直接减少通道或滤波器。
    *   **非结构化/半结构化剪枝**：包括 AutoSparse、连续稀疏化（CS）、动态稀疏训练（DST）、参数无关微分剪枝（PDP）及 Wanda 等。支持 N:M 稀疏模式。
*   **量化技术**：
    *   **定点量化**：支持张量级、通道级或权重级的定点表示。
    *   **高粒度量化（HGQ）**：通过梯度下降学习每个权重的最优位宽，并利用 EBOPs（有效位运算）指标平衡精度与硬件成本。
*   **训练流程**：采用三阶段工作流：预训练（Pre-training） -> 压缩训练（Training，学习掩码/位宽） -> 微调（Fine-tuning，固定掩码后恢复精度）。

#### 6. 主要结论与发现

#### 7. 优点
*   **端到端集成**：将剪枝、量化、超参搜索和硬件合成（hls4ml）整合进单一工作流，极大降低了物理学家的使用门槛。
*   **灵活性**：支持双后端（PyTorch/Keras），且允许用户自定义复杂的压缩策略（如先剪枝后量化或联合进行）。
*   **硬件感知度高**：引入了 EBOPs 等指标，使模型优化目标直接指向硬件资源消耗而非仅仅是参数量。

#### 8. 不足与局限
*   **后端功能不平衡**：目前 FITCompress 算法仅支持 PyTorch 后端；TensorFlow 后端的层替换功能尚不支持包含残差连接或注意力机制的复杂非线性架构。
*   **预训练限制**：在 FITCompress 中，过长的预训练可能导致权重范围过大，从而增加量化所需的最小位宽。
*   **编译链支持**：目前主要针对 hls4ml 编译器优化，对其他硬件编译链（如 TVM 或 Vitis AI）的支持尚不明确。
