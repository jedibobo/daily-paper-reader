## 摘要
最近关于视觉骨干网络架构的研究主要集中在优化具有高并行处理能力的硬件平台的效率。这一类别越来越多地包括手机和嵌入式 AI 加速模块等嵌入式系统。相比之下，CPU 无法以同样的方式并行化操作，因此模型受益于一种特定的设计理念，即通过保持较高的每秒乘加操作数（MACpS）来平衡操作量（MACs）和硬件执行效率。为此，我们研究了针对标准卷积的两种改进，旨在降低计算成本：分组卷积和减小卷积核尺寸。虽然这两种改进都大幅减少了推理所需的总 MACs，但维持低延迟仍需保持硬件效率。我们在多种 CPU 设备上的实验证实，这些改进成功地在 CPU 上保留了较高的硬件效率。基于这些见解，我们推出了 CPUBone，这是一个针对基于 CPU 推理进行优化的新型视觉骨干模型系列。CPUBone 在广泛的 CPU 设备上实现了最先进的速度-准确度权衡（SATs），并能有效地将其效率迁移到目标检测和语义分割等下游任务中。模型和代码可在 https://github.com/altair199797/CPUBone 获取。

## Abstract
Recent research on vision backbone architectures has predominantly focused on optimizing efficiency for hardware platforms with high parallel processing capabilities. This category increasingly includes embedded systems such as mobile phones and embedded AI accelerator modules. In contrast, CPUs do not have the possibility to parallelize operations in the same manner, wherefore models benefit from a specific design philosophy that balances amount of operations (MACs) and hardware-efficient execution by having high MACs per second (MACpS). In pursuit of this, we investigate two modifications to standard convolutions, aimed at reducing computational cost: grouping convolutions and reducing kernel sizes. While both adaptations substantially decrease the total number of MACs required for inference, sustaining low latency necessitates preserving hardware-efficiency. Our experiments across diverse CPU devices confirm that these adaptations successfully retain high hardware-efficiency on CPUs. Based on these insights, we introduce CPUBone, a new family of vision backbone models optimized for CPU-based inference. CPUBone achieves state-of-the-art Speed-Accuracy Trade-offs (SATs) across a wide range of CPU devices and effectively transfers its efficiency to downstream tasks such as object detection and semantic segmentation. Models and code are available at https://github.com/altair199797/CPUBone.

#### 1. 核心问题与整体含义（研究动机和背景）
当前视觉骨干网络（Backbone）的设计趋势主要针对具有**高并行处理能力**的硬件（如 GPU、手机 NPU 或 AI 加速模块）。然而，在许多实际应用场景中，**CPU** 仍然是主要的计算单元。
*   **研究动机**：CPU 的并行化能力有限，现有的“高效”模型（如针对 GPU 优化的模型）在 CPU 上往往无法将低计算量（MACs）转化为低延迟。
*   **核心矛盾**：单纯追求减少 MACs 是不够的，必须平衡 **MACs（操作总量）** 与 **MACpS（每秒执行的操作数，即硬件执行效率）**。
*   **研究目标**：探索在 CPU 上既能显著降低计算量，又能保持高硬件执行效率的设计策略，从而构建专为 CPU 推理优化的视觉骨干网络。

#### 2. 方法论：核心思想与关键技术
论文提出了 **CPUBone** 架构，其核心思想是通过改进卷积方式来优化 CPU 上的推理表现：
*   **分组卷积策略（Grouping）**：引入 `groups=2` 的设置。
    *   **GrFuMBConv（分组融合 MBConv）**：在融合倒残差块的第一层使用分组卷积。实验发现，在通道数较少（<256）时，融合版本在 CPU 上具有极高的 MACpS。
    *   **GrMBConv（分组 MBConv）**：在通道数较多（≥256）时使用非融合版本，以保持硬件效率。
*   **卷积核缩减（Kernel Size Reduction）**：将传统的 3×3 卷积核减小为 **2×2**。
    *   **数学原理**：MACs 与卷积核面积成正比（$M \propto K_H \times K_W$），从 3×3 降至 2×2 可理论减少约 56% 的计算量。
    *   **硬件发现**：实验证明，2×2 卷积在 CPU 上能维持甚至提升 MACpS，而在 GPU 上则会导致效率剧降。
*   **宏观架构设计**：
    *   借鉴 LowFormer 的结构，在最后两个阶段集成 **LowFormer Attention**。
    *   **上采样优化**：将 Attention 中的转置卷积替换为**最近邻插值上采样**，以进一步降低 CPU 延迟。
    *   **分阶段优化**：仅在计算量集中的后两个阶段使用 2×2 卷积，前两个阶段保留 3×3 卷积以维持感受野和精度。

#### 6. 主要结论与发现
*   **CPU 特异性**：在 CPU 上，分组卷积（groups=2）和 2×2 卷积核是极佳的权衡点，能大幅降低 MACs 而不损失硬件执行效率。
*   **MACpS 陷阱**：深度可分离卷积（DWConv）虽然 MACs 极低，但在 CPU 和 GPU 上的 MACpS 都很低，并非总是最优选。
*   **性能领先**：CPUBone 在同等精度下，推理速度显著优于现有的轻量化模型（如在 Pi5 上比 EfficientFormerV2 快 50% 以上，且精度更高）。
*   **下游任务增益**：在目标检测和语义分割中，CPUBone 展现了极高的推理效率，部分场景下比同类模型快 3-4 倍。

#### 7. 优点
*   **硬件感知设计**：深入探讨了 CPU 低并行特性的本质，提出了不同于 GPU 优化的设计准则。
*   **实用性强**：针对树莓派、老旧手机等低算力设备提供了切实可行的视觉方案。
*   **分析透彻**：对 MACpS 的实验分析为后续研究提供了宝贵的硬件性能数据参考。

#### 8. 不足与局限
*   **硬件通用性受限**：CPUBone 是为 CPU “量身定制”的。由于 2×2 卷积和特定分组策略在 GPU 上效率极低，该模型不适合在 GPU 或通用异构平台上部署。
*   **感受野风险**：大量使用 2×2 卷积核可能会限制模型的感受野，虽然在 ImageNet 上表现良好，但在处理超大分辨率图像或极小目标时可能存在局限。
*   **Transformer 改进有限**：模型主要改进集中在卷积层（MBConv），对 Attention 机制本身的 CPU 优化探索相对较少。
