## 摘要
大语言模型（LLMs）的微调已成为领域自适应的关键，但其内存密集型的特性超出了大多数 GPU 的能力。为了应对这一挑战并使 LLM 微调普及化，我们提出了 SlideFormer，这是一个专为单 GPU 环境设计的新颖系统。我们的创新点包括：(1) 一个轻量级的异步引擎，将 GPU 视为滑动窗口，并将 GPU 计算与 CPU 更新及多级 I/O 重叠；(2) 一种高效的异构内存管理方案，显著降低了峰值内存占用；(3) 优化了 Triton 内核以解决关键瓶颈，并集成了先进的 I/O。这种协同设计使得在单块 RTX 4090 上微调最新的 123B+ 模型成为可能，支持高达 8 倍的批处理大小和 6 倍的模型规模。在评估中，与基准测试相比，SlideFormer 实现了 1.40 倍至 6.27 倍的吞吐量提升，同时将 CPU/GPU 内存占用减少了约一半，在 NVIDIA 和 AMD GPU 上均能保持超过 95% 的峰值性能。

## Abstract
Fine-tuning Large Language Models (LLMs) has become essential for domain adaptation, but its memory-intensive property exceeds the capabilities of most GPUs. To address this challenge and democratize LLM fine-tuning, we present SlideFormer, a novel system designed for single-GPU environments. Our innovations are: (1) A lightweight asynchronous engine that treats the GPU as a sliding window and overlaps GPU computation with CPU updates and multi-tier I/O. (2) A highly efficient heterogeneous memory management scheme significantly reduces peak memory usage. (3) Optimized Triton kernels to solve key bottlenecks and integrated advanced I/O. This collaborative design enables fine-tuning of the latest 123B+ models on a single RTX 4090, supporting up to 8x larger batch sizes and 6x larger models. In evaluations, SlideFormer achieves 1.40x to 6.27x higher throughput while roughly halving CPU/GPU memory usage compared to baselines, sustaining >95% peak performance on both NVIDIA and AMD GPUs.

    *   **普及化需求**：旨在降低微调门槛，让个人开发者和小型实验室能在单块显卡上微调超大规模模型。

### 2. 方法论：SlideFormer 系统设计
SlideFormer 采用了软硬件协同设计的思路，包含三大核心支柱：
*   **层滑动异步引擎（Layer-Sliding Architecture）**：
    *   **核心思想**：将GPU视为一个“滑动窗口”，仅在显存中保留当前计算所需的层。
    *   **异步流水线**：通过轻量级线程机制（非多进程，避免IPC开销），实现“当前层GPU计算”、“上一层梯度回传（d2h）”、“CPU端参数更新”以及“下一层参数预取（h2d）”的完全并行。
    *   **重叠条件**：定义了隐藏因子 $\eta$，当反向传播时间大于梯度传输与更新时间之和时，可实现零开销卸载。
*   **高效异构内存管理**：
    *   **预分配GPU缓存池**：采用固定大小的层级队列，消除运行时的内存申请与释放开销，防止显存碎片化。
    *   **主机端共享缓冲区**：在CPU内存中为梯度和类型转换设置层间共享的缓冲区，将CPU端的梯度占用从 $2N$ 降低到 $1/n$（$n$为层数），显著降低主机内存峰值。
*   **集成优化内核与高级 I/O**：
    *   **融合算子（Fused Kernels）**：集成 Triton 优化的算子，特别是 **Fused LinearCrossEntropy (LCE)**，解决了输出层 Logits 占用巨大显存的瓶颈（减少>80%输出层显存）。
    *   **GPUDirect Storage (GDS)**：在启用 NVMe 卸载时，绕过 CPU 直接在 NVMe 和 GPU 间传输数据，降低 CPU 负载和 PCIe 总线冲突。

*   **NVMe 性能依赖**：虽然支持 NVMe 卸载，但性能受限于 SSD 的 I/O 带宽。在 SSD 数量较少时，吞吐量会有明显下降（30%-50%）。

*   **应用限制**：该系统专注于全参数微调，对于需要极高性能的预训练场景或多卡并行场景，其层滑动机制可能不如传统的 3D 并行策略高效。
