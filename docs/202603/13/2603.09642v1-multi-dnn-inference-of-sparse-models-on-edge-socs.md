## 摘要
现代边缘应用日益需要多 DNN 推理系统在异构处理器上执行任务，通过并发执行以及将每个模型与最适合的加速器进行匹配来提升性能。然而，现有系统每个任务仅支持单个模型（或少数稀疏变体），这阻碍了匹配效率，并导致较高的服务水平目标（SLO）违规率。我们为多 DNN 推理系统引入了模型拼接（model stitching），通过在无需重新训练的情况下重组稀疏模型的子图来创建模型变体。我们展示了一个演示系统 SparseLoom，证明了模型拼接可以部署到 SoC 上。实验表明，与最先进的多 DNN 推理系统相比，SparseLoom 将 SLO 违规率降低了高达 74%，吞吐量提高了高达 2.31 倍，并将内存开销平均降低了 28%。

## Abstract
Modern edge applications increasingly require multi-DNN inference systems to execute tasks on heterogeneous processors, gaining performance from both concurrent execution and from matching each model to the most suited accelerator. However, existing systems support only a single model (or a few sparse variants) per task, which impedes the efficiency of this matching and results in high Service Level Objective violation rates. We introduce model stitching for multi-DNN inference systems, which creates model variants by recombining subgraphs from sparse models without re-training. We present a demonstrator system, SparseLoom, that shows model stitching can be deployed to SoCs. We show experimentally that SparseLoom reduces SLO violation rates by up to 74%, improves throughput by up to 2.31x, and lowers memory overhead by an average of 28% compared to state-of-the-art multi-DNN inference systems.

*   **核心问题**：
    1.  **变体匮乏**：现有系统通常只为每个任务提供极少数的稀疏模型变体（如仅有原始模型、量化版或剪枝版），导致在面对多变的 SLO（精度与延迟要求）时，难以找到最优的模型与硬件匹配方案。

### 2. 核心方法论：SparseLoom
SparseLoom 的核心思想是**模型缝合（Model Stitching）**，即在无需重新训练的情况下，通过重组来自同一基础模型的不同稀疏变体的子图（层块）来生成海量的新变体。

*   **关键技术模块**：
    1.  **模型缝合（Model Stitching）**：将基础模型及其剪枝、量化变体划分为相同的子图结构，通过排列组合这些子图，在不增加训练成本的前提下，指数级扩展了可选的模型空间。
    2.  **性能分析器（Performance Profiler）**：
        *   **精度估计器**：利用 XGBoost 回归模型，根据原始变体的精度预测缝合后新变体的精度，避免了对成千上万个变体进行逐一测试。
        *   **延迟估计器**：通过累加各子图在特定处理器上的实测延迟来预测端到端延迟。
    3.  **稀疏感知优化器（Sparsity-Aware Optimizer）**：联合优化“处理器放置顺序”和“变体选择”。它会遍历所有可能的处理器执行顺序（如 NPU-GPU-CPU），为每个任务选择能满足 SLO 且延迟最低的缝合变体。
    4.  **热点子图预加载器（Hot-Subgraph Preloader）**：引入“热度（Hotness）”指标（结合子图的使用频率和唯一性），在给定的内存预算内优先预加载最关键的子图，以消除运行时切换延迟。
