## 摘要
视觉-语言-动作（VLA）模型在具身智能领域占据主导地位，但受限于推理开销。虽然模型量化缓解了边缘部署的这些瓶颈，但由于两个关键挑战，静态量化方法对于 VLA 而言仍非最优：(1) 时间动态敏感性，即固定精度因忽略了随阶段变化的误差容忍度而浪费资源；(2) 实时分配，即如何识别实时敏感性以指导位宽分配仍是一个未解决的问题。为了应对这些挑战，我们提出了 DyQ-VLA，一种针对 VLA 的动态量化框架。具体而言，一种敏感性感知切换策略利用实时运动学代理来触发位宽切换，而运动学引导模块则动态分配最优位宽。实验表明，DyQ-VLA 仅需原始内存占用的 30.9%，同时保持了 99.5% 的原始性能，实现了 1.49 倍的仿真加速和高达 1.43 倍的真实世界加速。

## Abstract
Vision-Language-Action (VLA) models are dominant in embodied intelligence but are constrained by inference overheads. While model quantization alleviates these bottlenecks for edge deployment, static quantization approaches remain suboptimal for VLAs due to two critical challenges: (1) Temporal-dynamic sensitivity, where fixed precision wastes resources by ignoring stage-varying error tolerances; and (2) Real-time allocation, where identifying real-time sensitivity to guide bit allocation remains unsolved. To address these challenges, we propose DyQ-VLA, a dynamic quantization framework for VLAs. Specifically, a sensitivity-aware switching strategy leverages real-time kinematic proxies to trigger the bit-width switch, while a kinematic-guided module dynamically allocates the optimal bit-width. Experiments show that DyQ-VLA requires only 30.9% of the original memory footprint while maintaining 99.5% of its original performance, achieving 1.49x simulation and up to 1.43x real-world speedups.

#### 1. 核心问题与整体含义（研究动机和背景）
视觉-语言-动作（VLA）模型虽然在具身智能领域表现卓越，但其巨大的计算和内存开销限制了在资源受限的边缘设备上的实时部署。传统的**静态量化**方法（如固定位宽量化）在处理 VLA 模型时存在两个核心缺陷：
*   **时间动态敏感性忽视**：VLA 模型在不同执行阶段对误差的容忍度不同。例如，粗略移动阶段对误差不敏感，而精细操作阶段（如抓取、插入）则极度敏感。静态量化为了保证成功率必须全程维持高精度，导致资源浪费。
*   **实时分配难题**：缺乏一种轻量级且可靠的实时指标（Proxy）来指导模型在推理过程中动态调整位宽。

#### 2. 方法论
DyQ-VLA 提出了一种动态量化框架，核心思想是**利用机器人的实时运动学状态来推断模型的量化敏感度**。
*   **运动学代理指标**：提取“运动精细度”（Motion Fineness，反映宏观平移趋势）和“角抖动”（Angular Jerk，反映微观旋转突变）作为敏感度指标。
*   **敏感度感知切换策略**：通过双时间窗口（宏观和微观）融合上述指标，并引入**滞后算子（Hysteresis）**。当检测到高敏感度（如即将进行精细操作）时，立即切换回高精度（BF16）；当处于稳定阶段时，则延迟下调位宽以防止震荡。
*   **运动学引导位宽分配**：通过离线校准建立敏感度与硬件原生位宽（2, 4, 8-bit）的映射表。
*   **系统实现**：采用 **W4A-Dynamic** 模式（权重固定为 4-bit 以避免带宽瓶颈，激活值动态量化）。设计了**异步 CPU-GPU 流水线**，在 GPU 处理视觉预填充时，CPU 同步计算运动学指标并下达位宽指令，实现零延迟调度。
