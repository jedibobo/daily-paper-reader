## 摘要
大语言模型（LLMs）正越来越多地被部署为 AI 智能体，它们在简短的“推理-行动”循环中运行，将模型计算与外部调用交织在一起。与传统的聊天应用不同，这些智能体工作负载要求推理服务系统在来自不同 AI 智能体的多个请求并发到达时，平衡低延迟、稳定的 Token 发射率和吞吐量。受隐私、合规性和成本限制的驱动，最近的部署趋势显示出向在消费级 GPU 上本地运行小语言模型（SLMs）的转变。当异构请求在单个 GPU 上重叠时，长预填充（prefills）和短解码（decodes）会争夺资源，从而产生队头阻塞（head-of-line blocking），破坏交互性能的稳定性。通过分析智能体工作负载，我们观察到它们的执行自然地分为：处理长系统提示词的冷预填充（cold prefills）、将工具输出附加到缓存上下文的恢复预填充（resume prefills），以及对延迟敏感的短解码。与传统的聊天机器人服务相比，这种混合负载加剧了资源竞争。我们提出了 AgentServe，这是一个单 GPU 服务系统，通过将预填充与解码隔离、对恢复预填充应用动态预算，并通过预先建立的具有自适应控制的 CUDA Green Context 插槽分配 GPU 资源，确保在此类条件下稳定的多智能体执行。评估结果表明，AgentServe 在保持具有竞争力的吞吐量的同时，显著提高了延迟稳定性，在不同设置下，相比最先进的基准模型，其首字延迟（TTFT）提升高达 2.8 倍，逐字延迟（TPOT）提升高达 2.7 倍。

## Abstract
Large language models (LLMs) are increasingly deployed as AI agents that operate in short reasoning-action loops, interleaving model computation with external calls. Unlike traditional chat applications, these agentic workloads require inference serving systems to balance low latency, stable token emission, and throughput under multiple request arrivals from different AI agents. Recent deployments highlight a shift toward running small language models (SLMs) locally on consumer-grade GPUs, driven by privacy, compliance, and cost constraints. When heterogeneous requests overlap on a single GPU, long prefills and short decodes contend for resources, creating head-of-line blocking that destabilizes interactive performance. By analyzing agent workloads, we observe that their execution naturally separates into cold prefills, which process long system prompts, resume prefills, which append tool outputs to cached contexts, and short decodes, which are latency-critical. This mix intensifies contention compared to conventional chatbot serving. We present AgentServe, a single-GPU serving system that ensures stable multi-agent execution under such conditions by isolating prefills from decodes, applying dynamic budgeting to resume prefills, and allocating GPU resources through pre-established CUDA Green Context slots with adaptive control. Evaluation results show that AgentServe significantly improves latency stability while sustaining competitive throughput, achieving up to 2.8x TTFT improvement and 2.7x TPOT improvement over state-of-the-art baselines across different settings.

## 1. 核心问题与整体含义
*   **研究背景**：随着小语言模型（SLM）性能提升，出于隐私、成本和合规性考虑，在本地消费级 GPU 上部署 AI 智能体（Agent）成为趋势。
*   **核心痛点**：智能体工作负载具有独特的“推理-行动”循环，包含长系统提示词（冷预填充）、频繁的工具输出追加（恢复预填充）以及对延迟极其敏感的短生成（解码）。
*   **问题本质**：在单 GPU 环境下，重型预填充任务会抢占资源，导致解码任务出现“队头阻塞”（HoL Blocking），表现为 Token 输出卡顿（TPOT 尖峰）和首字延迟（TTFT）过高，严重破坏交互体验。

## 2. 方法论
AgentServe 采用了**算法与系统的协同设计**，核心思想是通过物理隔离与动态调整来消除干扰：
*   **三阶段负载分类**：将请求细分为 **Cold Prefill**（处理长系统提示词）、**Resume Prefill**（将工具结果追加至缓存）和 **Short Decode**（结构化输出）。
*   **TPOT 驱动的调度算法**：
    *   实时监测解码阶段的逐字延迟（TPOT）。
    *   根据 TPOT 反馈动态调整“恢复预填充”的 Token 预算。
    *   动态计算并预留解码所需的最小流式多处理器（SM）数量。
*   **CUDA Green Context 资源隔离**：
    *   利用 NVIDIA CUDA 的 Green Context 特性，在硬件层面将 GPU 的 SM 划分为不同插槽。
    *   预先建立不同比例的资源分区（10%~100%），实现预填充与解码的物理空间隔离。
*   **单引擎共享内存**：在同一进程内管理 KV Cache，避免了分布式或多引擎架构中昂贵的跨设备/跨进程数据传输。

## 3. 实验设计
*   **场景与数据集**：使用 **ToolBench** 构建模拟环境，涵盖了 **ReAct**（频繁工具调用）和 **Plan-and-Execute**（复杂规划）两种主流智能体工作范式。
*   **Benchmark（对比方法）**：
    *   **vLLM**：代表采用分块预填充（Chunked Prefill）的吞吐量优化系统。
    *   **SGLang**：代表具有前缀缓存和 PD 分离逻辑的高性能推理引擎。
    *   **llama.cpp**：代表通用的轻量级本地推理框架。
*   **评估指标**：TTFT（首字延迟）、TPOT（逐字延迟）的均值与 P95 尾部延迟、吞吐量（Tokens/s）以及 SLO（服务等级目标）达成率。

*   **GPU 硬件**：
    *   **NVIDIA RTX A5000** (24GB GDDR6, 64 SMs)：代表中端边缘/工作站部署环境。
    *   **NVIDIA RTX 5090** (32GB GDDR7, 128 SMs)：代表下一代高性能消费级硬件。

*   **模型规模**：评估了 Qwen2.5-3B、Qwen2.5-7B 和 Llama-3-8B 三种主流 SLM。
*   **训练时长**：本文专注于推理服务优化，未涉及模型训练过程。

## 5. 实验数量与充分性
*   **实验规模**：涵盖了 3 种模型规模、2 代 GPU 架构、2 种 Agent 范式以及 3 到 6 个并发 Agent 的多种组合。
*   **消融实验**：专门设计了“无调度算法”和“无 Green Context 隔离”的对比实验，验证了软硬件协同设计的必要性。
*   **客观性**：实验通过模拟真实的智能体交互循环（非简单的长文本生成），并引入了严格的 SLO 达成率指标，能够客观反映多智能体并发时的系统稳定性。

## 6. 主要结论与发现
*   **延迟大幅降低**：相比最先进的基准，AgentServe 将 TTFT 提升了最高 **2.8 倍**，将 TPOT 提升了最高 **2.7 倍**。
*   **稳定性增强**：在多智能体并发时，AgentServe 能有效消除由于长预填充导致的 TPOT 尖峰，保证了 Token 发射的平滑性。
*   **吞吐量与延迟兼顾**：在维持低延迟的同时，其整体吞吐量依然优于 SGLang 和 vLLM，证明了细粒度资源隔离比粗粒度的分块或静态分区更高效。
*   **SLO 达成率**：在 RTX 5090 上，AgentServe 几乎能维持 100% 的会话级 SLO 达成率，而基准系统在并发增加时性能迅速退化。

## 7. 优点
*   **细粒度控制**：首次在消费级 GPU 上通过 CUDA Green Context 实现 SM 级别的预填充/解码空间隔离。
*   **阶段感知**：针对 Agent 负载中“恢复预填充”这一被忽视的中间态进行了专门的预算控制。
*   **低开销**：通过预创建 Context 和共享内存，规避了传统 PD 分离方案中的切换和传输开销。

## 8. 不足与局限
*   **硬件限制**：CUDA Green Context 是较新的 NVIDIA 驱动特性，对旧款 GPU 或非 NVIDIA 硬件（如 AMD/Intel）缺乏兼容性。
*   **负载假设**：该设计基于 Agent 解码通常较短的假设，若遇到需要长文本生成的 Agent 任务，SM 分区的收益可能会收窄。
*   **显存压力**：在单张消费级 GPU 上运行多个 Agent 且进行资源分区，对显存管理提出了极高要求，实验中未深入探讨极端显存碎片化下的表现。
