## 摘要
具有思维链（CoT）推理能力的大语言模型（LLM）在复杂问题解决任务中取得了最先进的性能，但其冗长的推理轨迹和庞大的上下文需求使其在边缘端部署变得不切实际。这些挑战包括高昂的 Token 生成成本、巨大的 KV 缓存占用，以及将推理能力蒸馏到适用于移动设备的更小模型时的低效率。现有方法通常依赖于将大模型的推理轨迹蒸馏到小模型中，这些轨迹往往冗长且风格冗余，不适合设备端推理。在这项工作中，我们提出了一种轻量级方法，通过结合 LoRA 适配器与有监督微调（SFT），使小型 LLM 具备推理能力。我们进一步在这些适配器上通过强化学习引入了预算强制（budget forcing），在精度损失极小的情况下显著缩短了响应长度。为了解决受内存限制的解码问题，我们利用了并行测试时缩放（parallel test-time scaling），在延迟略微增加的情况下提高了准确率。最后，我们提出了一种动态适配器切换机制（仅在需要时激活推理）以及 Prompt 编码期间的 KV 缓存共享策略，从而减少了设备端推理的首字响应时间（TTFT）。在 Qwen2.5-7B 上的实验表明，我们的方法在严格的资源限制下实现了高效、准确的推理，使 LLM 推理在移动场景中变得切实可行。展示我们方案在移动设备上运行的视频可在项目页面查看。

## Abstract
Large language models (LLMs) with chain-of-thought reasoning achieve state-of-the-art performance across complex problem-solving tasks, but their verbose reasoning traces and large context requirements make them impractical for edge deployment. These challenges include high token generation costs, large KV-cache footprints, and inefficiencies when distilling reasoning capabilities into smaller models for mobile devices. Existing approaches often rely on distilling reasoning traces from larger models into smaller models, which are verbose and stylistically redundant, undesirable for on-device inference. In this work, we propose a lightweight approach to enable reasoning in small LLMs using LoRA adapters combined with supervised fine-tuning. We further introduce budget forcing via reinforcement learning on these adapters, significantly reducing response length with minimal accuracy loss. To address memory-bound decoding, we exploit parallel test-time scaling, improving accuracy at minor latency increase. Finally, we present a dynamic adapter-switching mechanism that activates reasoning only when needed and a KV-cache sharing strategy during prompt encoding, reducing time-to-first-token for on-device inference. Experiments on Qwen2.5-7B demonstrate that our method achieves efficient, accurate reasoning under strict resource constraints, making LLM reasoning practical for mobile scenarios. Videos demonstrating our solution running on mobile devices are available on our project page.

*   **核心问题**：虽然思维链推理显著提升了 LLM 处理复杂问题的能力，但其生成的推理轨迹极其冗长（Verbose），导致 Token 生成成本高、KV 缓存占用大、延迟高且功耗大。
*   **研究背景**：边缘设备（如手机）受限于 DRAM 容量和带宽。现有的推理模型多通过蒸馏大模型获得，风格冗余且不针对端侧优化。
*   **整体含义**：本文提出了一套端到端的轻量化框架，通过模块化适配器、长度控制和硬件感知优化，证明了在严格资源限制的移动端实现高效、准确推理的可行性。

### 2. 方法论：核心思想与关键技术
*   **模块化推理（LoRA Adapters）**：在冻结的基座模型上使用 LoRA 适配器进行有监督微调（SFT），使模型具备“推理模式”。通过切换适配器，单一基座模型可兼顾普通对话和复杂推理。
*   **预算强制（Budget Forcing via RL）**：
    *   使用 **GRPO（群体相对策略优化）** 算法。
    *   引入**乘法性软屏障奖励函数**：将准确率奖励与长度惩罚相乘，强制模型在不损失精度的前提下缩短推理过程。
*   **动态路由切换器（Switcher）**：一个轻量级的 MLP 分类头，在 Prefill（预填充）阶段判断查询复杂度。若无需推理则绕过 LoRA，节省计算资源。
*   **掩码 LoRA 训练（Masked LoRA）**：在训练推理适配器时，对 Prompt 部分禁用 LoRA，使其与基座模型共享相同的预填充 KV 缓存，从而消除切换模式时的重新编码延迟。
*   **并行测试时缩放（Parallel Test-Time Scaling）**：利用端侧解码时的内存受限特性（算力有余），并行生成多个推理流，并通过一个轻量级的**验证器头（Verifier Head）**进行加权投票，提升准确率。
*   **量化技术（FPTQuant & QAMR）**：采用 **W4A16KV8** 配置（4位权重、16位激活、8位 KV 缓存），并提出“量化感知模块化推理（QAMR）”，在量化后的基座上训练适配器以减少分布偏移。

### 6. 主要结论与发现
*   **推理压缩**：通过预算强制 RL，推理轨迹长度平均缩短了 **2.4 倍**（最高达 8 倍），且准确率几乎无损。
*   **LoRA 效能**：LoRA Rank 128 在 7B 模型上能达到接近全参数微调的效果，但在 3B 模型上差距较大，说明小模型对适配器容量更敏感。
*   **并行增益**：在 7B 模型上，并行生成 8 个流并结合验证器，比贪婪解码准确率提升了约 **10%**。
*   **量化表现**：W4A16 量化后的推理模型性能与全精度版本差距在 **2%** 以内。

### 7. 优点（亮点）
*   **端到端协同**：不仅关注算法，还考虑了 KV 缓存共享、Prefill 分块等底层硬件实现细节。
*   **实用性极强**：提出的 Switcher 和 Masked LoRA 解决了端侧部署中“首字延迟（TTFT）”和“不必要推理”的痛点。
*   **效率平衡**：成功将原本属于云端的复杂推理能力“塞进”了移动端内存和算力预算内。

### 8. 不足与局限
*   **小模型瓶颈**：在 3B 及以下规模的模型上，LoRA 适配器的表现仍显著弱于全参数微调，限制了极低端设备的表现。
*   **任务遗忘**：SFT 推理训练虽然提升了复杂逻辑能力，但在简单代码任务（如 HumanEval）上出现了性能退化（遗忘现象）。
*   **预算先验**：目前的预算强制仍基于 Token 长度，未考虑 Token 的语义密度（即某些关键步骤可能确实需要更多 Token）。
