## 摘要
尽管现有的 CPU 大语言模型（LLM）推理框架已趋于成熟，但它们未能充分挖掘多核 CPU 平台的计算潜力。多核 CPU 广泛部署于 Web 服务器和高端网络设备中，通常被组织成多个将核心与内存分组的 NUMA 节点。当前框架在很大程度上忽视了跨 NUMA 内存访问带来的巨大开销，限制了此类平台上的推理可扩展性和智能化应用。为了解决这一局限性，我们构建了 ArcLight，这是一种从底层开始为多核 CPU 设计的轻量级 LLM 推理架构。ArcLight 集成了高效的内存管理和线程调度，并引入了精细控制的张量并行，以缓解跨节点内存访问墙问题。实验结果表明，ArcLight 显著超越了主流框架的性能上限，推理吞吐量提升高达 46%。此外，ArcLight 保持了对任意 CPU 设备的兼容性。ArcLight 已在 https://github.com/OpenBMB/ArcLight 公开发布。

## Abstract
Although existing frameworks for large language model (LLM) inference on CPUs are mature, they fail to fully exploit the computation potential of many-core CPU platforms. Many-core CPUs are widely deployed in web servers and high-end networking devices, and are typically organized into multiple NUMA nodes that group cores and memory. Current frameworks largely overlook the substantial overhead of cross-NUMA memory access, limiting inference scalability and intelligence enabling on such platforms. To address this limitation, we build ArcLight, a lightweight LLM inference architecture designed from the ground up for many-core CPUs. ArcLight integrates efficient memory management and thread scheduling, and introduces finely controlled tensor parallelism to mitigate the cross-node memory access wall. Experimental results show that ArcLight significantly surpasses the performance ceiling of mainstream frameworks, achieving up to 46% higher inference throughput. Moreover, ArcLight maintains compatibility with arbitrary CPU devices. ArcLight is publicly available at https://github.com/OpenBMB/ArcLight.

*   **研究动机**：虽然 GPU 是 LLM 推理的主流，但多核 CPU 在 Web 服务器和网络设备中广泛存在。现有的 CPU 推理框架（如 `llama.cpp`）在扩展到数十个核心时，未能有效处理 **NUMA（非统一内存访问）** 架构带来的挑战。
*   **核心问题**：在多核 CPU 中，跨 NUMA 节点访问远程内存的速度远慢于本地访问（实验显示差距达 4 倍）。现有框架通常将内存视为统一池（UMA），导致严重的“跨节点内存访问墙”瓶颈，限制了推理性能的进一步提升。

### 2. 方法论
ArcLight 从底层开始构建，采用了极简主义和模块化的设计：
*   **解耦架构**：分为高层解码前端和底层推理后端。后端仅包含内存管理、线程管理、张量库、计算图构建器和调度器五个核心模块。
*   **NUMA 感知内存管理**：不再使用单一内存池，而是在每个 NUMA 节点本地分配独立的缓冲区（权重、激活值、KV Cache），实现显式的张量-节点绑定。
*   **多视图线程组织**：支持将线程池动态划分为多个逻辑线程组，以便并行执行多个独立的子计算图。
*   **跨 NUMA 张量并行（TP）**：
    *   借鉴 GPU 的张量并行思想，将模型权重进行行/列切分并放置在不同 NUMA 节点。
    *   引入 `Scatter`（分发）和 `Gather`（聚合）算子来管理并行子图。
    *   **异步子图执行**：允许不同线程组在没有数据依赖时异步运行，减少线程空闲等待时间。
*   **双缓冲机制**：针对激活缓冲区采用双缓冲交替策略，显著降低运行时的内存占用。

### 6. 主要结论与发现
*   **性能提升**：在多核环境下，ArcLight 的推理吞吐量比 `llama.cpp` 高出多达 **46%**。
*   **内存瓶颈验证**：实验证实本地内存访问速度（约 100 GB/s）是跨节点访问（约 25 GB/s）的 4 倍，证明了 NUMA 优化的必要性。

*   **开源贡献**：提供了简洁的模块化框架，可作为教育和工业部署的良好起点。
