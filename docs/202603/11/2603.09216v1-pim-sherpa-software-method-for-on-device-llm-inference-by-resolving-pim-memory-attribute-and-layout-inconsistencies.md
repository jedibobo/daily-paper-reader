## 摘要
大语言模型（LLM）的端侧部署正在移动和边缘平台迅速普及。LLM 推理包含计算密集型的预填充（prefill）阶段和内存带宽密集型的解码（decode）阶段，学术界和工业界普遍认为解码阶段非常适合存内计算（PIM）。然而，实际的 PIM 系统在这两个阶段之间面临两个障碍：一是内存属性不一致，即预填充阶段倾向于将权重放置在可缓存区域以供重用，而解码阶段则需要将权重放置在不可缓存区域以可靠地触发 PIM；二是主机友好型布局与 PIM 感知型布局之间的权重布局不一致。为了解决这些问题，我们提出了 PIM-SHERPA，这是一种通过解决 PIM 内存属性和布局不一致性来实现高效端侧 LLM 推理的纯软件方法。PIM-SHERPA 提供了两种方法：DRAM 双缓冲（DDB），它将单一的 PIM 感知权重保留在不可缓存区域，同时将下一层的交织（swizzled）权重预取到小型可缓存缓冲区中；以及带有交织内存复制的在线权重重排（OWR），它在执行 GEMM 之前立即进行按需交织内存复制。与基准 PIM 仿真系统相比，PIM-SHERPA 在 Llama 3.2 模型上实现了约 47.8% - 49.7% 的内存容量节省，同时保持了与理论最大值相当的性能。据我们所知，这是首个在产品级 PIM 系统上识别内存属性不一致性并提出有效解决方案的工作。

## Abstract
On-device deployments of large language models (LLMs) are rapidly proliferating across mobile and edge platforms. LLM inference comprises a compute-intensive prefill phase and a memory bandwidth-intensive decode phase, and the decode phase has been widely recognized as well-suited to processing-in-memory (PIM) in both academia and industry. However, practical PIM-enabled systems face two obstacles between these phases, a memory attribute inconsistency in which prefill favors placing weights in a cacheable region for reuse whereas decode requires weights in a non-cacheable region to reliably trigger PIM, and a weight layout inconsistency between host-friendly and PIM-aware layouts. To address these problems, we introduce \textit{PIM-SHERPA}, a software-only method for efficient on-device LLM inference by resolving PIM memory attribute and layout inconsistencies. PIM-SHERPA provides two approaches, DRAM double buffering (DDB), which keeps a single PIM-aware weights in the non-cacheable region while prefetching the swizzled weights of the next layer into small cacheable buffers, and online weight rearrangement with swizzled memory copy (OWR), which performs the on-demand swizzled memory copy immediately before GEMM. Compared to a baseline PIM emulation system, PIM-SHERPA achieves approximately 47.8 - 49.7\% memory capacity savings while maintaining comparable performance to the theoretical maximum on the Llama 3.2 model. To the best of our knowledge, this is the first work to identify the memory attribute inconsistency and propose effective solutions on product-level PIM-enabled systems.

### 1. 核心问题与研究动机
*   **背景**：大语言模型（LLM）推理分为计算密集型的**预填充（Prefill）**阶段和内存带宽密集型的**解码（Decode）**阶段。存内计算（PIM）技术被认为是加速解码阶段（GEMV算子）的理想方案。
*   **核心冲突**：
    1.  **内存属性不一致**：预填充阶段需要权重位于**可缓存（Cacheable）**区域以实现数据复用；而解码阶段为了触发PIM逻辑，必须要求权重位于**不可缓存（Non-cacheable）**区域，否则缓存命中会阻止请求到达内存控制器。
    2.  **权重布局不一致**：主机CPU偏好连续交织的**主机友好型布局**；而PIM为了最大化SIMD效率，要求数据按**PIM感知型布局**（如特定Bank内的列优先存储）排列。
*   **研究动机**：现有的方案要么通过“权重复制”（双倍内存占用）解决，要么需要修改硬件内存控制器。本文旨在寻求一种**纯软件方法**，在不增加硬件成本且节省内存的前提下解决上述不一致性。

### 2. 方法论
PIM-SHERPA 提出了两种纯软件技术，核心在于将权重统一以 PIM 布局存储在不可缓存区，并在运行时动态处理：
*   **交织内存复制 (Swizzled Memory Copy, SMC)**：
    *   这是核心转换机制。在运行时，通过软件逻辑将 PIM 感知布局的权重实时重排（Swizzle）为 CPU 友好的布局，并搬运至可缓存的临时缓冲区。
*   **DRAM 双缓冲 (DRAM Double Buffering, DDB)**：
    *   **核心思想**：利用两个小型可缓存 Buffer。当 CPU 在 Buffer 0 上执行当前层的 GEMM 计算时，后台线程同步将下一层的权重从不可缓存区通过 SMC 转换并预取到 Buffer 1。
    *   **延迟隐藏**：通过计算与数据搬运的重叠，隐藏权重重排的开销。
*   **在线权重重排 (Online Weight Rearrangement, OWR)**：
    *   **适用场景**：针对超长输入序列。由于 GEMM 计算时间随序列长度线性增长，而 SMC 转换时间固定，因此在计算前直接进行同步转换，其开销在长序列中可忽略不计。

*   **内存效率高**：解决了端侧设备（手机）内存容量受限的痛点，使得在有限内存下运行更大模型成为可能。
*   **软件解耦**：基于 ExecuTorch 框架实现，不影响上层模型逻辑，具有良好的移植性。

### 8. 不足与局限
*   **短序列性能损失**：在输入序列较短（如 < 32）时，计算无法覆盖重排延迟，会导致首字延迟（TTFT）明显增加。

*   **应用限制**：主要针对 Transformer 架构的 LLM，对于其他非重复性结构的神经网络，双缓冲的收益可能受限。
