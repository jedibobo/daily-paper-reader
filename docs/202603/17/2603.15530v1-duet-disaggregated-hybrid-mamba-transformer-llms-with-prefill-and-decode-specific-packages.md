## 摘要
大语言模型在运行过程中分为明显的受计算限制的预填充阶段和受内存带宽限制的解码阶段。混合 Mamba-Transformer 模型继承了这种不对称性，同时增加了状态空间模型（SSM）递归和逐元素操作，这些操作在以矩阵乘法为核心的加速器上映射效果较差。这种不匹配导致了性能瓶颈，表明同构架构无法满足所有需求。我们推出了 DUET，这是一种解耦加速器，将预填充和解码阶段分配给专门的封装。预填充封装利用带有封装外内存的脉动阵列小芯片，以实现高效的大矩阵乘法和长序列 SSM。解码封装利用带有高带宽封装内内存的向量单元阵列，以加速逐 token 的 SSM 和向量-矩阵乘法。两种架构均可在运行时配置，以支持具有混合 Mamba 和注意力层的混合模型。在 Nemotron-H-56B、Zamba2-7B 和 Llama3-8B 上针对四种工作负载的评估表明，与 B200 GPU 相比，DUET 的首字响应时间快 4 倍，吞吐量高 1.4 倍，token 间延迟低 1.5 倍。

## Abstract
Large language models operate in distinct compute-bound prefill followed by memory bandwidth-bound decode phases. Hybrid Mamba-Transformer models inherit this asymmetry while adding state space model (SSM) recurrences and element-wise operations that map poorly to matmul-centric accelerators. This mismatch causes performance bottlenecks, showing that a homogeneous architecture cannot satisfy all requirements. We introduce DUET, a disaggregated accelerator that assigns prefill and decode phases to specialized packages. The Prefill package utilizes systolic array chiplets with off-package memory for efficient large matrix multiplications and long-sequence SSMs. The Decode package utilizes vector-unit arrays with high-bandwidth in-package memory to accelerate token-by-token SSM and vector-matrix multiplications. Both architectures are runtime-configurable to support hybrid models with mixed Mamba and attention layers. Evaluations on Nemotron-H-56B, Zamba2-7B, and Llama3-8B across four workloads show that DUET achieves 4x faster time to first token, 1.4x higher throughput, and 1.5x lower time between tokens over the B200 GPU.

### 1. 核心问题与研究动机
*   **推理阶段的不对称性**：LLM 推理分为“预填充”（Prefill，计算密集型）和“解码”（Decode，内存带宽密集型）两个阶段。
*   **混合架构的挑战**：新兴的混合模型（如 Nemotron-H, Zamba）结合了 SSM 和注意力机制。SSM 引入了递归依赖和大量的逐元素操作，这些操作在以矩阵乘法（Matmul）为核心的传统 GPU（如 NVIDIA B200）上映射效率极低。
*   **同构硬件的局限**：现有的 LLM 服务框架虽然在软件层面将 Prefill 和 Decode 分配给不同的 GPU，但硬件本身是同构的，导致 Prefill 阶段带宽利用不足，而 Decode 阶段计算单元大量闲置。

### 2. 方法论：DUET 架构设计
DUET 的核心思想是**硬件级别的解耦**，为两个阶段提供专门定制的封装（Package）：

*   **Prefill 封装（计算导向）**：
    *   **核心硬件**：采用基于脉动阵列（Systolic Array）的小芯片（Chiplet），配合封装外 GDDR 内存。
    *   **关键技术**：引入**状态驻留（State-stationary）数据流**。在处理长序列 SSM 时，将递归状态保留在处理元件（PE）内部，通过三级微流水线融合逐元素操作，避免了中间结果频繁读写 SRAM 的瓶颈。
*   **Decode 封装（带宽导向）**：
    *   **核心硬件**：采用向量单元阵列（Vector-unit Array），紧密集成封装内高带宽内存（HBM）。
    *   **关键技术**：优化了点积还原路径（Dot-product reduction），支持 O(log N) 延迟的向量操作，能够快速处理逐 Token 的 SSM 状态更新和向量-矩阵乘法（GEMV）。
*   **运行时可配置性**：
    *   两种封装均支持运行时配置，通过模式位（Mode bits）在 SSM 模式和注意力（GEMM/GEMV）模式之间无缝切换，以适配混合层架构。

### 8. 不足与局限
*   **评估局限**：所有结果均基于周期精确模拟，而非真实硅片测试，实际流片后的时钟频率和热管理可能存在偏差。
