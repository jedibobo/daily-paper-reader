## 摘要
事件相机产生异步、高动态范围的数据流，非常适合检测小型、快速移动的无人机，然而大多数基于事件的检测器将稀疏的事件流转换为密集张量，从而丢弃了神经形态感知的表示效率。我们提出了 SparseVoxelDet，据我们所知，这是首个用于事件相机的全稀疏目标检测器，其中骨干特征提取、特征金字塔融合和检测头均通过 3D 稀疏卷积仅在被占用的体素位置上运行；在流水线的任何阶段都不会实例化密集特征张量。在 FRED 基准测试（629,832 个标注帧）上，SparseVoxelDet 在每帧仅处理 14,900 个活跃体素（占 T.H.W 网格的 0.23%）的情况下，实现了 83.38% 的 mAP@50，而密集 YOLOv11 基准模型处理 409,600 个像素（mAP@50 为 87.68%）。将 IoU 阈值从 0.50 放宽到 0.40 后，mAP 恢复到 89.26%，这表明剩余的精度差距主要由边界框回归精度而非检测能力决定。相对于等效的密集 3D 体素张量，稀疏表示实现了 858 倍的 GPU 显存压缩和 3,670 倍的存储减少，其数据结构大小随场景动态而非传感器分辨率缩放。对 119,459 个测试帧的错误取证确认，71% 的失败是定位偏差而非漏检。这些结果表明，原生稀疏处理是事件相机目标检测的一种可行范式，它在不需要神经形态计算硬件的情况下利用了神经形态传感器数据的结构稀疏性，并提供了一个表示成本受场景活动而非像素数量支配的框架，随着事件相机向更高分辨率扩展，这一特性变得越来越有价值。

## Abstract
Event cameras produce asynchronous, high-dynamic-range streams well suited for detecting small, fast-moving drones, yet most event-based detectors convert the sparse event stream into dense tensors, discarding the representational efficiency of neuromorphic sensing. We propose SparseVoxelDet, to our knowledge the first fully sparse object detector for event cameras, in which backbone feature extraction, feature pyramid fusion, and the detection head all operate exclusively on occupied voxel positions through 3D sparse convolutions; no dense feature tensor is instantiated at any stage of the pipeline. On the FRED benchmark (629,832 annotated frames), SparseVoxelDet achieves 83.38% mAP at 50 while processing only 14,900 active voxels per frame (0.23% of the T.H.W grid), compared to 409,600 pixels for the dense YOLOv11 baseline (87.68% mAP at 50). Relaxing the IoU threshold from 0.50 to 0.40 recovers mAP to 89.26%, indicating that the remaining accuracy gap is dominated by box regression precision rather than detection capability. The sparse representation yields 858 times GPU memory compression and 3,670 times storage reduction relative to the equivalent dense 3D voxel tensor, with data-structure size that scales with scene dynamics rather than sensor resolution. Error forensics across 119,459 test frames confirms that 71 percent of failures are localization near-misses rather than missed targets. These results demonstrate that native sparse processing is a viable paradigm for event-camera object detection, exploiting the structural sparsity of neuromorphic sensor data without requiring neuromorphic computing hardware, and providing a framework whose representation cost is governed by scene activity rather than pixel count, a property that becomes increasingly valuable as event cameras scale to higher resolutions.

#### 1. 论文的核心问题与整体含义（研究动机和背景）
事件相机（Event Cameras）具有高动态范围和低延迟的特性，非常适合检测高速移动的微小目标（如无人机）。然而，现有的事件检测算法大多将异步、稀疏的事件流转换为密集的帧或张量（如 H×W 的图像），这不仅破坏了神经形态数据的原生稀疏性，还导致了巨大的计算浪费。
**研究动机：** 论文观察到事件数据在 3D 体素网格中的占用率通常低于 1%，因此提出了一种名为 **SparseVoxelDet** 的全稀疏检测架构。其核心目标是让计算成本随“场景活动（事件数量）”而非“传感器分辨率”缩放，从而实现极高的计算和存储效率。

#### 2. 论文提出的方法论：核心思想与关键技术
SparseVoxelDet 是首个在骨干网络、特征融合和检测头全流程中均不实例化密集张量的检测器。
*   **事件体素化（Sparse Voxelization）：** 将事件流划分为 $T$ 个时间轴分量，提取 6 通道特征（正负极性的计数、新鲜度/Recency、时间标准差），仅存储活跃体素的坐标和特征。
*   **SparseSEResNet 骨干网络：** 采用 3D 稀疏卷积（Sparse 3D Convolutions）处理体素网格。借鉴了 SEW-ResNet 的残差连接模式，并加入挤压激励（SE）注意力机制，提取多尺度稀疏特征。
*   **SparseFPN（稀疏特征金字塔）：** 通过稀疏转置卷积进行上采样，并利用稀疏加法融合不同尺度的特征，最终输出 stride-4 的精细特征图。
*   **时间压缩与检测头：** 通过时间轴上的最大池化（Temporal Max-Pooling）将 3D 特征压缩为 2D 稀疏特征，随后使用共享的 MLP（等效于点对点卷积）预测分类、边界框（LTRB 格式）和中心度（Centerness）。
*   **全流程稀疏：** 整个流水线基于哈希表存储活跃位置，仅在有事件的坐标上执行乘加运算。

#### 3. 实验设计
*   **数据集：** 使用 **FRED (Florence RGB-Event Drone)** 基准测试集，包含 231 个序列、629,832 个标注帧，涵盖白天、黄昏、黑夜及多种天气。
*   **对比方法：** 与密集检测器 **YOLOv11** (Event/RGB)、**RT-DETR**、**Faster R-CNN** 以及多模态融合模型 **ER-DETR** 进行对比。
*   **评估指标：** 标准 COCO 指标（mAP@50, mAP@50:95）、活跃体素计数、显存压缩比等。

#### 4. 资源与算力
*   **硬件：** 使用单块 **NVIDIA RTX 3090 (24 GB)** GPU 进行训练。
*   **训练细节：** 采用 AdamW 优化器，混合精度（FP16）训练。虽然设置了 50 个 Epoch，但由于数据集规模巨大且收敛快，模型在第 4 个 Epoch 左右即通过早停机制（Early Stopping）达到最优性能。
*   **效率提升：** 相比密集 3D 张量，该方法实现了 **858 倍的 GPU 显存压缩**和 **3,670 倍的磁盘存储减少**。

#### 5. 实验数量与充分性
*   **实验规模：** 论文在 119,459 帧测试集上进行了详尽的推理，并开展了针对分辨率（640x640 vs 1280x720 原生分辨率）的消融实验。
*   **错误取证（Error Forensics）：** 这是一个亮点，作者对近 12 万帧测试结果进行了系统性分解，分析了定位误差、置信度分离度、目标大小及事件密度对性能的影响。
*   **充分性评价：** 实验设计非常客观且充分。通过多随机种子（Seeds）验证了稳定性，并深入探讨了为何稀疏模型在 IoU 阈值较高时表现较弱的底层原因。

#### 6. 论文的主要结论与发现
*   **性能表现：** SparseVoxelDet 在 640x640 分辨率下达到 **83.38% mAP@50**，虽然比最强的密集模型 YOLOv11 (87.68%) 低 4.3 个百分点，但处理的像素位置减少了约 28 倍。
*   **瓶颈定位：** 错误分析显示，71% 的失败属于“定位偏差”（IoU 在 0 到 0.5 之间），而非“漏检”。当 IoU 阈值放宽至 0.40 时，mAP 提升至 89.26%，说明模型具备极强的检测能力，但边界框回归不够精准。
*   **扩展性：** 当传感器分辨率从 640 增加到 1280（像素增加 2.25 倍）时，活跃体素仅增加 9%，证明了稀疏处理在应对高分辨率传感器时的巨大优势。

#### 7. 优点
*   **极致高效：** 彻底摆脱了密集张量的束缚，将计算资源集中在有意义的信号上。
*   **内存友好：** 极高的压缩比使其非常适合部署在显存受限的边缘计算设备（如无人机机载平台）。
*   **分析深入：** 论文不仅给出了分数，还通过“错误取证”解释了稀疏卷积在处理物体边缘（而非填充内部）时面临的回归挑战。

#### 8. 不足与局限
*   **回归精度：** 由于稀疏卷积只在有事件的边缘处采样，缺乏物体内部的密集上下文，导致在高 IoU 要求下（如 mAP@50:95）表现不如密集模型。
