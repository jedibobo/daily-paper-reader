## 摘要
嵌入式视觉中的一项关键任务是视觉里程计（VO），它通过视觉传感器估计相机运动，是许多嵌入式功耗受限系统（从自主机器人到增强现实和虚拟现实可穿戴设备）的核心组件。最新一类 VO 系统将深度学习模型与受生物启发的事件相机相结合，这类相机对运动模糊和光照条件具有鲁棒性。然而，最先进的（SoA）基于事件的 VO 算法需要大量的内存和计算资源。例如，领先的方法 DEVO 每帧需要 733 MB 内存和 1550 亿次乘加（MAC）操作。我们提出了 TinyDEVO，这是一种专为资源受限的微控制器单元（MCU）设计的基于事件的 VO 深度学习模型。我们将 TinyDEVO 部署在超低功耗（ULP）9 核 RISC-V 微控制器上，实现了约 1.2 帧/秒的吞吐量，平均功耗仅为 86 mW。得益于我们的神经网络架构优化和超参数调优，与 DEVO 相比，TinyDEVO 在三个最先进的数据集上将内存占用减少了 11.5 倍（至 63.8 MB），每帧操作数减少了 29.7 倍（至每帧 52 亿次 MAC），同时保持了 27 cm 的平均轨迹误差，即仅比 DEVO 高出 19 cm。我们的工作首次证明了在超低功耗设备上运行基于事件的 VO 流水线的可行性。

## Abstract
A key task in embedded vision is visual odometry (VO), which estimates camera motion from visual sensors, and it is a core component in many embedded power-constrained systems, from autonomous robots to augmented and virtual reality wearable devices. The newest class of VO systems combines deep learning models with bio-inspired event-based cameras, which are robust to motion blur and lighting conditions. However, state-of-the-art (SoA) event-based VO algorithms require significant memory and computation. For example, the leading approach DEVO requires 733 MB of memory and 155 billion multiply-accumulate (MAC) operations per frame. We present TinyDEVO, an event-based VO deep learning model designed for resource-constrained microcontroller units (MCUs). We deploy TinyDEVO on an ultra-low-power (ULP) 9-core RISC-V-based MCU, achieving a throughput of approximately 1.2 frames per second with an average power consumption of only 86 mW. Thanks to our neural network architectural optimizations and hyperparameter tuning, TinyDEVO reduces the memory footprint by 11.5x (to 63.8 MB) and the number of operations per frame by 29.7x (to 5.2 billion MACs per frame) compared to DEVO, while maintaining an average trajectory error of 27 cm, i.e., only 19 cm higher than DEVO, on three state-of-the-art datasets. Our work demonstrates, for the first time, the feasibility of an event-based VO pipeline on ultra-low-power devices.

### 2. 核心方法论
TinyDEVO 通过对原始 DEVO 架构进行系统性的轻量化改造实现：
*   **架构优化（Model Reduction）**：

    *   **算子替换**：针对 MCU 难以处理的 Softmax 操作进行了优化或有选择地保留，以平衡精度与延迟。
*   **超参数调优（Patch Graph Optimization）**：
    *   通过调整补丁数量（$N_{patches}$）、移除窗口（$R_w$）和补丁寿命（$PLT$），显著减少了补丁图中的边数（$N_{edges}$），因为更新块的计算量与边数成正比。
*   **混合精度部署**：
    *   在 GAP9 处理器上，将卷积层量化为 **int8**，将几何计算块（相关性块和捆绑调整块）保持为 **FP16/BF16**，以兼顾速度和数值稳定性。

*   **实验规模**：论文进行了详尽的消融实验，包括通道数缩减的影响、不同模块（TC, PYR, SA, GRU）移除后的精度损失分析，以及针对补丁图参数的帕累托（Pareto）前沿分析。

### 6. 主要结论与发现
*   **极致压缩**：TinyDEVO 相比 DEVO 实现了 **11.5 倍** 的内存缩减（降至 63.8 MB）和 **29.7 倍** 的计算量缩减（降至 5.2 GMAC/帧）。
*   **精度权衡**：在大幅压缩的情况下，TinyDEVO 的平均轨迹误差仅比原始 DEVO 高出约 **19 cm**，在 RPG 数据集上甚至优于传统的几何方法 EVO。
*   **可行性验证**：首次证明了在亚 100mW 功耗包络内运行完整深度事件 VO 流水线的可行性。

### 7. 优点
*   **硬件感知优化**：并非盲目压缩模型，而是针对 GAP9 处理器的 NE16 加速器和内存层级进行了定制化设计。
*   **精度保持**：通过帕累托分析找到了精度与算力的“膝点（Knee point）”，在极低算力下保留了核心的定位能力。
*   **端到端实现**：不仅有算法改进，还提供了在真实硬件上的完整功耗和延迟剖析。

### 8. 不足与局限
*   **实时性尚存差距**：1.2 FPS 的帧率对于高速运动的机器人（如竞速无人机）来说仍然偏低，属于“软实时”范畴。
*   **精度损失**：虽然 19 cm 的误差增量在可接受范围内，但在长距离导航中可能会产生更明显的漂移。
*   **场景依赖**：实验主要集中在室内和特定数据集，在极端动态或极度缺乏纹理的室外环境下的泛化能力仍需进一步验证。
