## 摘要
大语言模型 (LLM) 的部署正日益转向 Google 张量处理单元 (TPU) 等具有成本效益的加速器，同时优先考虑性能和总体拥有成本 (TCO)。然而，现有的 LLM 推理算子和推理系统在很大程度上仍以 GPU 为中心，目前尚无成熟的方法能将 LLM 工作负载高效地映射到 TPU 架构上，特别是在现代推理服务中常见的动态且不规则 (ragged) 的执行模式下。在本文中，我们提出了 Ragged Paged Attention (RPA)，这是一种使用 Pallas 和 Mosaic 实现的面向 TPU 的高性能且灵活的注意力算子。RPA 通过三项关键技术解决了这些挑战：(1) 细粒度分块 (tiling)，以实现在不规则内存上的高效动态切片；(2) 自定义软件流水线，将 KV 缓存更新与注意力计算融合；(3) 分布感知编译策略，为解码 (decode)、预填充 (prefill) 和混合工作负载生成专用算子。在 TPU7x 上对 Llama 3 8B 进行的评估表明，RPA 在解码阶段实现了高达 86% 的内存带宽利用率 (MBU)，在预填充阶段实现了 73% 的模型 FLOPs 利用率 (MFU)。作为 vLLM 和 SGLang 中的主要 TPU 后端，RPA 为高效的 TPU 推理提供了生产级基础，并为算子设计提供了实用见解。

## Abstract
Large Language Model (LLM) deployment is increasingly shifting to cost-efficient accelerators like Google's Tensor Processing Units (TPUs), prioritizing both performance and total cost of ownership (TCO). However, existing LLM inference kernels and serving systems remain largely GPU-centric, and there is no well-established approach for efficiently mapping LLM workloads onto TPU architectures--particularly under the dynamic and ragged execution patterns common in modern serving. In this paper, we present Ragged Paged Attention (RPA), a high-performance and flexible attention kernel for TPUs, implemented using Pallas and Mosaic. RPA addresses these challenges through three key techniques: (1) fine-grained tiling to enable efficient dynamic slicing over ragged memory, (2) a custom software pipeline that fuses KV cache updates with attention computation, and (3) a distribution-aware compilation strategy that generates specialized kernels for decode, prefill, and mixed workloads. Evaluated on Llama 3 8B on TPU7x, RPA achieves up to 86% memory bandwidth utilization (MBU) in decode and 73% model FLOPs utilization (MFU) in prefill. Integrated as the primary TPU backend in vLLM and SGLang, RPA provides a production-grade foundation for efficient TPU inference and offers practical insights into kernel design.

### 1. 核心问题与研究动机
*   **研究背景**：随着 LLM 部署规模扩大，TPU 因其高性价比（TCO）成为重要选择。然而，现有的高性能注意力算子（如 FlashAttention 系列）主要针对 GPU 优化。
*   **核心问题**：
    *   **架构失配**：TPU 的 XLA 编译器擅长处理静态形状，而现代推理框架（如 vLLM）采用“不规则”执行模式（混合不同长度的请求、混合预填充和解码阶段），导致 XLA 难以自动优化。
    *   **内存管理难题**：TPU 的内存布局是分块（Tiled）且粗粒度的，在处理非连续的 Paged KV Cache 更新和动态切片时，容易产生高昂的 Scatter/Gather 开销。
    *   **缺乏公开实现**：在开源社区中，缺乏针对 TPU 优化且能直接用于生产环境的高性能注意力内核。

### 2. 方法论：核心思想与关键技术
RPA 利用 JAX 的底层内核语言 **Pallas** 和编译器 **Mosaic** 实现，核心技术包括：
*   **细粒度分块（Fine-Grained Tiling）**：通过引入“打包维度（Packing Dimension）”，强制 XLA 选择最小的分块大小，从而在不规则内存缓冲区上实现高效的动态切片，避免了昂贵的解包和重新打包操作。
*   **KV 缓存更新融合流水线（KV-update-fused Pipeline）**：
    *   设计了自定义软件流水线，利用异步 DMA 操作将 KV Cache 的更新（写入 HBM）与注意力计算（TensorCore 运算）重叠。
    *   通过合并 K 和 V 头部（KV Merge），减少了向量加载/存储指令的数量，并提高了 DMA 传输效率。
*   **分布感知编译策略（Distribution-Aware Compilation）**：根据运行时序列长度的分布（如全解码、全预填充或混合模式），分发专门优化的内核版本，以最大化寄存器利用率并减少填充（Padding）浪费。
*   **KV 转置融合**：在数据加载路径中利用步进向量加载（Strided Vector Loads）直接实现转置，消除了显式的转置算子开销。

*   **解码性能**：在上下文长度达到 8K 以上时，RPA 能达到 **86% 的 MBU**，证明其能极好地饱和 TPU 的内存带宽。

### 7. 优点
*   **填补空白**：填补了 TPU 领域缺乏高性能、开源、支持 Paged Attention 内核的空白。
*   **深度融合**：不同于常规做法，RPA 将 KV Cache 更新直接融合进注意力内核，极大地减少了 TensorCore 的 Scatter 开销。
*   **灵活性强**：支持 MHA、MQA、GQA 等多种注意力变体，且能通过简单的块大小调整适配 TPU v4 到 v7 各代硬件。

### 8. 不足与局限
*   **预处理开销**：在短序列或计算未饱和的情况下，预处理（如维度重排）占总延迟的 2%–8%，虽然可以通过离线处理缓解，但目前仍存在。
*   **混合批处理的复杂性**：在高度不规则的混合批处理中，由于必须对齐到静态分块大小，仍存在不可避免的计算浪费（Padding）。
*   **硬件特性利用不全**：目前尚未利用 TPU 的 **SparseCore (SC)**，这在处理极度稀疏或复杂的索引计算时可能进一步提升性能。
*   **量化支持有限**：目前仅支持每张量的简单量化，更精细的（如每通道、每头）量化方案尚需进一步开发。
