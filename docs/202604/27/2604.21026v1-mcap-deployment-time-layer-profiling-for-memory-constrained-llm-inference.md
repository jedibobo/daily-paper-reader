## 摘要
将大语言模型部署到异构硬件时，通常受限于内存而非计算能力。我们引入了 MCAP（蒙特卡洛激活剖析），这是一种加载时的逐层重要性估计器，能够在目标设备上实现动态精度和内存放置决策。MCAP 生成轻量级的逐层信号，用于驱动精度调度（如 W4A8 与 W4A16）和驻留层级（GPU、RAM、SSD），从而允许同一组权重在不同的内存预算下运行。我们的系统 NVE 在 NVIDIA T4 上的解码吞吐量比 llama.cpp Q4_0 高出 1.5-1.8 倍，并使模型能够在无需修改权重的情况下，在以前无法实现的内存状态下运行。

## Abstract
Deploying large language models to heterogeneous hardware is often constrained by memory, not compute. We introduce MCAP (Monte Carlo Activation Profiling), a load-time per-layer importance estimator that enables dynamic precision and memory placement decisions on the target device. MCAP produces a lightweight per-layer signal that drives both precision dispatch (W4A8 vs. W4A16) and residency tier (GPU, RAM, SSD), allowing a single set of weights to operate across diverse memory budgets. Our system, NVE, achieves 1.5-1.8x higher decode throughput than llama.cpp Q4_0 on NVIDIA T4 and enables models to run in memory regimes previously infeasible without modifying weights.

大语言模型（LLM）在异构硬件（如旧款 GPU、移动端、集成显卡）上的部署瓶颈在于**内存容量**而非计算能力。现有的后训练量化（PTQ）方法在校准阶段就固定了精度，导致量化后的模型与特定硬件深度绑定，缺乏灵活性。

### 2. 方法论：核心思想与关键技术
论文构建了一个名为 **NVE** 的推理引擎（Rust+CUDA 编写），核心包含三个耦合组件：

*   **MCAP (蒙特卡洛激活剖析器)：**
    *   **核心思想：** 在加载模型时，使用 12 个校准提示词进行 60 秒的无梯度前向传播，计算每一层的激活期望。
    *   **重要性评分：** 结合了注意力机制代理（$Q$ 和 $V$ 投影的 $L2$ 范数）和 FFN 层的输出量级。

    *   根据 MCAP 评分，将层路由至不同的内核：高重要性层使用 **W4A16**（4位权重，16位激活），低重要性层使用 **W4A8**（利用 CUDA 的 `dp4a` 指令实现 4位权重 × 8位激活）。
*   **三级虚拟权重分页 (3-Tier Virtual Weight Paging)：**
    *   将权重分布在 **GPU VRAM (Tier 0)**、**CPU RAM (Tier 1)** 和 **SSD (Tier 2)**。
    *   利用 MCAP 信号驱动初始放置，并结合 LRU 算法和基于点互信息（PMI）的预取机制，确保高频访问的“热层”驻留在 GPU 中。
