## 摘要
大型视觉语言模型 (VLM) 正在实现交互式视频推理，从而催生了流式长视频理解。在这种场景下，视频帧连续到达，系统在保留长期上下文的同时，需在严格的延迟约束下生成响应。一个核心挑战是 KVCache 管理：随着视频流的增长，KVCache 迅速膨胀，增加了计算和内存开销。现有的基于检索的方法利用注意力稀疏性，并将非活跃的 KVCache 从 GPU 卸载到 CPU 内存，但其 Token 级的设计导致了高昂的管理开销和碎片化的数据移动。我们提出了 Mosaic，这是首个用于流式长视频理解的聚类驱动型 VLM 推理系统。我们的核心见解是，VLM 的 KVCache 表现出一种隐式的跨模态聚类结构：检索到的 KV 状态形成的群体由视觉连贯性和语义相关性共同塑造。基于这一观察，Mosaic 将跨模态聚类作为 KVCache 组织、维护和检索的基本单位。评估结果表明，Mosaic 的性能优于最先进的基准方法，实现了高达 1.38 倍的加速。

## Abstract
Large vision-language models (VLMs) are enabling interactive video reasoning, giving rise to streaming long-video understanding. In this setting, frames arrive continuously, while the system preserves long-term context and generates responses under strict latency constraints. A central challenge is KVCache management: as video streams grow, KVCache expands rapidly, increasing computation and memory overhead. Existing retrieval-based approaches exploit attention sparsity and offload inactive KVCache from GPU to CPU memory, but their token-level design causes high management overhead and fragmented data movement. We present Mosaic, the first cluster-driven VLM inference system for streaming long-video understanding. Our key insight is that VLM KVCache exhibits an implicit cross-modal clustering structure: retrieved KV states form groups jointly shaped by visual coherence and semantic relevance. Based on this observation, Mosaic uses cross-modal clusters as the basic unit of KVCache organization, maintenance, and retrieval. Evaluations show that Mosaic outperforms state-of-the-art baselines, achieving up to 1.38x speedup.

### 1. 核心问题与整体含义（研究动机和背景）
随着大视觉语言模型（VLM）在自动驾驶、具身智能和视频问答等领域的应用，**流式长视频理解**成为核心挑战。这类任务具有两个显著特征：视频极长（成百上千帧）且帧是连续到达的。
*   **现有瓶颈：** 随着视频增长，KVCache（键值缓存）迅速膨胀，导致内存和计算开销剧增。
*   **现有方案缺陷：** 目前基于检索的 KVCache 卸载技术（将非活跃数据移至 CPU）主要采用 **Token 粒度**。这导致了高昂的管理开销（每帧都要评分、索引）和严重的 I/O 碎片化（频繁传输细小的数据块，带宽利用率低）。
*   **研究动机：** 作者发现 VLM 的 KVCache 存在隐式的“跨模态聚类”结构，即检索到的 KV 状态通常是成组出现的，且受视觉连贯性和语义相关性共同驱动。

### 2. 方法论：核心思想与关键技术
Mosaic 提出了首个**聚类驱动**的 VLM 推理系统，将 KVCache 的组织、维护和检索单位从 Token 提升到“簇（Cluster）”。

*   **跨模态构造器（Cross-Modal Constructor）：**
    *   采用**嵌套式视觉-语义聚类**。首先在视觉空间（由视觉编码器诱导）对帧进行粗粒度分区；然后在每个视觉簇内，根据 Transformer 层的语义空间（Key 向量）进行细粒度聚类。
    *   在 GPU 上维护一个轻量级索引，存储每个簇的质心（Representative Vector）作为检索依据。
*   **自适应维护器（Self-Adaptive Maintainer）：**
    *   **凝聚力感知更新：** 实时跟踪簇内方差（Variance）。新帧到达时，若加入后方差超过自适应阈值，则标记该簇为“无效”。
    *   **延迟分裂（Deferred Splitting）：** 为了减少 I/O，无效簇不会立即在 CPU/GPU 间移动进行重聚类，而是先存入 GPU 缓冲区，直到该簇下次被检索加载时才执行分裂操作。
*   **高性能执行器（High-Performance Executor）：**
    *   **批处理执行：** 将单帧操作转化为矩阵运算，提高 GPU 利用率。
    *   **重叠感知预取（Overlap-Aware Prefetch）：** 利用相邻层查询向量（Query）的相似性，在当前层计算时异步预取下一层可能需要的簇，隐藏 I/O 延迟。

*   **模型：** LLaVA-OneVision-7B, Qwen2.5-VL-7B, Qwen2.5-VL-3B。

### 5. 实验数量与充分性
*   **实验规模：** 涵盖了 3 种不同架构和规模的模型，在 6 个不同场景的数据集上进行了测试。
*   **消融实验：** 详细验证了嵌套聚类、延迟分裂、批处理执行和跨层预取各自的贡献。

### 6. 主要结论与发现
*   **性能提升：** Mosaic 相比最先进的基线方法（如 ReKV），实现了高达 **1.38 倍** 的推理加速。
*   **显存优化：** 在处理 1024 帧视频时，显存占用减少了约 **19%**，相比原始模型减少了 **2.22 倍**，有效防止了长视频下的 OOM（显存溢出）。
*   **准确率：** 在所有测试中，Mosaic 的准确率均优于或持平于基线，证明了聚类检索能比 Token 检索更好地保留长程上下文。

### 7. 优点
*   **视角新颖：** 首次识别并利用了 VLM KVCache 的跨模态聚类特性，从系统底层改变了管理粒度。
*   **算法与系统协同：** 不仅有数学上的聚类优化，还针对硬件特性设计了延迟分裂和预取机制，解决了 I/O 瓶颈。
*   **即插即用：** 无需对模型进行重新训练或微调，具有很强的通用性。

### 8. 不足与局限
*   **阈值依赖：** 自适应分裂阈值（$\tau$）的设定可能需要针对不同模型架构进行微调。
*   **极端稀疏性风险：** 虽然聚类能保留更多上下文，但在极端压缩情况下，仍可能丢失某些极细粒度的视觉细节。
*   **硬件环境：** 性能增益高度依赖于 CPU-GPU 之间的互连带宽，在带宽极低的设备上效果可能受限。
