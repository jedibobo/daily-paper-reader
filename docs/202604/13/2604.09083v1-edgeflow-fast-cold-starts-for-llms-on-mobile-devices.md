## 摘要
在移动设备上部署大语言模型（LLMs）是实现 LLM 应用数据隐私和离线可访问性的一种新兴趋势。现代移动神经网络处理器（NPUs）使得这种部署变得日益可行。然而，现有的移动端 LLM 推理框架由于不可避免的冷启动（即在模型未驻留在设备内存时启动 LLM 推理）而面临高启动延迟的问题。在本文中，我们发现移动端 LLM 冷启动的关键瓶颈在于不重要的模型参数浪费了闪存带宽。我们设计了 EdgeFlow，这是一个通过自适应调整 LLM 参数精度来缓解冷启动问题的移动端 LLM 推理框架。具体而言，EdgeFlow 利用了：1）一种感知 NPU 的自适应量化算法，根据权重的重要性和 NPU 约束，以更细的粒度为权重分配不同的精度；2）一种 SIMD 友好的打包格式，可加速将各种精度的权重转换为固定大小的 NPU 原生数据类型；3）一种协同的粒度流水线，以细粒度和动态的方式协调 CPU 与 NPU 的计算。实验结果表明，在模型准确率相当的情况下，与 llama.cpp、MNN 和 llm.npu 这三种最先进的移动端 LLM 推理框架相比，EdgeFlow 将冷启动延迟降低了高达 4.07 倍。

## Abstract
Deploying large language models (LLMs) on mobile devices is an emerging trend to enable data privacy and offline accessibility of LLM applications. Modern mobile neural processing units (NPUs) make such deployment increasingly feasible. However, existing mobile LLM inference frameworks suffer from high start-up latency due to their inevitable cold starts, i.e., launching LLM inferences when the model is not hosted in device memory. In this paper, we identify the key bottleneck of mobile LLM cold starts as the waste of flash bandwidth on unimportant model parameters. We design EdgeFlow, a mobile LLM inference framework that mitigates the cold start issue by adaptively adjusting the precisions of LLM parameters. Specifically, EdgeFlow leverages 1) an NPU-aware adaptive quantization algorithm that assigns different precisions to weights in a finer granularity according to their importance and NPU constraints, 2) an SIMD-friendly packing format that accelerates the transformation of various-precision weights into fixed-sized NPU-native data types, and 3) a synergistic granular pipeline that coordinates CPU and NPU computation in a fine-grained and dynamic manner. Experimental results show that EdgeFlow reduces cold-start latency by up to 4.07x compared with three state-of-the-art mobile LLM inference frameworks, i.e., llama.cpp, MNN, and llm.npu, under comparable model accuracy.

*   **研究动机**：随着 LLM 在移动端的普及，隐私和离线访问需求增加。然而，移动端内存有限，模型常被系统清理，导致每次调用时需从闪存（Flash）重新加载。
*   **核心瓶颈**：现有框架在冷启动时面临巨大的“首字延迟”（TTFT）。作者发现，瓶颈不在于计算，而在于**闪存带宽的浪费**——现有框架对所有权重进行均匀量化，导致大量不重要的参数占用了宝贵的传输带宽。
*   **研究目标**：通过软硬件协同设计，在保证模型准确率的前提下，最大限度地降低从闪存加载模型并启动推理的时间。

### 2. 方法论
EdgeFlow 采用了三项核心技术：
*   **NPU 感知的自适应量化 (NPU-Aware Adaptive Quantization)**：
    *   **核心思想**：根据权重对模型准确率的重要性，以“输出通道”为粒度分配不同的位宽（1-8 bit）。
    *   **技术细节**：引入“相对误差（Relative Error）”指标，通过建模随机变量的期望值来快速估算量化误差，并使用**贪心位宽分配算法**在带宽预算内最小化总误差。
    *   **NPU 适配**：针对移动端 NPU（如高通 Hexagon）仅支持均匀、对称量化的限制，提出“NPU 感知平滑”技术，将激活值的方差静态转移到权重中。
*   **SIMD 友好的打包格式 (SIMD-Friendly Packing Format)**：
    *   **核心思想**：解决多精度权重解包慢的问题。将权重分解为 1、2、4 位的“微权重（Weightlets）”，并采用交错布局（Interleaved Layout）。
    *   **技术细节**：利用 Arm Neon 指令集的 SIMD（单指令多数据）并行性，通过掩码（Mask）、对齐（Align）和合并（Merge）三步操作，极速将压缩权重还原为 NPU 原生的 INT8 格式。
*   **协同粒度流水线 (Synergistic Granular Pipeline)**：
    *   **核心思想**：打破 CPU 和 NPU 之间粗粒度、静态的任务划分。
    *   **技术细节**：
        *   **细粒度算子放置**：将矩阵乘法（Matmul）放在 NPU，将低算术强度的算子（如 Norm、Add）放在 CPU。
        *   **动态调度**：引入“位置引导优先级”，优先处理 Prompt 前部的 Chunk 以解锁后续 NPU 任务；支持“任务窃取（Task Stealing）”，当 CPU 空闲时主动承担 NPU 队列中的任务。
