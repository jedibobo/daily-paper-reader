## 摘要
在具有多个异构加速引擎的片上系统（SoC）上部署深度神经网络（DNN）具有挑战性，且大多数部署框架无法充分利用异构性。我们提出了 MATCHA，这是一个统一的 DNN 部署框架，它能为并行异构加速器生成高并发调度，并利用约束规划（constraint programming）来优化 L3/L2 内存分配与调度。通过在各个硬件单元之间使用模式匹配、分块（tiling）和映射，实现了并行执行和高加速器利用率。在 MLPerf Tiny 基准测试中，使用具有两个异构加速器的 SoC，MATCHA 与最先进的 MATCH 编译器相比，提高了加速器利用率，并将推理延迟降低了高达 35%。

## Abstract
Deploying DNNs on System-on-Chips (SoC) with multiple heterogeneous acceleration engines is challenging, and the majority of deployment frameworks cannot fully exploit heterogeneity. We present MATCHA, a unified DNN deployment framework that generates highly concurrent schedules for parallel, heterogeneous accelerators and uses constraint programming to optimize L3/L2 memory allocation and scheduling. Using pattern matching, tiling, and mapping across individual HW units enables parallel execution and high accelerator utilization. On the MLPerf Tiny benchmark, using a SoC with two heterogeneous accelerators, MATCHA improves accelerator utilization and reduces inference latency by up to 35% with respect to the the state-of-the-art MATCH compiler.

### 1. 核心问题与研究动机
*   **核心问题**：现代边缘片上系统（SoC）集成了多种异构加速器（如SIMD单元、向量处理器、矩阵乘法引擎等），但现有的部署框架（如TFLite、DORY、MATCH等）通常只能以**串行方式**利用这些资源。即在任一时刻，只有一个加速器在工作，其余处于闲置状态，导致硬件利用率低下。
*   **研究动机**：为了在严苛的延迟和功耗约束下提升边缘推理性能，需要一种能够同时利用层级并行（不同分支在不同加速器运行）和分块级并行（同一层的不同分块在不同加速器运行）的异步调度框架。

### 2. 方法论
MATCHA（MATCH Asynchronous）是一个基于TVM和MATCH构建的优化编译器，其核心思想是将DNN映射问题转化为一个**分块中心（Tile-centric）的约束规划（CP）问题**。
*   **分块级模式匹配**：不同于传统框架将整个算子映射到单一设备，MATCHA允许将一个算子拆分为多个分块（Tiles），并根据各加速器的效率（$\eta$）和速度（$\alpha$）分配给不同的硬件单元。
*   **约束规划优化**：利用CP求解器联合优化模式匹配、设备分配和平台调度。目标函数是最小化总完工时间（Makespan），同时满足内存容量和数据依赖约束。
*   **内存与调度规划**：将L2/L3内存分配建模为二维装箱问题（时间×地址），支持静态分配、动态交换和按需加载。
*   **底层映射细化**：集成ZigZag工具，针对加速器内部的L1/L2内存层级进行循环分块、展开和顺序优化。
*   **异步运行时**：生成支持多ISA的二进制文件，通过邮箱（Mailbox）和中断机制实现主机与加速器之间的异步任务分发和同步。
