## 摘要
我们展示了在任何 GPU 平台上首个内核融合的 SAR 距离-多普勒流水线。通过将 FFT、匹配滤波乘法和 IFFT 融合进单个 Metal 计算调度中，并将所有中间数据保留在 32 KiB 片上内存内，我们在 Apple M1 GPU 上仅用 370 毫秒便处理了一个 4096 × 4096 的复数 SAR 场景，相比多调度基准（8.16 秒）实现了 22 倍的加速。我们还报告了首个利用 Apple simdgroup_matrix 8×8 硬件 MMA 的 FFT 实现，该实现采用原位 Cooley-Tukey 频域抽取公式，与 Stockham 算法相比内存占用减半。雷达图像质量得以保持：所有五个点目标的信噪比（SNR）偏差与未融合的 FP32 参考值相比均为 0.0 dB。

## Abstract
We present the first kernel-fused SAR Range Doppler pipeline on any GPU platform. By fusing FFT, matched-filter multiply, and IFFT into a single Metal compute dispatch -- keeping all intermediate data in 32\,KiB on-chip memory -- we process a $4096\!\times\!4096$ complex SAR scene in \textbf{370\,ms} on an Apple M1 GPU, a \textbf{22$\times$} speedup over the multi-dispatch baseline (8.16\,s). We further report the first FFT to exploit Apple's \texttt{simdgroup\_matrix} 8$\times$8 hardware MMA, enabled by an in-place Cooley--Tukey decimation-in-frequency formulation that halves the memory footprint versus Stockham. Radar image quality is preserved: all five point targets show 0.0\,dB SNR deviation from the unfused FP32 reference.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：传统的 SAR 距离-多普勒算法（Range Doppler Algorithm, RDA）在 GPU 上运行时，通常将 FFT（快速傅里叶变换）、匹配滤波乘法和 IFFT（逆快速傅里叶变换）作为独立的内核（Kernel）分别调度。这导致了大量的中间数据在显存（Device Memory）与计算核心之间反复读写，产生了巨大的内存带宽开销。
*   **研究背景**：虽然 NVIDIA GPU 已有成熟的 SAR 加速方案，但在 Apple Silicon 和 Metal 框架下尚无针对性优化。此外，现有的 GPU 实现大多未实现深度内核融合（Kernel Fusion）。

### 2. 论文提出的方法论
*   **单调度内核融合（Single-Dispatch Fusion）**：将“正向 FFT -> 点对点复数乘法 -> 逆 FFT”三个步骤融合进同一个 Metal 计算调度中。
*   **片上内存管理**：利用 Apple Silicon GPU 的 32 KiB 线程组内存（Threadgroup Memory）存储中间数据。对于 4096 点的复数 float32 数据，其工作集正好为 32 KiB，从而实现了数据在整个处理流程中无需写回显存。
*   **硬件加速矩阵乘法（MMA）FFT**：
    *   首次在 Apple GPU 上利用 `simdgroup_matrix` 8×8 硬件加速单元实现 FFT。
    *   采用**原位 Cooley-Tukey 频域抽取（DIF）**算法，相比传统的 Stockham 算法，其内存占用减半，解决了 MMA 模式下因实部/虚部拆分布局导致的内存不足问题。
*   **共轭 FFT 实现 IFFT**：通过“共轭 -> 正向 FFT -> 共轭”的数学等价性，复用了正向 FFT 的计算逻辑，减少了代码复杂度和指令开销。
