## 摘要
生物信号表现出显著的跨受试者和跨会话变异性，导致严重的领域偏移，从而降低了面向边缘的小型人工智能模型在部署后的性能。因此，设备端自适应对于保护用户隐私和确保系统可靠性至关重要。然而，由于全反向传播（BP）极高的内存占用和计算成本，现有的基于微控制器（MCU）且功耗低于100毫瓦的穿戴式平台仅能支持浅层或稀疏的自适应方案。在本文中，我们提出了 BioTrain，这是一个能够在毫瓦级功耗和亚兆字节内存限制下，实现最先进生物信号模型全网络微调的框架。我们利用脑电图（EEG）和眼电图（EOG）数据集，通过离线和设备端基准测试对 BioTrain 进行了验证，涵盖了首日新受试者校准以及针对信号漂移的长期自适应。实验结果表明，在进行新受试者校准时，全网络微调相比未自适应的基准模型实现了高达 35% 的准确率提升，且比仅更新最后一层的方法高出约 7%。在 GAP9 MCU 平台上，BioTrain 在低于 50 毫瓦的功耗范围内，实现了 EEG 模型 17 样本/秒和 EOG 模型 85 样本/秒的高效设备端训练吞吐量。此外，BioTrain 高效的内存分配器和网络拓扑优化支持使用较大的批次大小，从而降低了峰值内存占用。对于 GAP9 上的全片上反向传播，与使用批归一化且批次大小为 8 的传统全网络微调相比，BioTrain 将内存占用降低了 8.1 倍，从 5.4 MB 降至 0.67 MB。

## Abstract
Biosignals exhibit substantial cross-subject and cross-session variability, inducing severe domain shifts that degrade post-deployment performance for small, edge-oriented AI models. On-device adaptation is therefore essential to both preserve user privacy and ensure system reliability. However, existing sub-100 mW MCU-based wearable platforms can only support shallow or sparse adaptation schemes due to the prohibitive memory footprint and computational cost of full backpropagation (BP). In this paper, we propose BioTrain, a framework enabling full-network fine-tuning of state-of-the-art biosignal models under milliwatt-scale power and sub-megabyte memory constraints. We validate BioTrain using both offline and on-device benchmarks on EEG and EOG datasets, covering Day-1 new-subject calibration and longitudinal adaptation to signal drift. Experimental results show that full-network fine-tuning achieves accuracy improvements of up to 35% over non-adapted baselines and outperforms last-layer updates by approximately 7% during new-subject calibration. On the GAP9 MCU platform, BioTrain enables efficient on-device training throughput of 17 samples/s for EEG and 85 samples/s for EOG models within a power envelope below 50 mW. In addition, BioTrain's efficient memory allocator and network topology optimization enable the use of a large batch size, reducing peak memory usage. For fully on-chip BP on GAP9, BioTrain reduces the memory footprint by 8.1x, from 5.4 MB to 0.67 MB, compared to conventional full-network fine-tuning using batch normalization with batch size 8.

*   **核心问题**：生物信号具有极强的**跨受试者变异性**和**非平稳性**（随时间漂移）。预训练的 AI 模型在部署到新用户或长期使用时，性能会显著下降。
*   **研究背景**：现有的边缘设备（如功耗低于 100mW 的微控制器 MCU）由于内存（RAM）极度受限，通常只能进行“线性探测”（仅更新最后一层）或稀疏更新。然而，这些浅层自适应方法难以应对复杂的生物信号波动。
*   **整体含义**：BioTrain 旨在打破“内存墙”，在亚兆字节（Sub-MB）内存和毫瓦级（Sub-50mW）功耗约束下，实现**全网络反向传播（Full-BP）微调**，从而在保护隐私的同时提供个性化的医疗监测。

### 2. 方法论：核心思想与关键技术
BioTrain 是一个基于编译器（Deeploy）的部署框架，其核心思想是通过**内存编排**和**算法优化**实现高效的设备端训练：
*   **拓扑优化（Edge-FT）**：将传统的批归一化（Batch Normalization, BN）替换为**组归一化（Group Normalization, GN）**。GN 不依赖跨样本统计，配合**梯度累积（Gradient Accumulation）**技术，可以在极小的物理批次（甚至 Batch Size=1）下模拟大批次训练效果，大幅降低峰值内存。
*   **编译器驱动的内存管理**：
    *   **前端**：自动分解梯度计算图，支持 GN 和卷积梯度的算子拆分。
    *   **中端（平铺与分配）**：利用静态内存分配和**平铺（Tiling）策略**，将长序列生物信号切片处理，确保数据在 L1（高速缓存）和 L2（主存）之间高效流动。
    *   **后端**：集成了针对 RISC-V 优化的梯度内核（基于 PULPTrainLib），支持 FP32 全精度训练。
*   **自动化流程**：从 PyTorch 模型导出到生成裸机 C 代码，实现了端到端的自动化部署。

### 3. 实验设计
*   **数据集**：
    1.  **EEG 数据集**：运动想象任务（舌头运动 vs 休息），5 个受试者，42 个会话。
    2.  **EOG 数据集**：眼动分类（11 类），5 个受试者，2 个会话。
*   **实验场景**：
    *   **场景 A（首日校准）**：新用户首次使用时的模型初始化。
    *   **场景 B（长期自适应）**：模拟模型在多天使用中针对信号漂移的持续更新。
*   **对比基准（Baselines）**：
    *   **No-FT**：无自适应（零样本）。
    *   **LP（Linear Probing）**：仅更新最后一层分类器。
    *   **Full-FT**：传统的全网络微调（带 BN）。
    *   **Edge-FT**：BioTrain 提出的优化方案（带 GN 和梯度累积）。

*   **硬件平台**：**GAP9 MCU**（集成 9 核 RISC-V 处理器，128KB L1 SRAM, 1.5MB L2 SRAM）。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了两种不同的生物信号模态（电生理信号的不同频率和特征），针对每个受试者和会话进行了多次重复实验（5 次随机种子取平均）。
*   **充分性**：实验设计较为充分，不仅对比了准确率，还深入分析了内存占用、功耗、能量效率和吞吐量。

### 6. 主要结论与发现
*   **准确率提升**：全网络微调（Edge-FT）在 EEG 任务上比未适配基准提升了 **35%**，比仅更新最后一层（LP）提升了约 **7%**。
*   **内存缩减**：通过 BioTrain 的优化，峰值内存占用从 5.4 MB 降低到 **0.67 MB**（缩减 **8.1 倍**），使得全网络 BP 能够完全在片上 L2 内存中运行。
*   **能效极高**：使用标准的 320mAh 电池，设备可以支持超过 **200 次（EEG）到 900 次（EOG）**完整的训练会话（每会话 40 轮迭代）。

### 7. 优点
*   **端到端自动化**：解决了嵌入式开发中手动编写梯度下降代码的巨大困难。
*   **内存效率极高**：通过编译器层面的平铺技术和算法层面的 GN 替换，成功在极小内存设备上运行了原本需要数 MB 内存的模型。
*   **实用性强**：针对生物信号的“首日校准”和“长期漂移”两个痛点给出了切实可行的方案。

### 8. 不足与局限
*   **精度限制**：目前主要支持 **FP32（单精度浮点）**训练。虽然 GAP9 支持浮点运算，但在更低端的定点 MCU 上可能无法直接运行，未来需要引入量化训练（Quantized Training）。
*   **模型规模**：实验主要针对小型 CNN（参数量在 4k-8k 左右），对于参数量更大的 Transformer 或深层网络，内存压力依然巨大。
*   **应用范围**：目前仅验证了监督学习场景，对于缺乏标签的无监督/自监督在线学习尚未深入探讨。
