## 摘要
稀疏矩阵乘法 (SpMM) 是科学计算和机器学习中的基础算子。尽管先前的工作利用 Tensor Core 加速了 SpMM，但现有的稀疏算子均未利用现代 GPU 架构的异步特性，例如 NVIDIA 的张量内存加速器 (TMA) 和线程束专业化 (warp specialization)。本研究系统地探讨了这些特性对 SpMM 性能的影响，并引入了两个协同设计的算子。针对结构化稀疏性，我们利用分块压缩稀疏行 (BCSR) 格式，优化了一个线程束专业化的生产者-消费者流水线，实现了 TMA 数据传输与 WGMMA 计算的重叠。针对非规则稀疏性，我们设计了一个窗口压缩稀疏行 (WCSR) 算子，该算子通过 TMA 加载稀疏操作数，并将大型行窗口拆分到不同的线程块中以实现负载均衡。我们的 WCSR 算子在 SuiteSparse 矩阵上的表现优于所有先前的 SpMM 算子（相比 AccSpMM 提升 1.47 倍，相比 cuSPARSE 提升 6.24 倍）。我们的 BCSR 算子在 90% 分块稀疏度、64K token 的 Qwen2.5-7B 预填充阶段，相比 cuDNN/cuBLAS 实现了 2.66 倍的综合端到端加速。

## Abstract
Sparse Matrix-Matrix Multiplication (SpMM) is a fundamental kernel across scientific computing and machine learning. While prior work accelerates SpMM using Tensor Cores, no existing sparse kernel exploits the asynchronous features of modern GPU architectures, such as NVIDIA's Tensor Memory Accelerator (TMA) and warp specialization. This work systematically studies how these features impact SpMM performance and introduces two co-designed kernels. For structured sparsity, we optimize a warp-specialized producer-consumer pipeline overlapping TMA data transfer with WGMMA computation using Block Compressed Sparse Row (BCSR) format. For irregular sparsity, we design a Window Compressed Sparse Row (WCSR) kernel that loads the sparse operand via TMA and splits large row-windows across thread blocks for load balancing. Our WCSR kernel outperforms all prior SpMM kernels on SuiteSparse matrices (1.47x over AccSpMM, 6.24x over cuSPARSE). Our BCSR kernel achieves a combined 2.66x end-to-end speedup on Qwen2.5-7B prefill at 90% block sparsity with 64K tokens over cuDNN/cuBLAS.

然而，现有的稀疏算子库（如 cuSPARSE）和研究成果（如 AccSpMM）大多仍沿用传统的同步编程模型，未能充分利用这些硬件特性来掩盖内存访问延迟。论文的研究动机在于：通过软硬件协同设计，系统性地探索异步架构特性对 SpMM 性能的影响，并填补高性能异步稀疏算子的空白。

#### 2. 论文提出的方法论
论文提出了 **AsyncSparse** 框架，包含两种针对不同稀疏模式设计的内核：

*   **针对结构化稀疏的 BCSR 内核**：
    *   **核心思想**：采用分块压缩稀疏行（BCSR）格式，利用 Hopper 的生产者-消费者模型。
    *   **技术细节**：将一个线程块（CTA）分为一个生产者线程束组（Producer Warpgroup）和两个消费者线程束组（Consumer Warpgroups）。生产者利用 TMA 异步地从全局内存抓取稀疏和密集操作数到共享内存；消费者则利用 WGMMA 指令直接从共享内存读取数据进行计算。
    *   **流水线设计**：通过多阶段循环缓冲区（Circular Buffer）和异步屏障（mbarrier）实现数据搬运与计算的完全重叠。
*   **针对非规则稀疏的 WCSR 内核**：
    *   **核心思想**：设计窗口压缩稀疏行（WCSR）格式，以更细粒度压缩列向量。
    *   **技术细节**：由于密集矩阵访问是非连续的，TMA 无法直接处理间接寻址，因此采用单线程束组设计。所有线程协作抓取密集矩阵数据，同时利用 TMA 加载稀疏操作数。
    *   **负载均衡**：引入基于任务的分解机制，将大型行窗口拆分为固定大小的子任务，分配给不同的线程块执行，并使用原子加（atomicAdd）汇总结果。
