## 摘要
近年来，视频语言模型（VLM）已应用于各个领域。然而，VLM 的视觉 token 序列过长，可能导致难以忍受的推理延迟和 GPU 显存占用。现有方法提出了基于 token 粒度的 VLM 键值（KV）缓存混合精度量化，但这在搜索过程中耗时较长，且在计算过程中硬件效率低下。本文介绍了一种名为 WindowQuant 的新方法，它采用窗口自适应混合精度量化来优化 KV 缓存。WindowQuant 由两个模块组成：窗口级量化搜索和窗口级 KV 缓存计算。窗口级量化搜索根据相应视觉 token 窗口与文本提示之间的相似度得分，快速确定 KV 缓存窗口的最佳位宽配置，从而保持模型精度。此外，窗口级 KV 缓存计算在量化前对 KV 缓存窗口进行重排序，避免了推理计算中混合精度量化导致的硬件效率低下问题。大量实验表明，WindowQuant 在各种数据集上均优于最先进的 VLM 模型和 KV 缓存量化方法。

## Abstract
Recently, video language models (VLMs) have been applied in various fields. However, the visual token sequence of the VLM is too long, which may cause intolerant inference latency and GPU memory usage. Existing methods propose mixed-precision quantization to the key-value (KV) cache in VLMs based on token granularity, which is time-consuming in the search process and hardware inefficient during computation. This paper introduces a novel approach called WindowQuant, which employs window-adaptive mixed-precision quantization to optimize the KV cache. WindowQuant consists of two modules: window-level quantization search and window-level KV cache computation. Window-level quantization search quickly determines the optimal bit-width configuration of the KV cache windows based on the similarity scores between the corresponding visual token windows and the text prompt, maintaining the model accuracy. Furthermore, window-level KV cache computation reorders the KV cache windows before quantization, avoiding the hardware inefficiency caused by mixed-precision quantization in inference computation. Extensive experiments demonstrate that WindowQuant outperforms state-of-the-art VLM models and KV cache quantization methods on various datasets.

### WindowQuant：基于窗口级相似度的混合精度 KV 缓存量化总结

#### 1. 核心问题与整体含义
随着视频语言模型（VLM）的发展，处理长视频产生的视觉 Token 序列急剧增加，导致 **KV 缓存（KV Cache）** 消耗了海量的 GPU 显存并显著增加了推理延迟。现有的量化方法存在两难境地：均匀量化（所有 Token 相同位宽）会导致精度大幅下降；而 Token 级的混合精度量化虽然能保持精度，但其搜索过程极其耗时，且在硬件计算时由于数据对齐问题导致效率低下。本文的研究动机在于利用视觉信息与文本提示（Prompt）之间的相关性，在更粗的“窗口级”粒度上实现高效的混合精度量化。

#### 2. 方法论
WindowQuant 提出了一个自适应的窗口级混合精度量化框架，包含两个核心模块：
*   **窗口级量化搜索（Window-level Quantization Search）：**
    *   **相似度计算：** 将视觉 Token 划分为固定大小的窗口，计算每个窗口与文本 Prompt 之间的余弦相似度。
    *   **位宽分配：** 根据相似度得分和预设阈值（$T_{low}, T_{high}$），将窗口划分为 FP16（高相关）、INT4（中相关）和 INT2（低相关）。
    *   **层级敏感度：** 观察到不同层对量化的敏感度不同，通过计算层间隐藏状态的相似度来动态调整每层的量化阈值。
    *   **首窗口保护：** 强制保持第一个视觉 Token 窗口为 FP16，以保留关键的上下文锚点信息。
*   **窗口级 KV 缓存计算（Window-level KV Cache Computation）：**
    *   **窗口重排序：** 在物理内存中将相同位宽的 KV 缓存窗口重新排列在一起，解决混合精度导致的数据不连续和硬件计算效率低的问题。
    *   **算子融合（Operator Fusion）：** 设计了融合反量化与矩阵乘法的算子，减少 GPU 全局内存的读写次数，提升吞吐量。

#### 6. 主要结论与发现
*   **性能提升：** 相比 FP16 基准，WindowQuant 将解码吞吐量提升了约 **3 倍**，显存占用减少了约 **15%**（随帧数增加效果更明显）。
*   **精度保持：** 在多个 QA 任务上，WindowQuant 的表现甚至优于 FP16 模型，这可能是因为量化过程过滤了视频中的冗余噪声信息。
*   **硬件友好：** 窗口重排序技术成功解决了混合精度量化在现代 GPU 上的计算瓶颈。

#### 7. 优点
*   **粒度创新：** 采用窗口级而非 Token 级，在搜索速度和硬件执行效率之间取得了极佳的平衡。
*   **去噪效应：** 巧妙地利用相似度过滤无关视觉信息，实现了压缩与性能的双赢。
*   **系统级优化：** 不仅停留在算法层面，还通过算子融合和内存重排进行了底层系统优化。

#### 8. 不足与局限
*   **Prefill 阶段限制：** 由于 Prefill 阶段存在 Mask 矩阵，重排序会影响因果关系，因此目前仅在生成第一个 Token 后才进行重排量化，这限制了首 Token 延迟（TTFT）的优化。
*   **任务敏感性：** 在某些需要极高精细度视觉特征的任务（如动作计数 AC）中，窗口级量化仍会导致轻微的精度下降。
*   **显存瓶颈：** 尽管优化了 KV 缓存，但 FFN 层的中间激活值在 Prefill 阶段依然占据较大显存，这部分尚未在本文中得到解决。
