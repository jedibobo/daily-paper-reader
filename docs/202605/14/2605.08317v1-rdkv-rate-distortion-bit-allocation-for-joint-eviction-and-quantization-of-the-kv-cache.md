## 摘要
大语言模型（LLMs）在多种任务中表现出强大的性能，但其在长输入上下文下的推理受到内存容量和带宽的限制。键值（KV）缓存的大小随序列长度线性增长，且在每个解码步骤中都需要从片外高带宽内存（HBM）重新读取到片内内存，导致推理过程受内存限制。现有方法通过剔除（eviction）或量化（quantization）来减少缓存，但通常将两者孤立对待。在本文中，我们将 KV 缓存压缩建模为一个率失真（rate-distortion）问题，在此框架下，剔除和量化是同一种比特分配方案的两个极端。这揭示了对两者进行联合优化的必要性，从而提出了我们的方法 RDKV（率失真 KV 缓存压缩）。RDKV 根据压缩对注意力计算造成的失真来推导每个 token 或通道的权重。基于这些权重，它在预填充阶段后执行一次反向注水（reverse water-filling）算法，为每个 token 或通道分配从全精度到零比特不等的位宽。在 LongBench、RULER 和 InfiniteBench 上的实验表明，RDKV 的表现平均优于最佳评估基准 9.1%。在 LongBench 上，它仅需 2.48% 的缓存保留率即可恢复 97.81% 的全缓存准确率。与全缓存 FlashAttention-2 解码相比，在 128K 上下文长度下，它实现了 4.5 倍的解码加速和 1.9 倍的峰值内存减少，同时保持了相当的性能。

## Abstract
Large language models (LLMs) have shown strong performance across diverse tasks, but their inference with long input contexts is bottlenecked by memory size and bandwidth. The Key-Value (KV) cache size grows linearly with sequence length and needs to be re-read from off-chip high-bandwidth memory (HBM) to on-chip memory at every decoding step, resulting in memory-bound inference. Existing methods reduce the cache by either eviction or quantization, but typically treat the two in isolation. In this paper, we cast KV cache compression as a rate-distortion problem, under which eviction and quantization are two end-points of the same bit allocation scheme. This exposes the need to optimize them jointly, motivating our method, RDKV (Rate-Distortion KV cache compression). RDKV derives the weight of each token or channel from the distortion that compression induces on the attention computation. Based on these weights, it assigns each token or channel a bit-width ranging from full precision down to zero bits guided by reverse water-filling, applied once after the prefilling stage. Experiments on LongBench, RULER, and InfiniteBench show that RDKV outperforms the best evaluated baseline by 9.1% on average. On LongBench it recovers 97.81% of full-cache accuracy with only 2.48% cache retention. Compared with full-cache FlashAttention-2 decoding, it achieves 4.5x decode speedup and 1.9x peak memory reduction with 128K context length, while maintaining comparable performance.

#### 1. 核心问题与整体含义（研究动机和背景）
大语言模型（LLMs）在处理长文本时，推理效率受到 **KV 缓存（Key-Value Cache）** 的严重制约。KV 缓存随序列长度线性增长，导致显存占用巨大，且在解码阶段需要频繁从片外高带宽内存（HBM）读取，造成严重的内存受限（Memory-bound）。
现有的压缩技术主要分为两类：**剔除（Eviction）**（丢弃不重要的 token）和 **量化（Quantization）**（降低位宽）。然而，现有方法通常将两者孤立处理。本文的研究动机在于：剔除和量化实际上是比特分配方案的两个极端（剔除即分配 0 比特），应当在一个统一的框架下进行联合优化，以实现最优的率失真（Rate-Distortion）权衡。

#### 2. 方法论：核心思想与关键技术
RDKV 提出将 KV 缓存压缩建模为一个**率失真优化问题**，其核心步骤如下：
*   **失真权重计算（Distortion Weighting）：**
    *   **V 缓存：** 基于 token 级别的注意力权重推导权重 $w_t$，反映剔除该 token 对注意力输出的扰动。
    *   **K 缓存：** 基于通道（Channel）级别的 Q 和 K 列范数乘积推导权重 $w_c$，反映剔除该通道对注意力 Logit 矩阵的影响。
*   **联合比特分配（Joint Bit Allocation）：**
    *   将压缩任务转化为**多重选择背包问题（MCKP）**。
    *   利用**逆向注水算法（Reverse Water-filling）**和拉格朗日松弛法，在给定的总比特预算下，为每个单元动态分配位宽 $\{0, 2, 4, 8, 16\}$ 比特。
    *   预算紧时，更多单元被分配 0 比特（剔除）；预算宽裕时，单元获得更高精度（量化）。
*   **硬件高效的 TriZone 布局：**
    *   为了将混合位宽分配转化为实际的加速，设计了 TriZone 存储布局。
    *   将缓存分为三个区域：Zone A（打包的量化数据）、Zone B（全精度保留数据）、Zone C（新生成的 token）。
    *   通过 Triton 算子将反量化融合进注意力内核，避免了显存解包开销和分支预测失败。

#### 7. 优点
*   **理论严谨：** 首次将 KV 缓存压缩系统性地建模为率失真问题，为剔除和量化的结合提供了坚实的数学基础。
*   **联合优化：** 避免了人为设定剔除比例或量化阈值，通过注水算法自动寻找最优分配。

*   **静态分配限制：** 目前的比特分配是在预填充阶段后一次性完成的（Frozen），无法根据解码过程中注意力模式的动态漂移进行实时调整。
