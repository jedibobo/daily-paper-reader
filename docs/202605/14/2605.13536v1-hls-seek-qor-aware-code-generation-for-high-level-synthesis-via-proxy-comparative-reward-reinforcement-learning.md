## 摘要
高层次综合 (HLS) 将算法级 C/C++ 描述编译为硬件，其结果质量 (QoR)——即延迟和资源利用率——主要受 pragma 配置和代码结构的约束。现有的基于大语言模型 (LLM) 的 HLS 方法侧重于功能正确性的训练，但完全忽略了 QoR。我们观察到，用于 HLS 的强化学习 (RL) 并不需要绝对的综合结果，而仅需要候选方案之间的相对比较。基于这一见解，我们提出了 HLS-Seek，这是一个 QoR 感知的自然语言到 HLS (NL-to-HLS) 框架，它使用比较代理奖励模型取代了昂贵的在环综合 (synthesis-in-the-loop) 强化学习，并实现了 99.53% 的 Pareto 支配准确率。为了防止奖励欺骗 (reward hacking)，我们引入了不确定性感知的蒙特卡洛 (MC) dropout 切换机制，该机制针对低置信度的候选方案选择性地调用真实的 Vitis HLS 综合，并在线更新代理模型，从而构建了一个自我改进的奖励系统。HLS-Seek 仅凭 7B 参数就在 HLS-eval 上实现了 81.5% 的语法正确率 pass@1 和 81.4% 的 Func@5，超越了 GPT-5.1 和其他前沿模型，同时训练速度比真实奖励强化学习快 8.5 倍。在 QoR 评估中，HLS-Seek 在 16/30 个内核上实现了最低延迟，并在 9 个内核上 Pareto 支配了 HLS 专用基准模型。

## Abstract
High-Level Synthesis (HLS) compiles algorithmic C/C++ descriptions into hardware, with Quality of Results (QoR) -- latency and resource utilization -- critically governed by pragma configurations and code structure. Existing LLM-based HLS approaches train for functional correctness but ignore QoR entirely. We observe that reinforcement learning (RL) for HLS does not require absolute synthesis results -- only relative comparisons between candidates. Based on this insight, we propose \textbf{HLS-Seek}, a QoR-aware NL-to-HLS framework that replaces expensive synthesis-in-the-loop RL with a comparative proxy reward model achieving 99.53\% Pareto-dominance accuracy. To prevent reward hacking, we introduce \textit{uncertainty-aware Monte Carlo (MC) dropout switching} that selectively invokes real Vitis HLS synthesis for low-confidence candidates and online updates the proxy, creating a self-improving reward system. HLS-Seek achieves 81.5\% syntax correctness pass@1 and 81.4\% Func@5 on HLS-eval with only 7B parameters, surpassing GPT-5.1 and other frontier models while achieving 8.5$\times$ faster training than real-reward RL. On QoR evaluation, HLS-Seek achieves the lowest latency on 16/30 kernels and Pareto-dominates HLS-specific baselines on 9 kernels.

高层次综合（HLS）将 C/C++ 代码转换为硬件描述语言，其核心挑战在于优化**结果质量（QoR）**，即在满足功能正确性的前提下，最小化延迟并优化资源利用率（LUT、DSP 等）。
*   **研究动机**：现有的基于大语言模型（LLM）的 HLS 生成方法主要关注功能正确性，对 QoR 缺乏感知。
*   **核心瓶颈**：在强化学习（RL）中直接使用 EDA 工具（如 Vitis HLS）作为奖励反馈极其昂贵，单次综合需数分钟至数小时，导致训练效率低下。
*   **核心见解**：RL 训练不需要绝对的 QoR 数值，只需要候选代码之间的**相对优劣关系**。

#### 2. 方法论：核心思想与关键技术
HLS-Seek 提出了一个三阶段训练框架，核心是**代理比较奖励模型**。
*   **核心思想**：利用轻量级代理模型预测两段代码的 QoR 优劣，并引入不确定性感知机制，在模型“没把握”时调用真实综合工具。
*   **关键技术细节**：
    1.  **三阶段训练流程**：
        *   **多样性 SFT**：在广泛的 HLS 语料库上训练，使模型熟悉各种 pragma 配置。
        *   **冷启动 SFT**：使用帕累托近优（Pareto-Proximal）数据和思维链（CoT）推理轨迹进行微调，赋予模型硬件推理能力。
        *   **GRPO 强化学习**：使用组相对策略优化，结合格式、编译、功能和 QoR 四部分奖励。
    2.  **代理比较奖励模型**：采用 Siamese（孪生）架构，通过比较两段代码的帕累托支配关系或延迟高低来给出奖励。
    3.  **不确定性感知切换（MC Dropout）**：通过 10 次随机前向传播计算方差。若方差超过阈值（不确定性高），则触发真实 Vitis HLS 综合，并将结果用于在线更新代理模型，防止“奖励欺骗”。
