## 摘要
为神经架构搜索 (NAS) 开发有效的代理模型（性能预测器）通常需要昂贵的微调或复杂的表示工程。我们提出了一种低成本的嵌入策略，利用语言模型 (LM) 的归纳偏置来消除这些开销。通过将架构表示为 PyTorch 类定义文本，我们证明了现成的语言模型在无需针对 NAS 进行专门微调的情况下，即可作为具有竞争力的特征提取器。最终的预测器是通过将提取的面向代码的语言模型嵌入 (COLE) 传递给一个轻量级回归头构建而成的。我们还研究了提高嵌入质量和利用率的策略。我们在 NAS-Bench-201 和 einspace 搜索空间上的实验表明，在使用冻结的语言模型时，原始代码输入比其他基于文本的编码（例如 ONNX 转文本编码）具有更高的预测性能。我们还观察到，在 NAS-Bench-201 中，COLE 使用 BANANAS 算法驱动了更优的代理辅助搜索。在针对 CIFAR-100 性能进行优化时，使用 COLE 代替结构路径编码进行架构表示，可使达到搜索空间内最佳架构 1% 误差范围内（按测试准确率计）所需的评估预算减少 34%。由于任何神经架构都可以表示为代码，这些发现使 COLE 成为推进 NAS 的通用且高效的基础。

## Abstract
Developing effective surrogates (performance predictors) for Neural Architecture Search (NAS) typically requires expensive fine-tuning or the engineering of complex representations. We propose a low-cost embedding strategy that leverages the inductive bias of Language Models (LMs) to eliminate these overheads. By representing architectures as PyTorch class definition text, we demonstrate that off-the-shelf LMs act as competitive feature extractors without NAS-specialized fine-tuning. The final predictor is constructed by passing the extracted Code-Oriented LM Embeddings (COLE) through a lightweight regression head. We also investigate strategies to improve embedding quality and utilization. Our experiments on the NAS-Bench-201 and einspace search spaces reveal that raw code inputs yield higher predictive performance than other text-based encodings (e.g., ONNX-to-text encodings) when using frozen LMs. We also observe COLE drives superior surrogate-assisted search using the BANANAS algorithm in NAS-Bench-201. When optimizing for CIFAR-100 performance, replacing structural path encodings with COLE for architecture representation allows for a 34% decrease in the evaluation budget required to reach within 1% of the fittest architecture in the search space (by test accuracy). As any neural architecture can be represented as code, these findings establish COLE as a versatile and efficient foundation for advancing NAS.

### 2. 方法论
*   **核心思想**：将神经架构转化为标准的 PyTorch 类定义文本，输入到冻结的语言模型中提取嵌入向量，再通过轻量级回归头预测性能。
*   **关键技术细节**：
    *   **代码字符串化 (Native Code Stringification)**：开发了确定性的转换工具，将 NAS-Bench-201 的拓扑字符串或 einspace 的派生树转换为可执行的 PyTorch 代码（`torch.nn.Module`）。
    *   **冻结嵌入提取**：使用预训练的语言模型（如 CodeLlama）作为特征提取器，参数完全冻结。采用**标记均值池化 (Mean Pooling)** 处理最后一层的隐藏状态，生成高维嵌入。
    *   **代理模型头 (Surrogate Head)**：
        *   **降维**：使用主成分分析 (PCA) 将高维嵌入（如 4096 维）降至 128 维，以克服维度灾难。
        *   **回归器**：一个简单的三层多层感知机 (MLP)。
    *   **损失函数**：采用 **Pairwise Hinge Loss**，重点优化肯德尔秩相关系数（Kendall’s Tau），因为在 NAS 中，架构的相对排名比绝对准确率预测更重要。

### 6. 主要结论与发现
*   **代码是最佳表示**：在冻结 LM 的情况下，原始 PyTorch 代码的表现优于 ONNX 或派生树等专门设计的文本编码，因为代码与 LM 的预训练分布最一致。
*   **搜索效率显著提升**：在 CIFAR-100 任务中，使用 COLE 替代路径编码，使达到最优性能 1% 误差范围内所需的评估预算**减少了 34%**。
*   **PCA 至关重要**：在高维嵌入空间中，PCA 降维能显著提升回归头在小样本数据下的泛化能力。
*   **模型规模并非唯一因素**：较小的模型（如 Yi-Coder 1.5B）在某些情况下能提供与 7B 或 34B 模型相当的性能。

### 7. 优点
*   **通用性强**：任何可以写成代码的架构都可以使用 COLE，无需为新搜索空间重新设计复杂的矩阵编码。
*   **冷启动友好**：无需针对特定任务微调 LM，即可获得高质量的架构特征。
*   **语义丰富**：代码能同时捕捉拓扑结构和超参数（如卷积核大小、填充等），而传统编码往往难以兼顾。

### 8. 不足与局限
*   **推理开销**：虽然不需要微调，但使用 7B 参数模型提取嵌入的推理成本仍远高于传统的图算法或路径编码。
*   **搜索空间覆盖**：实验主要集中在 NAS-Bench-201 和 einspace，尚未在 DARTS 等更大、更复杂的搜索空间上进行大规模验证。
*   **依赖预训练质量**：COLE 的效果高度依赖于底层 LM 对代码（尤其是 PyTorch 语法）的理解能力，对于极度冷门的框架或自定义算子，效果可能会打折扣。
