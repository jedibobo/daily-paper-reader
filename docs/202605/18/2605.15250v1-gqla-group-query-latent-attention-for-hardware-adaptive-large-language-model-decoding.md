## 摘要
DeepSeek-V2/V3 中使用的多头潜变量注意力（MLA）将键（Key）和值（Value）共同压缩进一个低秩潜变量中，几乎完美地匹配了 H100 的算力上限（roofline）。然而，其训练后的权重仅暴露了一条解码路径——即吸收后的 MQA 形式——这使得高效推理受限于 H100 级别的计算带宽比，丧失了沿头轴（head axis）的张量并行性，并且在受出口限制的 H20 等商用推理 GPU 上无法获得多标记预测（MTP）带来的增益。我们提出了分组查询潜变量注意力（GQLA），这是对 MLA 的一种极简修改，其训练权重在相同参数下暴露了两条代数等价的解码路径：一条是与 MLA 相同的 MQA 吸收路径，另一条是具有每组扩展缓存的 GQA 路径。运行时会根据目标硬件选择匹配的路径，无需重新训练，也无需自定义算子，因此一套 GQLA 权重即可同时达到 H100（MQA 吸收路径，s_q=1）和 H20（GQA + MTP 路径，s_q=2）的性能上限，同时在 GQA 路径上支持高达 8 路的零冗余张量并行。为了避免从头开始预训练，我们将 TransMLA 扩展为 TransGQLA，可将预训练的 GQA 检查点转换为 GQLA 模型；在 LLaMA-3-8B 上，它在 MQA 吸收路径上将每个标记的 KV 缓存压缩至 GQA 基线的 28.125%，同时在每组路径上结构性地保留了 GQA 级别的流量。

## Abstract
Multi-head Latent Attention (MLA), the attention used in DeepSeek-V2/V3, jointly compresses keys and values into a low-rank latent and matches the H100 roofline almost perfectly. Its trained weights, however, expose only one decoding path - an absorbed MQA form - which ties efficient inference to H100-class compute-bandwidth ratios, forfeits tensor parallelism along the head axis, and yields no Multi-Token Prediction (MTP) gain on commodity inference GPUs such as the export-restricted H20. We propose Group-Query Latent Attention (GQLA), a minimal modification of MLA whose trained weights expose two algebraically equivalent decoding paths over the same parameters: an MQA-absorb path identical to MLA's, and a GQA path with a per-group expanded cache. The runtime picks the path that matches the target hardware - no retraining, no custom kernels - so a single set of GQLA weights pins the rooflines of both H100 (MQA-absorb, s_q=1) and H20 (GQA + MTP, s_q=2), while supporting up to 8-way zero-redundancy tensor parallelism on the GQA path. To avoid pretraining from scratch we extend TransMLA into TransGQLA, which converts a pretrained GQA checkpoint into a GQLA model; on LLaMA-3-8B it compresses the per-token KV cache to 28.125% of the GQA baseline on the MQA-absorb path while structurally preserving GQA-level traffic on the per-group path.

### 1. 核心问题与整体含义
*   **研究背景**：现代大语言模型（LLM）推理的瓶颈在于 KV Cache 的内存带宽。DeepSeek 提出的 MLA 通过低秩压缩极大地减少了 KV 缓存，在 H100 GPU 上表现近乎完美。
*   **核心痛点**：MLA 的设计与 H100 的高计算带宽比（Compute-to-Bandwidth Ratio）深度绑定。在算力受限的硬件（如 H20）上，MLA 强制使用的“MQA 吸收路径”会导致计算受限（Compute-bound），无法发挥硬件带宽优势，且不支持头维度的张量并行（TP），导致多 Token 预测（MTP）增益消失。
*   **研究动机**：开发一种既能保留 MLA 的高压缩率，又能根据部署硬件动态调整计算模式的注意力机制。

### 2. 方法论
*   **核心思想**：对 MLA 进行极小修改，将原本在所有 Query 头上复制的上投影（Up-projections）改为按“组（Group）”索引。这使得同一套训练好的权重可以暴露两条代数等价的解码路径。
*   **关键技术细节**：
    *   **MQA 吸收路径（MQA-absorb Path）**：与 MLA 相同，将上投影吸收到 Query 和 Output 投影中，直接对潜变量（Latent）进行计算。KV 缓存极小，适合 H100 等高算力设备。
    *   **GQA 路径（GQA Path）**：将潜变量展开为每组的 Key 和 Value，运行标准的 GQA。虽然 KV 缓存略大，但计算强度降低，适合 H20 等算力较低、带宽较高的设备，且天然支持头轴张量并行。
    *   **TransGQLA 转换技术**：扩展了 TransMLA 流程，通过“头合并（Head-merging）”、“RoPE 解耦（RoRoPE）”和“频率折叠（FreqFold）”等步骤，将现有的 GQA 模型（如 LLaMA-3）直接转换为 GQLA 结构，无需从头预训练。
    *   **稀疏扩展**：GQLA 的 Query/KV 比例（通常为 16）完美匹配 Tensor Core 的 MMA 指令块大小，使其在稀疏注意力场景下比 MLA 更高效。
