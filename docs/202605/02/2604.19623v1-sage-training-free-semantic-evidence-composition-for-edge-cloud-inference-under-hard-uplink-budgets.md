## 摘要
边缘-云混合推理将困难的输入卸载到强大的远程模型，但上行链路通道对每个请求可传输的比特数施加了硬性约束。我们表明，仅基于注意力重要性（协作推理中的标准方法）来选择传输内容，在硬预算下具有固有的局限性。两项发现支持了这一主张：首先，用低重要性但具有互补性的单元替换高重要性单元可以提高服务器的准确率，这表明关键不在于单个单元的重要性，而在于传输集合对输入不同方面的覆盖程度；其次，在没有任何内容信息的情况下，空间均匀选择在适度预算下也能达到具有竞争力的准确率，这证实了空间覆盖本身就具有独立价值。基于这一分析，我们提出了 SAGE（语义注意力引导证据），这是一种有原则的、无需训练的方法，它将重要性过滤与嵌入多样性采样相结合。在 ImageNet-1K 数据集上，SAGE 在传输不到一半可用证据单元的情况下，达到了服务器卸载准确率上限的 93%，显著优于仅基于重要性的组合方式。

## Abstract
Edge-cloud hybrid inference offloads difficult inputs to a powerful remote model, but the uplink channel imposes hard per-request constraints on the number of bits that can be transmitted. We show that selecting transmitted content based solely on attention-based importance, the standard approach in collaborative inference, is inherently limited under hard budgets. Two findings support this claim. First, replacing high-importance units with low-importance but complementary ones improves server accuracy. This shows that what matters is not individual importance but how well the transmitted set covers diverse aspects of the input. Second, spatially uniform selection without any content information achieves competitive accuracy at moderate budgets. This confirms that spatial coverage alone carries independent value. Based on this analysis, we propose SAGE (Semantic Attention-Guided Evidence), a principled, training-free method that combines importance filtering with embedding-diversity sampling. SAGE achieves 93% of the server ceiling in offloaded accuracy while transmitting fewer than half of the available evidence units on ImageNet-1K, substantially outperforming importance-only composition.

#### 1. 核心问题与整体含义（研究动机和背景）
在边缘-云协同推理系统中，边缘设备通常将难以处理的输入（Hard Inputs）卸载到强大的远程服务器。然而，实际部署中面临**硬性上行链路预算（Hard Uplink Budgets）**的限制，即每个请求可传输的比特数受带宽、延迟和能耗的严格约束。
*   **研究动机**：现有的主流方法（如基于注意力的 Patch 选择）主要优化平均通信成本，但在处理真正需要卸载的“困难样本”时，往往会选择过多的 Patch 从而超出预算。
*   **核心发现**：论文指出，仅依赖“重要性”（Attention Score）来选择传输内容具有局限性。高重要性的单元往往在空间和语义上高度冗余（聚集在同一区域），而忽略了对输入图像全局信息的“覆盖度”（Coverage）。实验证明，互补的低重要性单元有时比冗余的高重要性单元对服务器准确率的贡献更大。

#### 2. 论文提出的方法论：SAGE
SAGE（Semantic Attention-Guided Evidence）是一种**无需训练（Training-free）**的算法，旨在通过平衡“重要性”与“多样性”来优化证据组合。其核心流程如下：
*   **核心思想**：先通过注意力机制过滤出候选集，再通过嵌入向量的多样性采样确保信息覆盖。
*   **关键技术步骤**：
    1.  **注意力预过滤（Attention Prefilter）**：从原始 $N$ 个 Patch 中，根据边缘模型的注意力得分选出前 $2B$ 个候选单元（$B$ 为传输预算）。这确保了候选单元都具有一定的任务相关性。
    2.  **种子选择**：将候选集中注意力得分最高的 Patch 作为第一个选定单元。
    3.  **迭代最远点采样（Iterative FPS）**：在剩余候选集中，利用 Patch 的嵌入向量（Embedding）计算余弦相似度。每次迭代选择一个与已选集合中“最大相似度最小”的单元（即最不相似、最互补的单元），直到达到预算 $B$。
*   **算法特点**：完全利用预训练 Vision Transformer (ViT) 的内部特征，无需修改模型参数或进行任何微调。

#### 3. 实验设计
*   **数据集**：ImageNet-1K 验证集（50,000 张图像）。
*   **场景**：边缘设备使用轻量级模型（DeiT-Tiny），服务器使用强大模型（DeiT-Base）。
*   **Benchmark 与对比方法**：
    *   **Random**：随机选择。
    *   **Uniform Grid**：固定空间网格采样（验证空间覆盖的独立价值）。
    *   **Attention Prefix**：仅按注意力得分排序选择（现有主流方法）。
    *   **ToMe / BAT**：来自计算加速领域的 Token 合并与多样性剪枝方法。
    *   **Full Transmission**：传输所有 196 个 Patch（性能上限）。
*   **评估指标**：卸载准确率（Offloaded Accuracy）、系统整体准确率（Overall Accuracy）、空间覆盖率。

#### 4. 资源与算力
*   **训练算力**：**无需训练**。该方法直接应用于冻结的预训练模型。
*   **推理设备**：

    *   服务器端：NVIDIA T4 GPU。
*   **计算开销**：SAGE 的选择过程复杂度极低（约 3.5 M FLOPs），仅占边缘模型前向传播开销的 **<0.3%**，在端到端延迟中几乎可以忽略不计。

#### 5. 实验数量与充分性
*   **实验规模**：在 ImageNet-1K 全量验证集上进行了测试，涵盖了从极低预算（B=16）到高预算（B=128）的多个梯度。
*   **消融实验**：
    *   测试了不同的预过滤比例（$k \times B$）。
    *   分析了不同注意力熵（Attention Entropy）下的表现，证明 SAGE 在注意力分散的困难样本上提升最明显。
*   **泛化性验证**：更换了 4 组不同的客户端-服务器骨干网络组合（包括 DeiT-Small, ViT-Base, CaiT 等），均表现出一致的性能提升。
*   **客观性**：对比了多种不同类型的基准线，并考虑了不同的卸载阈值（$\eta$），实验设计较为全面、公平。

#### 6. 主要结论与发现
*   **覆盖度至关重要**：在硬预算下，空间覆盖度具有独立价值。即使是盲目的均匀网格采样，在某些预算下也能接近基于注意力的选择方法。
*   **SAGE 的优越性**：在 ImageNet-1K 上，SAGE 仅需传输不到一半的 Patch（B=96），即可达到服务器性能上限的 93%。在低预算（B=32/48）下，比仅靠注意力的方案准确率高出 **2-4 个百分点**。
*   **部署友好性**：在 NB-IoT、LTE-M 等低带宽通道下，减少传输量是降低延迟的唯一有效手段，而 SAGE 在保证准确率的同时极大地压缩了传输需求。

#### 7. 优点
*   **无需训练**：即插即用，兼容现有的预训练 ViT 模型，部署成本极低。
*   **理论与直觉结合**：通过服务器注意力分析，有力地证明了“重要性不等于价值”的观点，为语义通信提供了新视角。
*   **极低开销**：算法逻辑简单，对边缘设备的计算负担微乎其微。
*   **鲁棒性强**：跨模型架构、跨预算、跨卸载阈值均表现稳定。

#### 8. 不足与局限
*   **任务局限性**：目前仅在图像分类任务上进行了验证，尚未探讨在目标检测、分割或视频任务中的表现。
*   **服务器端优化缺失**：服务器模型是冻结的，未针对接收到的稀疏/非连续 Patch 进行专门的适应性优化（如掩码建模微调）。
*   **静态预算**：虽然讨论了硬预算，但未实现根据瞬时信道质量动态调整 $B$ 的闭环控制。
