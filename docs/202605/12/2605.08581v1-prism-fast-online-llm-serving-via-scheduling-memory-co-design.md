## 摘要
现代在线大语言模型（LLM）服务（如检索增强生成 RAG 和智能体系统）日益呈现出两个显著特征：提示词分段（如系统指令、检索到的段落、工具输出）和热点倾斜，即一小部分分段在用户请求中频繁重复出现。若未能协同利用这些模式，可能会导致热点分段的重复预填充（prefill）并延长首字延迟（TTFT），从而损害吞吐量和用户感知的响应速度。然而，现有工作独立地处理这些模式：KV 缓存管理主要利用分段重用，而调度则通过重排请求来提高缓存局部性，但两者都未能将请求准入与 KV 缓存保留对齐。为了填补这一空白，我们首先分析了调度和 KV 缓存管理如何共同影响 TTFT。在此指导下，我们提出了 PRISM（前缀重用优化集成调度与内存），它协同设计了查询感知调度器（QAS）和需求感知基数树（DART），以使请求准入与精确前缀 KV 保留保持一致。评估结果显示，与最强基准相比，PRISM 在 4B 和 13B 模型上分别将每 QPS 的平均 P99 TTFT 降低了 23.3% 和 37.1%，同时将精确前缀 KV 缓存命中率分别提高了 5.9 和 12.2 个百分点。

## Abstract
Modern online large language model (LLM) services, such as Retrieval-Augmented Generation (RAG) and agent systems, increasingly expose two prominent characteristics: prompt segmentation (e.g., system instructions, retrieved passages, tool outputs) and hotspot skew, where a small set of these segments recurs frequently across user requests. Failing to jointly exploit these patterns could lead to repeated prefill of hot segments and prolonged TTFT, undermining both throughput and user-perceived responsiveness. However, existing work tackles these patterns independently: KV-cache management mainly exploits segment reuse while scheduling reorders requests to improve cache locality, yet neither aligns request admission with KV-cache retention. To address this gap, we first analyze how scheduling and KV-cache management jointly affect TTFT. Guided by this, we present PRISM (Prefix Reuse Optimization Integrated Scheduling and Memory), which co-designs a query-aware scheduler (QAS) with a demand-aware radix tree (DART) to align request admission with exact-prefix KV retention. Our evaluation results show that, versus the strongest baseline, PRISM reduces average per-QPS P99 TTFT by 23.3\% and 37.1\% while increasing exact-prefix KV-cache hit rate by 5.9 and 12.2 percentage points on 4B and 13B models, respectively.

### 论文总结：PRISM —— 通过调度-内存协同设计实现快速在线 LLM 推理服务

#### 1. 核心问题与整体含义（研究动机和背景）
现代在线大语言模型（LLM）服务（如 RAG 检索增强生成、智能体 Agent 系统）具有两个核心特征：
*   **提示词分段（Prompt Segmentation）：** 提示词由系统指令、检索文档、工具输出等可重用的片段组成。
*   **热点倾斜（Hotspot Skew）：** 少数特定片段在短时间内会被大量不同请求频繁重复调用。

**研究动机：** 现有的优化手段通常是孤立的。调度系统（Scheduling）负责重排请求以提高局部性，而内存管理（KV-cache Management）负责缓存重用。然而，两者缺乏协同：调度器知道哪些请求即将到来，但内存管理器在压力下可能会驱逐这些即将被使用的热点片段，导致昂贵的重复预填充（Prefill）计算，进而显著增加首字延迟（TTFT）。

#### 2. 方法论：PRISM 核心思想与关键技术
PRISM 提出了**调度与内存的协同设计**，主要包含两个组件：
*   **查询感知调度器 (QAS, Query-Aware Scheduler)：**
    *   **状态跟踪：** 实时维护片段的全局计数（队列中）、活跃计数（正在处理）和预测计数（下一批次）。
    *   **桶签名分组：** 根据片段优先级生成顺序敏感的签名，将具有相同热点前缀的请求分入同一个“桶”。
    *   **双车道调度：** 采用“热车道（Hot Lane）”优先处理高分桶以提升局部性，同时保留“冷车道（Cold Lane）”以 FIFO 方式处理普通请求，防止长尾请求饥饿。
*   **需求感知基数树 (DART, Demand-Aware Radix Tree)：**
    *   **锚点机制：** 在基数树中为可重用片段设置锚点（Anchors），并记录调度器传递的优先级元数据。
    *   **需求感知驱逐策略：** 当显存不足时，DART 不再简单使用 LRU（最近最少使用），而是优先保留调度器标记为“即将被使用”或“高频使用”的片段，优先驱逐私有后缀和低优先级片段。

#### 3. 实验设计
*   **场景与数据集：**
    *   **RAG 场景：** 基于 MultiHopRAG 构造，模拟具有 Zipf 分布热点特征的在线检索请求。
    *   **Agent 场景：** 基于 $\tau$-bench 构造，包含系统策略、工具定义和多轮对话上下文。
*   **对比基准 (Baselines)：**

    *   **ContextPilot：** 目前最先进的上下文重用服务系统。
*   **评估指标：** TTFT 的各分位数（P50/P90/P95/P99）、KV 缓存命中率、答案准确度（F1 Score）。

#### 4. 资源与算力
*   **硬件环境：** 实验在一台拥有 **14 vCPU** 和 **单张 NVIDIA A800 (80GB)** 显卡的服务器上完成。

*   **训练/运行：** 论文侧重于推理服务（Serving），未涉及模型训练。实验涵盖了不同 QPS（每秒请求数）负载下的长时间在线模拟运行。

#### 5. 实验数量与充分性
*   **实验规模：** 论文进行了多维度的测试，包括：
    *   不同模型（Qwen3-4B, Llama2-13B）的横向对比。
    *   不同负载（QPS 从低到饱和）的压力测试。
    *   **消融实验：** 验证了 DART 驱逐策略的有效性以及冷车道配额对公平性的影响。
    *   **敏感性分析：** 改变热点比例（$r_{hot}$）、Zipf 指数（$\alpha$）以及检索 Top-k 数量。
*   **充分性评价：** 实验设计较为全面，涵盖了从性能指标到鲁棒性分析的各个方面，对比实验在相同的硬件和引擎基准下进行，具有较高的客观性和公平性。

#### 6. 主要结论与发现
*   **延迟显著降低：** 在 4B 模型上，P99 TTFT 平均降低了 **23.3%**；在 13B 模型上降低了 **37.1%**。
*   **缓存效率提升：** 精确前缀 KV 缓存命中率在 4B 和 13B 模型上分别提升了 **5.9** 和 **12.2** 个百分点。
*   **系统稳定性：** PRISM 扩展了系统的“服务膝点（Service Knee）”，即在更高负载下仍能保持较低的延迟，提升了系统的最大可持续吞吐量。
*   **质量保持：** 在大幅降低延迟的同时，模型的回答质量（F1 分数）保持稳定甚至略有提升。

#### 7. 优点
*   **协同设计思路：** 突破了调度与内存管理的界限，通过轻量级的元数据交换解决了信息不对称问题。
*   **实际应用导向：** 针对 RAG 和 Agent 这类长前缀、高重用场景有极强的针对性。
*   **低开销：** 算法复杂度较低，控制平面的开销远小于模型推理本身的计算量。

#### 8. 不足与局限
*   **硬件覆盖有限：** 实验仅限于单 GPU 环境，未探讨在多 GPU 张量并行或流水线并行下的 KV 缓存一致性与调度挑战。
*   **精确匹配依赖：** DART 依赖于 Token 级别的精确前缀匹配，对于语义相似但 Token 不同的片段（如改写后的文档）无法直接重用。
*   **静态分段假设：** 实验假设片段边界是明确且可识别的，在处理完全非结构化的动态长文本时，其分段效果可能受限。
