# HeadQ: Model-Visible Distortion and a Prototype KV Quantizer-Corrector Codec 

# Jorge L. Ruiz Williams Independent Researcher jorgeruizwilliams@gmail.com May 8, 2026 

Abstract 

KV-cache quantizers usually optimize storage-space reconstruction, even though attention reads keys through logits and values through attention-weighted readout. We argue that persistent cache error should be measured in model-visible coordinates and instantiate that view as a prototype quantizer-corrector codec. For keys, the visible object is score error modulo constant shifts; this yields HeadQ, a key-side method that stores a low-rank residual side code in a calibration-learned query basis and applies it as an additive logit correction. For values, fixed-attention readout gives an A2-weighted token-distortion surrogate that supplies the value-side policy. Across six models, Fisher/score-space error predicts at-tention KL far better than raw key MSE; same-budget counterexamples, null-space interventions, query-PCA controls, cache-only matched-MSE K/V perturbation panels, and wrong-sign HeadQ falsify storage-MSE al-ternatives. Matched Pythia checkpoints localize the main anomaly to a small-model low-entropy route-flip boundary; under the stricter cache-only downstream control, centered block-FP4 and per-channel affine re-main positive while plain block-FP4 is the main failure case. In the main paper setting, where keys are quantized and values remain dense, HeadQ removes roughly 84 –94% of the excess perplexity on the strongest 2-bit rows. In the complementary full-KV codec composition, the same key corrector plus the A2 value policy improves all six 2-bit rows, establishing a concrete path from model-visible geometry to a full-KV codec; packed kernels and latency are left for systems work.

Storage MSE, by contrast, pays for all three pieces: the active coefficients, the token-shared shift, and the orthogonal residual. This decomposition explains why a fixed grid can waste resolution on key components that are large in storage coordinates but invisible to dense attention. Mean-key removal is one simple example of choosing a better representative. The broader lesson is practical: key quantization should preserve score-visible coordinates, not just raw Euclidean reconstruction. The decomposition gives three empirical predictions: key MSE should be a weak behavioral metric, score-space error should rank attention damage better, and non-equivariant quantizers should fail by converting benign key components into active score error. It also gives a method: preserve a compact approxima-tion to the score-visible residual and apply it where scores are computed.

# 5 Empirical Results 

The experiments move from metric prediction to deployed behavior. Score-space metrics should rank attention damage, softmax-null interventions should expose storage-coordinate failure, query-PCA should approximate the active residual without future queries, and HeadQ should improve downstream decode while composing with the value-side A2 objective from Section 3.2. # 5.2 Centering-Sensitive Quantizer Failure 

The theorem predicts a concrete failure mode for low-bit grids. A storage-coordinate quantizer can fail in two steps: a token-shared common compo-nent changes the quantization grid, and that changed grid inflates error in the one-dimensional active coefficients that control scores. This separates a harm-less reparameterization from a quantization failure: exact attention cancels the common component, but the quantizer may not. We isolate this with an α

intervention, 

Kα = ∆ K + α1¯k⊤, (23) that holds the residual ∆K fixed while restoring the token-shared mean com-ponent ¯k. Exact attention is unchanged for every α, but fixed-grid and group quantizers are not translation-equivariant. The α = 1 slice is easiest to read visually: Pythia is the sharpest example, but Qwen2, Qwen2.5, and TinyLlama show the same direction at smaller scale. Appendix Table 4 retains the full Fisher/KL control sweep. The intervention separates coordinate description from quantized behavior. The dense model is unchanged; only the quantized pipeline changes. KIVI-like per-channel keys avoid this null-range waste by construction [9], and KVQuant-style centering removes the offending component before a fixed grid sees it [10]. The common key component is not Fisher-active by itself; damage appears only when a non-equivariant quantizer lets invisible storage energy set the active score grid.

# 6 Discussion 

The design lesson is operator matching, and the resulting system object is a quantizer-corrector codec rather than a single storage rule. Keys are score ob-jects, so good key compression avoids softmax-null range waste, protects query-visible residuals, and applies corrections where scores are computed. Values are readout objects, so the natural first surrogate is A2-weighted token error. This view helps explain why KIVI-like per-channel keys, KVQuant-style centering, key-favored precision schedules, and value readout weighting can all help for different reasons [9, 10, 15]. The tests cover different failure modes of storage MSE. Fisher-versus-MSE predicts attention KL; the paired counterexample reverses the storage rank-ing; the softmax-null intervention changes quantized behavior without changing dense attention; the cache-only same-MSE panels show directly that one-token downstream damage tracks visible K/V burden; query-PCA makes the active residual deployable; and the wrong-sign control verifies the direction of the de-ployed patch. The Pythia anomaly then becomes a boundary case: ultra-sharp attention rows under rotated/product 3-bit perturbations enter a finite route-flip regime, while matched 410M and 1B checkpoints recover the local ranking. 

# 7 Relation to KV-Cache Quantization Work 

KV-cache quantization methods such as KIVI [9], KVQuant [10], GEAR [11], QuaRot-style rotations [13], AXELRAM [21], RotateKV [22], PolyKV [23], LOOKAT [24], and SpinQuant [25] reduce cache memory through practical serv-14 ing designs. Token-importance methods such as H2O [12] choose which cache entries to retain or protect. Generic vector quantization and rotation meth-ods, including TurboQuant-style approaches [14] and product quantization [26], optimize storage-centered objectives such as MSE or product-code distortion. The closest papers are asymmetry-oriented. Quantize What Counts [15] ar-gues for key-favored precision using spectral and Frobenius structure; AsymKV [16] and KVTuner [17] use layer-wise sensitivity. Our claim is narrower: key sensitivity should be measured in query-conditioned behavioral geometry, not only by layer-level or norm-level statistics. KIVI [9] is conceptually important because it discovers a granularity asym-metry: per-channel keys and per-token values. Our fixed-query view gives one key-side reason such asymmetries can arise. TriAttention [18] is especially rele-vant geometrically: it uses pre-RoPE Q/K centers to estimate key importance for long-reasoning KV compression, whereas our object is a post-RoPE score-space mean-key component whose logit contribution is constant across positions and therefore softmax-inert. KIVI-like per-channel K repairs bad key coordi-nates; KVQuant-style centering avoids softmax-null range waste; TurboQuant-style MSE can be strong for value-side readout reconstruction while missing key-side quotient geometry. The central distinction is storage-coordinate versus model-visible distortion. 

# 8 Limitations 

The paper is mechanistic and studies a prototype codec, not a serving system. Oracle active-axis probes are diagnostic upper bounds; the main downstream PPL rows quantize keys and keep values dense, so they isolate the key-side corrector; the cache-only same-MSE panels are one-token causal checks rather than sequence-level quality metrics; and the full-KV composition is unoptimized and includes side-code overhead before any packed metadata design. We do not study packed kernels, latency, or throughput. Two approximation limits are especially important. First, the A2 value surrogate drops the cross-token remainder in Eq. 13; correlated value residuals can therefore make the surrogate understate exact readout error. We treat it as an operator-matched ranking proxy backed by exact-output and cache-only causal checks, not as a universal equality. Second, the deployed query-PCA basis is static within a run. Although the downstream protocol partly reduces shift by fitting the basis from same-window dense prefill queries, long-context or out-of-domain query drift can still leave the learned basis stale; adaptive basis refresh is out of scope here. The Fisher/KL proxy is likewise local by design. The matched Pythia route-flip slice is the paper’s explicit counterexample to global validity: once finite per-turbations move probability mass between competing top tokens, the quadratic approximation can fail even though exact KL remains the correct behavioral target. The model panel uses short real-corpus windows and six models up to Mistral 15 7B, below the 7B–70B frontier of recent systems papers [15, 16, 17]. The six-model KL-ranking table is a point-estimate summary rather than a confidence-interval result, and not every auxiliary probe spans exactly the same model set. The matched Pythia probes explain one verified family-specific route-flip boundary rather than a universal scaling law. The deployed wrong-sign control tests correction direction but does not ex-haust all same-budget side-code alternatives. Random side bases can be model-and window-dependent, so we use them only as attention-level controls. Query quantization also remains out of scope: queries are not persistent cache entries, and we do not show that the key-side chart transfers cleanly to Q. 

# 9 Conclusion 

KV-cache distortion should be defined in the coordinates the model reads. For keys, that is score error modulo constant shifts; for values, it is attention-weighted readout error with an A2 diagonal surrogate. This yields a prototype KV quantizer-corrector codec: low-bit base K/V storage, a HeadQ key-side corrector, and a value-side A2 policy. The evidence supports that operator-matched view. Score-space metrics beat raw MSE, paired controls expose storage-coordinate failures, query-PCA makes the useful residual deployable, and full-KV composition shows that the whole codec can be assembled without reverting to storage MSE as the default target. The main rigorous evidence is strongest on the key-side corrector; the codec result shows how that correction composes with the value objective.
