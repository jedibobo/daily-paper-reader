## 摘要
具有存内计算功能的高带宽内存（HBM-PIM）通过直接在内存内部执行计算，为减少数据搬运提供了契机，但目前的商业平台仅提供有限的指令集，且需要专用的软件栈。在这项工作中，我们以 RISC-V 附加矩阵扩展（AME）作为语义参考，研究了 HBM-PIM 是否可以作为指令集架构（ISA）级矩阵加速的后端。我们提出了一种基于 PEP 的执行模型，将 AME 逐元素指令和矩阵指令映射到内存操作中的 HBM-PIM 微内核和数据指令。与现有的 HBM-PIM 不同，我们引入了一种无归约的外积数据流，尽管缺乏原生归约支持，但仍能在内存中完全实现累加。我们的方法支持在 PIM 模式下端到端执行逐元素操作、GEMV 和 GEMM，从而最大限度地减少了主机参与和片外传输。在 Samsung Aquabolt-XL 上的实验评估表明，AME 矩阵分块乘法在单个 HBM 伪通道上可达到高达 14.9 GFLOP/s（59.4 FLOP/周期）的性能。

## Abstract
High Bandwidth Memory with Processing-in-Memory (HBM-PIM) offers an opportunity to reduce data movement by executing computation directly inside memory, but current commercial platforms expose limited instruction sets and require specialized software stacks. In this work, we investigate whether HBM-PIM can serve as a backend for ISA-level matrix acceleration, using the RISC-V Attached Matrix Extension (AME) as a semantic reference. We propose a PEP-based execution model that maps AME element-wise and matrix instructions to HBM-PIM micro-kernels and data instructions in memory operations. Differently from SoA HBM-PIM, we introduce a reduction-free outer-product dataflow that enables accumulation entirely within memory despite the lack of native reduction support. Our approach supports end-to-end execution of element-wise operations, GEMV, and GEMM in PIM mode, minimizing host involvement and off-chip transfers. An experimental evaluation on Samsung Aquabolt-XL shows that AME matrix tile multiplication achieves up to 14.9 GFLOP/s (59.4 FLOP/cycle) on a single HBM pseudo-channel.

#### 1. 核心问题与研究动机
*   **核心问题**：现代处理器受限于“冯·诺依曼瓶颈”，即数据搬运的带宽和能耗限制。虽然高带宽内存（HBM）缓解了带宽压力，但存内计算（PIM）是进一步减少数据移动的关键。然而，现有的商业 HBM-PIM 平台（如三星 Aquabolt-XL）面临指令集受限、缺乏原生归约（Reduction）支持、以及软件栈专用化（难以移植）等挑战。
*   **研究动机**：探讨 HBM-PIM 是否能作为标准指令集架构（ISA）级矩阵扩展（如 RISC-V AME）的后端，从而实现更通用、可移植且高效的存内矩阵加速。

#### 2. 方法论
*   **核心思想**：以 RISC-V 附加矩阵扩展（AME）为语义参考，将 AME 的矩阵和逐元素指令映射到 HBM-PIM 的微内核中。
*   **关键技术细节**：
    *   **PEP 执行模型**：提出“PIM 执行基元”（PIM Execution Primitives, PEP），这是一组封装好的 PIM 原生指令序列，存储在 PIM 单元的命令寄存器文件（CRF）中。
    *   **无归约外积数据流**：针对 HBM-PIM 硬件不支持跨通道归约的限制，设计了一种基于**外积（Outer-product）**的计算策略。通过将第一个源矩阵的一列与第二个源矩阵的一个标量相乘，并直接累加到目标矩阵中，实现了完全在内存内部的累加，避免了将中间结果传回主机进行归约。
    *   **内存布局映射**：将 AME 的分块寄存器（Tile Registers）抽象为内存驻留结构。利用 HBM-PIM 的硬件特性，将输入分块映射到“偶数行地址库（Even Banks）”，将累加寄存器映射到“奇数行地址库（Odd Banks）”。
    *   **指令映射**：实现了 `mfadd`（加）、`mfmul`（乘）、`mfsub`（减，通过乘 -1 模拟）和 `mfmacc`（矩阵乘累加）等 AME 指令。

*   **Benchmark**：针对不同尺寸（如 $128 \times 4096$）的矩阵分块执行 AME 矩阵运算。
*   **对比方法**：

    *   与现有的学术研究（如 MPC-Wrapper、RNN-T 加速器）进行了性能对比，重点在于是否需要外部归约逻辑。

*   **实验规模**：

#### 6. 主要结论与发现
*   **性能表现**：在单伪通道上，`mfmacc` 指令达到了 **14.9 GFLOP/s** 的吞吐量，相当于 **59.4 FLOP/周期**。
*   **效率提升**：通过外积数据流，成功消除了对外部归约引擎的依赖，显著减少了片外数据传输。
*   **扩展性**：计算效率随分块尺寸增加而显著提升，证明该方案非常适合大规模张量运算。
*   **硬件限制**：发现现有 PIM ISA 缺乏比较指令（无法实现 `min/max`）和数据类型拓宽（Widening）支持，这限制了 AME 的完整覆盖。

#### 7. 优点
*   **软硬结合**：首次将标准的 RISC-V AME 语义与商业 HBM-PIM 硬件结合，提升了 PIM 的易用性。
*   **架构创新**：提出的无归约外积数据流巧妙地绕过了硬件设计的固有缺陷。
*   **真实验证**：在实际的 HBM-PIM 芯片上完成评估，而非停留在理论建模阶段。

#### 8. 不足与局限
*   **指令覆盖不全**：由于硬件限制，无法支持 AME 中的比较类指令（如 `mfmax`, `mfmin`）和混合精度/拓宽指令。
*   **固定维度约束**：受限于 PIM 单元与 Bank 的物理连接，矩阵行数被固定为 128，缺乏灵活性。
*   **单通道局限**：目前的评估主要集中在单伪通道，尚未充分展示多通道并行下的复杂同步开销。
*   **软件开销**：虽然减少了数据搬运，但 PEP 的设置（Setup）和主机触发指令仍存在一定的延迟占比。
